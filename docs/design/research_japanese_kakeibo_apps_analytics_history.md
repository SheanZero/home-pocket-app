## 2. 分析画面（Analytics）の比較分析

### 2.1 グラフ種類の比較

| 機能 | Money Forward | Zaim | OsidOri | MoneyNote |
|------|:---:|:---:|:---:|:---:|
| カテゴリ円グラフ | ✅ 大分類+中分類 | ✅ 大分類のみ | ✅ | ✅ |
| 日別棒グラフ | ✅ | ✅ | — | ✅ |
| 月次推移折れ線 | ✅ (年単位) | ✅ | ✅ | ✅ |
| 予算比較バー | ✅ 固定費/変動費分離 | ✅ シンプル直感的 | ✅ | — |
| 資産推移 | ✅ | ✅ (有料) | ✅ | — |
| 収支バランス | ✅ | ✅ | ✅ | ✅ |

### 2.2 各アプリの分析画面の優れた点

#### Money Forward ME
**強み:**
- 大分類・中分類の2レベル円グラフで詳細分析が可能
- 月次/年次データを切り替え表示
- 「残日数×使用金額」から月末予測を自動算出しホーム画面に表示
- 資産・投資口座との統合ビュー

**弱み:**
- グラフが金額順ソートのため、月間比較で視覚的一貫性が低い
- UI情報密度が高く、初心者には複雑
- 無料版は閲覧できるデータ期間に制限あり

#### Zaim
**強み:**
- カテゴリ順固定表示で月間比較が容易（一貫した視覚パターン）
- 予算管理が「シンプルで直感的で見やすい」と高評価
- 縦棒グラフ+点線で予算しきい値を可視化
- インタラクティブな折れ線グラフ（スライドで値表示）
- 色使いが見やすい（緑=余裕、赤=超過の信号パターン）

**弱み:**
- 無料版は残高履歴が1ヶ月分のみ
- 大分類のみの円グラフ（中分類ドリルダウン不可）
- 有料版でないとカスタムカテゴリ追加不可

#### OsidOri
**強み:**
- **共有 vs 個人の二重ビュー** — 家計と個人を分離表示
- 支出分担比率の可視化（割り勘・精算対応）
- 目標貯金の進捗グラフ
- 固定費トラッキングが無料で充実

**弱み:**
- フリーワード検索機能がない
- 無料版はカスタムカテゴリ1個まで
- QRコード決済との連携不可

#### MoneyNote
**強み:**
- 完全無料で全グラフ機能利用可能
- 20種類のテーマカラー + 33種類のアイコン
- カテゴリ別月平均と割合を自動計算
- 極めてクリーンなUI

**弱み:**
- 銀行連携・レシート読み取りなし
- 分析の深さが限定的（サブカテゴリなし）
- 家族共有機能なし

### 2.3 分析画面のUIデザイントレンド（2025-2026）

1. **「つける」→「見える化」への進化** — 単なる記録から洞察提供へ
2. **AI支出分析** — 支出パターンを検知し節約提案
3. **月次レポート自動生成** — PDF化して家族共有
4. **投資口座統合** — NISA・iDeCoとの一体管理
5. **ペア家計簿** — パートナーとの共有分析画面
6. **信号カラーパターン** — 緑(良好)/黄(注意)/赤(超過)の直感的表示

## 3. 履歴画面（Transaction History）の比較分析

### 3.1 表示形式の比較

| 機能 | Money Forward | Zaim | OsidOri | MoneyNote |
|------|:---:|:---:|:---:|:---:|
| タイムライン表示 | ✅ | ✅ | ✅ | ✅ |
| カレンダー表示 | ✅ | ✅ (デイリーメモ付) | — | ✅ |
| カテゴリフィルター | ✅ | ✅ | ✅ | ✅ |
| フリーワード検索 | ✅ | ✅ | ❌ | — |
| タグ/ラベル | — | ✅ (カスタムタグ) | — | — |
| レシート画像添付 | ✅ | ✅ | ✅ | — |
| スワイプ操作 | ✅ | ✅ | ✅ | ✅ |

### 3.2 各アプリの履歴画面の優れた点

#### Money Forward ME
**強み:**
- 大分類・中分類で階層的にトランザクションを整理
- 自動カテゴリ分類の精度が高い
- 月単位/年単位での残高推移を表示

**弱み:**
- アイコンベースのカテゴリ表示が「わかりにくい」
- 情報量が多く一覧性に課題

#### Zaim
**強み:**
- **縦タイムライン表示が見やすい** — カテゴリアイコン(丸バッジ)+口座アイコン
- 「情報量多く一目で理解しやすい」収支履歴一覧
- カスタムタグ（「プレゼント」「ご褒美」等）でグルーピング可能
- デイリーメモ機能（カレンダービュー）
- 必要タップ数が少ない（操作効率が高い）

**弱み:**
- 大量トランザクション時のパフォーマンス
- 高度な検索条件の組み合わせが限定的

#### OsidOri
**強み:**
- **共有/個人の切り替え表示** — パートナーに見せたい支出のみ選択共有
- カード連携の自動分類精度が高い（「ほぼすべて」正確）
- 分担比率と精算情報を履歴と統合表示

**弱み:**
- フリーワード検索がない（履歴検索が不便）
- レシート入力の評価が低い（3.00/5.0）

### 3.3 履歴画面のUIデザインパターン

1. **日付セクション分割** — 日付ヘッダー + その日のトランザクションリスト
2. **カテゴリアイコン** — 丸バッジ + 色分けで視覚的に即認識
3. **金額の右寄せ表示** — tabular figures で桁揃え
4. **収入は青/緑、支出は赤系** — 一般的な色分けルール
5. **カレンダー + リスト切り替え** — 月間俯瞰 ↔ 詳細リスト
6. **スワイプジェスチャー** — 左:削除、右:編集（高速操作）

## 4. Home Pocket への示唆

### 4.1 分析画面への示唆

Home Pocketの**独自性（灵魂账本 + 生存账本の二重帳簿）**を最大限活かすデザイン:

| 機能 | 業界標準 | Home Pocket 差別化ポイント |
|------|---------|--------------------------|
| カテゴリ円グラフ | 支出カテゴリ別 | **灵魂/生存の比率ドーナツ + カテゴリドリルダウン** |
| 月次推移 | 収入 vs 支出 | **灵魂投資推移 + 満足度スコア推移** |
| 予算比較 | カテゴリ別予算 | **灵魂投資目標 vs 実績（自己投資の充実度）** |
| 家族ビュー | 個人 vs 共有切り替え | **家族メンバー別の灵魂/生存バランス比較** |
| AI洞察 | 節約提案 | **「灵魂の充実度」スコアと改善提案** |

### 4.2 履歴画面への示唆

| 機能 | 業界ベストプラクティス | Home Pocket 実装案 |
|------|---------------------|-------------------|
| メインレイアウト | 日付セクション + タイムライン | 同様 + **帳簿タグ（灵魂/生存）バッジ** |
| フィルター | カテゴリ、日付範囲 | + **帳簿タイプフィルター**、満足度フィルター |
| カレンダービュー | 日別合計表示 | + **灵魂の日は特別マーク** |
| 検索 | フリーワード | Zaim式タグ + フリーワード |
| 家族共有 | OsidOri式選択共有 | **家族メンバー別ビュー + 共有選択** |
| タイル表示 | アイコン + 金額 + カテゴリ | + **満足度インジケーター（灵魂のみ）** |

### 4.3 競合にない Home Pocket の差別化要素

1. **灵魂の充実度ダッシュボード** — 自己投資の満足度を時系列で可視化（競合にない）
2. **二重帳簿の比率分析** — 生存支出 vs 灵魂投資のバランスを可視化
3. **家族の灵魂バランス比較** — 家族全員が自分の灵魂投資を大切にしているかの俯瞰
4. **満足度付きトランザクション** — 履歴で各灵魂支出の満足度を振り返り
5. **プライバシーファースト** — ゼロ知識アーキテクチャで安全な家族共有
