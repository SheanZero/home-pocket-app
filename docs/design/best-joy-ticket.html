日期处理对比 · Flutter 落地（_bestJoyValue）

变体日期落点解决「堆叠」落地 / 风险
A 登机牌右侧撕口存根的字段（DATE 标签 + 5/28 + 木）金额留主体、日期移到存根，彻底分区中。需打孔缺口（2 个 card 色圆）+ 虚线竖线。存根固定宽，ja/zh 周字短，安全。最像「票」。
B 日历瓦片左侧日历块（月条+大日号+周），替代图标位日期升级为视觉锚，三元素横向排开不再堆低中。瓦片是纯绘制（Container+Column）。代价：左侧不再显示品类图标——可把品类图标缩小并入标题前（如稿中 🎮 前缀）。信息层级最清晰。
C 邮戳右上角旋转虚线圆戳（5.28 / 木）日期脱离文字流、盖到票面，最有记忆/收藏感中高。Stack + Transform.rotate；dark 下虚线对比要调。body 需留右padding 避让戳。情绪最强但最「重」。
D 撕票两层上层与类型同行、右对齐（带 📅）纵向撕成两层：上=类型+日期，下=金额+满足度低。横虚线 + 两侧缺口即可。金额获得独立大号空间（21px）。最稳、最易落地，仍保留票根味。

共同：左 6px joy 强调条保留票根识别；满足度统一为 档位词+表情 chip（ADR-014）；日期格式沿用 DateFormatter.formatShortMonthDay + 周（A/C 拆成日号与周字分行渲染，需在 presentation 切串、数据层不动）；品类图标 = 一级父类目（parentCategoryIconFromId，B 变体作标题前缀）；字符串走 S.of(context)；金额 AppTextStyles.amount* tabular。
