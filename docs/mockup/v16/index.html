统一记账方案
方案 1 · 单页连续输入 ★ 已选
共享表单和语音底板都不换页，只在原位置更新状态。

聆听、解析和识别结果共用同一页面；停顿自动解析，中央麦克风复用为重新录音，连续记账状态始终保留。

演示状态
安全约束
自动结束只触发解析，不自动保存；解析中不可记录；重录从语音前内容重新开始；金额始终只有一个数据源。
购物项表单
与 App 对齐
名称、数量、用途、类型、分类、预估价格和备注共用新增 / 编辑表单；右上保存是唯一落库动作。
语音建议
语音只填写一个购物草稿，不新开页面、不自动保存，也不会猜测或修改公开 / 私有。

复用设备语音识别、语言和权限能力；购物名称 / 数量 / 价格需要新的专用解析模型。

演示状态
MVP 边界
一次一个商品；3 秒停顿自动解析；中央按钮可提前结束 / 重新录音；识别结果仍需用户检查并点击保存。
编辑交易
与统一记账同源
金额、分类、日期、商家、用途、满足度和备注沿用同一结构；编辑态不出现语音、常驻键盘或连续记账。
编辑专属动作
金额点按后在底部键盘修改；保存固定在底部，删除只保留右上入口并二次确认。
演示状态
不可变信息
编辑既有交易时不改变币种、账本归属、录入来源或隐私类型；外币金额仍单向换算为日元。
