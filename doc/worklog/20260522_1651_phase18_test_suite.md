## 遇到的问题与解决方案

### 问题 1: update_transaction_use_case_test.dart 已存在
**症状:** 计划要求"新建"该文件，但文件已存在且有 19 个测试
**原因:** 早期计划阶段已经创建了部分测试
**解决方案:** 扩展已有文件，添加缺失的 D-20 mock 断言测试组（_MockSyncEngine、_MockChangeTracker）

### 问题 2: D-09 详细行为测试复杂度过高
**症状:** 完整的 voice-correction 测试需要 mock CategorySelectionScreen 导航
**原因:** Phase 18 form widget 测试在 Navigator push 方面存在复杂性
**解决方案:** 改为结构性验证（.edit 模式下不需要 voice-correction 基础设施），详细行为测试留给 Phase 19/22

### 问题 3: flutter analyze 发现新测试文件有 lint 警告
**症状:** `_insertSeed` 本地函数以下划线开头；unused import；doc comment HTML 警告
**原因:** 命名规范和导入整理问题
**解决方案:** 重命名为 `insertSeed`，移除未使用的 `flutter_riverpod/misc.dart` import，重写 doc comment 避免 (a)/(b) HTML 解析歧义
