# 隐私政策

**守护家计簿（Home Pocket）**

最后更新日期：2026年7月1日（草案）

> 【重要·草案】本政策为草案，上线前将由日本法务复核。正式版发布时内容可能更新。

## 1. 基本方针

守护家计簿（以下称"本应用"）是一款本地优先（数据优先保存于设备内）、注重隐私的家计记账应用。本应用采用"零知识（zero-knowledge）"架构设计，开发者及外部服务器均无法读取用户的家计数据。

本政策依据本应用的真实行为，如实说明本应用如何处理用户信息。

## 2. 设备内保存与加密

用户录入的家计数据（交易、金额、分类、备注、照片等）原则上仅保存在用户设备内。数据通过4层加密加以保护：

1. 数据库加密（SQLCipher / AES-256）
2. 字段加密（ChaCha20-Poly1305）
3. 文件加密（照片等 / AES-256-GCM）
4. 传输加密（家庭同步时的设备间端到端加密）

加密密钥在用户设备内管理，开发者既无法访问这些密钥，也无法访问家计数据的内容。

## 3. 关于对外通信

本应用不进行任何以收集、发送家计数据为目的的广告或分析通信。但会发生以下有限的对外通信，现如实披露：

### 3-1. 汇率获取

为处理多币种金额的功能，本应用会连接外部汇率提供服务，获取最新汇率。该通信仅用于获取汇率，不会发送用户的个人信息或家计数据。

### 3-2. 推送通知令牌（仅在启用时）

仅当用户启用推送通知时，用于投递通知的推送令牌会注册至 Google（Firebase Cloud Messaging）。此注册是投递通知所必需的技术处理，不会发送家计数据本身。若不使用推送通知，则不会发生此通信。

## 4. 关于家庭同步

家庭成员之间的数据同步功能，是设备之间直接进行的端到端加密通信。同步数据不会保存至开发者的服务器。

## 5. 关于广告与追踪

本应用不展示广告，不集成任何第三方分析／追踪 SDK，不会收集用户行为记录或进行画像分析。

## 6. 第三方提供

除法律法规要求的情形外，本应用不会向第三方提供用户信息。前述汇率获取及推送通知令牌注册，仅限于提供相应服务所必需的范围。
