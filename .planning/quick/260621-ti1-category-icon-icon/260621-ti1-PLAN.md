## LOCKED — 「上一级类目 icon」取值规则（核心歧义，已定，不交给 executor 决定）

规则：对每个列表行 / 圆环扇区，用其 `categoryId` 调用共享 helper
`parentCategoryIconFromId(categoryId)`（位于
`lib/features/accounting/presentation/utils/category_display_utils.dart`）得到 `IconData`。

为什么这条规则正确且唯一：
1. donut 分类 legend 行的 `entry.value.categoryId` 来自 `rollupCategoryBreakdownsToL1(...)`，已经是 L1（顶层）类目 id（见 category_l1_rollup.dart，rows 已 rollup 到 L1）。
2. 悦己 segment 的 `JoyCategoryAmount.categoryId` 注释明确「已是 L1」（joy_category_amount.dart）。
3. `parentCategoryIconFromId`：L1 id 返回其自身 icon（即顶层类目 icon）；L2 id 返回 parent 的 icon。两条路径下结果都等于「上一级/顶层类目 icon」，且 helper 永不抛异常（未知 id 返回 `Icons.favorite_border`）。
4. 因此对本任务三处 surface 使用同一条规则即可，无需各自判断、无需新 icon 映射、无需新 ARB。

复用优先（禁止重复造轮子）：
- 必须用现有 `parentCategoryIconFromId`。不要新建 icon-name 到 IconData 的映射，不要复制 `category_drill_down_screen.dart` 里那张私有 `_resolveL1IconForCategory` 硬编码表。
- icon 渲染样式参考 `home_hero_card.dart:894-899` 已有写法：`Icon(parentCategoryIconFromId(categoryId), size: 小尺寸, color: palette.语义色)` 加一个 `SizedBox(width: 6~7)` 间距。

成员维度不改：
- donut_hero 的 `_buildMemberMode` legend 行（`leadingEmoji` 头像）保持不变——成员是「人」，显示头像 emoji，不加类目 icon。本任务只动「分类维度」与「悦己」两类 surface。

圆环 icon 叠加方式：
- 用 fl_chart 1.2.0 的 `PieChartSectionData.badgeWidget` 加 `badgePositionPercentageOffset` 在扇区上叠加一个小 `Icon`（已验证 1.2.0 支持 badgeWidget）。icon 颜色用 `palette.card`（与现有 on-ring `titleStyle` 同色，保证在彩色环带上可读）；只对「显示了百分比」的扇区加 badge——被抑制百分比的小扇区（`_onRingPctTitle` 返回空串，即 share 小于 `_onRingPctThreshold` 0.05）与长尾「其他」扇区不加 badge（避免拥挤、保持与 % 一致的露出规则）。
- badge 位置略偏外/偏上于 % 标签：title 在 `titlePositionPercentageOffset: 0.5`，badge 用一个略小于 0.5 的 `badgePositionPercentageOffset`（如约 0.35）使 icon 落在 % 文字上方的环带处。
- icon 尺寸要小（建议 11–12px），避免溢出 30px 环带宽度。
</decisions>
