### Task 2 — Date NLU extension + form wiring + LAST-wins (commit `8a847f7`)

- `voice_text_parser.dart` `_extractRelativeDate`:
  - Added zh keywords: `明天 (+1)`, `后天 (+2)`, `大前天 (-3)`.
  - Added ja keywords: `明日 (+1)`, `あした (+1)`, `あす (+1)`, `明後日 (+2)`, `あさって (+2)`.
  - Switched from first-match-wins to **rightmost-end-position-wins with length tiebreak**. Sort key is `text.lastIndexOf(key) + key.length`; ties prefer longer key so `一昨日` still beats the `昨日` substring it contains, and `大前天` beats `前天`.
  - English keyword block (`yesterday`, `today`, `day before yesterday`) uses the same rule.
- `voice_input_screen.dart` `_stopRecordingAndCommit`: after the existing `state.updateCategory(...)` push, added `if (data.parsedDate != null) state.updateDate(data.parsedDate!);` — null is no-op per silent-fill spec.
- `test/fixtures/voice_corpus_{zh,ja}.dart`: added `VoiceDateCase`/`VoiceDateCaseJa` typedefs + `voiceDateCorpusZh`/`voiceDateCorpusJa` lists (5 cases each: baselines + new keywords + LAST-wins).
- `test/integration/voice/voice_date_corpus_test.dart` (new): iterates both lists with whole-day-offset assertions; `expectedToday` captured once at `setUpAll` to tolerate sub-second clock drift between setUp and parse-time.

## Deviations from plan

- **LAST-wins algorithm refinement**: plan suggested simple `text.lastIndexOf` max. That broke `extracts 一昨日 (Japanese day before yesterday)` because `昨日` is a substring of `一昨日` with lastIndexOf=1 vs `一昨日`'s lastIndexOf=0. Refactored to **end-position + length tiebreak** so longer keys win when they contain shorter keys. Documented in worklog under "Technical Decisions".
- **D-09 test fix**: plan did not foresee the new transcript widget squeezing the test viewport. Added `tester.ensureVisible(merchantField)` to the existing test (Rule 1 — bug fix induced by Task 3 layout change).
- **Mic golden re-baseline**: plan said «if the golden test crops to the mic button only (likely), the golden is mic-only and untouched. Otherwise re-baseline». The matcher actually captures full screen → re-baselined per plan-authorized fallback.
