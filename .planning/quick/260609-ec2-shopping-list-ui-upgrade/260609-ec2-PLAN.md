<objective>
升级购物清单（shopping_list feature）的交互与样式，借鉴第三方 to-do 应用的样式但保留 home-pocket 既有的 全部/日常|悦己/分类 过滤体系。四项需求：① 左侧圆形完成图标（点击 toggle 完成）；② filter 左对齐 + 右侧 ≡ 进入手动拖拽排序模式（拖拽手柄仅排序模式显示）；③ 点击 item 本体进入编辑；④ 右侧显示数量（quantity>1 才显示）。

Purpose: 让购物清单交互更直观（图标 toggle / 整行编辑分离），并把"始终可拖拽"收敛为显式排序模式，减少误触。
Output: 改造后的 tile + filter bar + screen 门控 + 新 `shoppingReorderModeProvider` + 更新测试/golden。

约束（来自 CONTEXT，LOCKED）：
- `quantity` 字段已在 schema（默认 1），**无 Drift migration**，schemaVersion 保持 20。改动集中在 presentation 层。
- 排序模式是纯 UI 状态，不入库。
- 第三方参考图仅借鉴样式/交互，不照搬其过滤语义。
</objective>
