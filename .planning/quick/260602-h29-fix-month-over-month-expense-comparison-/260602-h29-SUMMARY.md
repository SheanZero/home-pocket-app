# Quick Task 260602-h29: Fix Month-over-Month Expense Comparison — Summary

## What Was Built

Fixed the unfair month-over-month comparison in `GetMonthlyReportUseCase._getPreviousMonthComparison`. Previously, a report for the current in-progress month compared against the entire previous month — showing e.g. June 1–2 vs all of May. Now it compares the same period (June 1–2 vs May 1–2).

### Task 1 — TDD: asOf parameter + same-period algorithm (commit: 0c08596a)

Added `DateTime? asOf` to `execute()` and `required DateTime asOf` to `_getPreviousMonthComparison`. The algorithm:

1. Computes `daysInPrevMonth` and `daysInCurMonth` using Dart's `DateTime(y, m+1, 0).day` idiom.
2. If `asOf` is in the current report month (`isCurrentMonth`):
   - If `asOf.day >= daysInCurMonth` (last day of month) → `effectiveDay = daysInPrevMonth` (full prior month).
   - Else → `effectiveDay = min(asOf.day, daysInPrevMonth)` (same-period, clamped).
3. If `asOf` is NOT in the current report month (historical view) → `effectiveDay = daysInPrevMonth` (full prior month, unchanged behavior).

Five TDD test cases added covering: mid-month, last-day, short-month clamp, historical view, cross-year.

### Task 2 — ARB labels + l10n regeneration + golden re-baseline (commit: 666190c3)

Updated `homeHeroPreviousMonthSubline` in all 3 ARB files:
- `app_ja.arb`: `"先月 {amount}"` → `"先月同期 {amount}"`
- `app_zh.arb`: `"上月 {amount}"` → `"上月同期 {amount}"`
- `app_en.arb`: `"Last month {amount}"` → `"Last month (same period) {amount}"`

Ran `flutter gen-l10n`, re-baselined 9 home_hero_card golden masters.
