# Quick Task 260602-h29: Same-period month-over-month expense comparison - Context

**Gathered:** 2026-06-02
**Status:** Ready for planning

<domain>
## Task Boundary

更正「与上个月的对比」：当前实现是 **当前月支出总额 vs 上个月整月支出总额**，对比不公平（本月只走到今天，却和上月整月比）。改为**同期对比**：

- 今天是 N 号 → 本月截止今天的支出总额 vs 上月截止 N 号的支出总额。
- 例：今天 2 号 → 本月(1–2号) vs 上月(1–2号)；今天 15 号 → 本月(1–15) vs 上月(1–15)。
- **边界：今天是当月最后一天** → 对比上月**全月**总额。

涉及代码主路径：`lib/application/analytics/get_monthly_report_use_case.dart` 的
`_getPreviousMonthComparison`（当前把上月按整月 `prevStart=1 日 … prevEnd=月末` 求和）。
该结果经 `MonthlyReport.previousMonthComparison.previousExpenses` / `expenseChange`
流向 `home_hero_card.dart`（`prev`、`trend` 百分比 chip、副标题）与
`state_shadow_books.dart` 的 `prevTotalExpenses` 聚合。
</domain>

<decisions>
## Implementation Decisions (LOCKED — do not revisit)

### D1. 适用范围：仅「当前进行中的月份」
- 仅当报表月份（由 `endDate` 推出的 anchorYear/anchorMonth）== **当前日历月**（由参考时间 now 判定）时，才把上月按「截止今天 day-of-month」截断。
- 查看**已结束的历史月份**时保持原行为：上月**整月 vs 整月**（历史月已走完，全额对比才公平）。
- 该规则对 home 首页与 analytics 历史视图都自动正确（无需调用方区分）。

### D2. 首页副标题文案改为「上月同期」
- 当前 `homeHeroPreviousMonthSubline` 文案是「上个月: ¥X」之类。改后 X 是「上月截止同一天」的金额，文案需体现**同期/同一时间段**的含义（如中文「上月同期」、日文「先月同期」、英文「last month (same period)」）。
- **必须同步更新 `lib/l10n/` 下 ja / zh / en 三个 ARB 文件**，并运行 `flutter gen-l10n` 重新生成。
- 措辞由执行者按各语言习惯定，保持简洁。

### Claude's Discretion (实现细节，已定方向)
- **参考时间注入**：给 `GetMonthlyReportUseCase.execute()` 增加可选参数 `DateTime? asOf`（默认 `DateTime.now()`），`_getPreviousMonthComparison` 用 `asOf` 判定当前月与截止日。保证纯函数可测，避免在用例内裸调 `DateTime.now()` 难以测试。
- **短月/溢出钳制（CRITICAL，必须正确）**：截止日 `cutoffDay` 必须按上月天数钳制——绝不能构造 `DateTime(prevYear, prevMonth, 30)` 这种当 prevMonth 只有 28 天时被 Dart 归一化为下月 2 号的值。
  - `daysInPrevMonth = DateTime(prevYear, prevMonth + 1, 0).day`
  - `daysInCurMonth  = DateTime(curYear,  curMonth  + 1, 0).day`
  - 若 `now` 是当月最后一天（`now.day >= daysInCurMonth`）→ 用上月**整月**（`effectiveDay = daysInPrevMonth`）。
  - 否则 `effectiveDay = min(now.day, daysInPrevMonth)`。
  - `prevEnd = DateTime(prevYear, prevMonth, effectiveDay, 23, 59, 59)`。
- **历史月份判定**：`now.year != anchorYear || now.month != anchorMonth` → 视为已结束 → 上月整月（等价于走 isLastDay 分支）。
- `expenseChange` 百分比基于钳制后的 `prevExpenses` 计算（用例内已从 prevTotals 派生，钳制 prevEnd 即可同时修正总额与百分比）。
- `previousExpenses == 0 && previousIncome == 0` 时仍返回 null（保持现有空态行为）。
</decisions>

<specifics>
## Specific Ideas

- 单测重点（TDD）：固定 `asOf` 注入，覆盖
  1. 月中（如 6/2，asOf=6/2）→ 上月只算 5/1–5/2。
  2. 当月最后一天（如 4/30）→ 上月（3 月）整月，**不是** 3/1–3/30。
  3. 短月钳制（如 asOf=3/30，上月 2 月 28 天）→ 上月整月 2/1–2/28，不溢出到 3 月。
  4. 历史月份（asOf 在更晚月份，报表查看过去某月）→ 整月 vs 整月。
  5. 跨年（1 月 → 上月 12 月，年份 -1）仍正确。
- 现有测试文件：`test/unit/application/analytics/get_monthly_report_use_case_test.dart`、`test/unit/features/analytics/domain/models/monthly_report_test.dart`。新增 asOf 参数后确认旧测试不回归（默认 now 行为对历史 fixture 应保持整月）。
</specifics>
