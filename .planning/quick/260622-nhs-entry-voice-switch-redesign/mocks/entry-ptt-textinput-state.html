文字输入态的「按住说话」显示样式（基于 N1）

聚焦商家 / 备注等文本框时，自定义数字键盘滑走、iOS 系统键盘升起 —— 此时 N1 的底部 「按住说话」长条会被系统键盘顶掉。本稿设计这一态的显示：把「按住说话」搬进系统键盘上方那条浮动工具条 （原本只有 [完成] [记录]），让语音在两种输入态下都「一按即说」、且工具条浮在键盘之上、天然避开 Home 手势区。

一致性原则：数字态 = 键盘上方全宽长条；文字态 = 系统键盘上方工具条里的「按住说话」段。 两态同一个会话逻辑（VoicePttSessionMixin）：按住→收起系统键盘+升聆听浮层→松手填同一张表单·停留。
👆 在右机工具条「按住说话」上按住 → 升起聆听浮层；松手 → 收起

本稿设计

系统键盘上方工具条：完成 · 🎤 按住说话 · 记录

为何这样文本框聚焦时系统键盘顶掉底部长条；把「按住说话」放进原 [完成][记录] 浮动工具条中段（樱粉浅底大段），保持语音一按即达。
为何安全工具条浮在系统键盘之上，离底边远，天然不碰 iOS 上滑手势区。
交互按住「按住说话」→ 收起系统键盘 + 升聆听浮层 → 松手把金额/分类/商家/日期填进同一张表单、回到当前页（不自动保存）。
  • 两种输入态都「一按即说」，体验一致
  • 复用现有浮动工具条（仅在 [完成][记录] 间插入一段），改动小
  • 浮在键盘上方，零手势冲突
  • 工具条变三段，[完成]/[记录] 略窄（仍达标命中区）
注：系统键盘自带的 🎙 是 iOS 听写（逐字录入到文本框），与我们的「按住说话」（整句解析填表）不同，二者并存不冲突。
结论：「按住说话」在两态各有落点 —— 数字态 = 键盘上方全宽长条（N1）；文字态 = 系统键盘上方浮动工具条的中段。 二者同一会话逻辑、同一聆听浮层、同样松手填表停留（D-2）；都避开 iOS 底部手势区。 配色对齐 app_palette.dart（樱粉 #D98CA0 / joyLight #FBEAEF / 记录键 #E09DB4→#D98CA0 / 录音红 #E5484D→#C93040）。
实现提示：给 KeyboardToolbar 在 [完成] 与 [记录] 之间加一个 hold-to-talk 段（按住起停，复用 PTT 会话）； 按住时先 unfocus 文本框收起系统键盘再升浮层。原 TapRegion 分组保留，避免按住时文本框过早失焦。
