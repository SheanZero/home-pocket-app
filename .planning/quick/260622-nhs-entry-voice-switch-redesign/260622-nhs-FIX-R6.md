## BUG 1 — 状态卡「正在聆听」、停了说话没反应 → 改为「停止聆听 + 点击重置重新录入」
**根因:** iOS 连续 re-arm 不可靠。`onStatus`/`_reArmPttListening`/`_reArmAfterTransientError` 在识别器 terminal
（done/notListening/`error_no_match`）后**调用 startListening 并乐观地设 `_listenStatus=listening`**，但 iOS 此时往往
不真正重启 mic → 状态停在「正在聆听」而麦克风已死、再说话无反应。
**改（更可靠的一次性聆听模型，符合用户期望）:**
- **取消无限 re-arm。** 连续会话内一次 `startListening` 监听一段（pauseFor:3s 容忍句内停顿）；当识别器自然终止
  （`done`/终止性 `notListening`/`error_no_match` 静音）→ **`_isRecording=false`、`_listenStatus=stopped`，不再 re-arm**。
- **面板状态实时正确:** stopped 时标题显示「停止聆听」(灰)、麦克风停止脉冲（降为非录音态）。
- **新增提示「点击重置重新录入」**：stopped 时在麦克风下方显示（新 ARB `voiceTapResetToRerecord` ×ja/zh/en + gen-l10n），
  与「轻点空白处退出」并存或替换，告诉用户点「重置」可重新录。
- **面板可见性与 `_isRecording` 解耦:** 现 build 用 `pttIsRecording ? VoiceRecordPanel : 键盘`，stopped 后 `_isRecording=false`
  会让面板消失、键盘回来 —— **错误**。引入 `_voiceModalOpen`（点开=true，轻点空白退出/关闭=false），build 用它做面板可见性；
  status(聆听/解析/停止) 与 isRecording 独立。退出 = 关面板保留内容；重置 = `resetPttSessionAndRestart` 全新录音(status→聆听)。
- `error_no_match` 仍**不弹 toast**(R5 已对)，但现在走 stopped+提示，而不是 re-arm。

## BUG 2 — 说「99999日元」解析填成「9」
**根因(假设):** R4 的 **partial 实时填表**把早期 partial（如「…用了9」）填进金额=9；final「今天买手机，一共用了99999日元」
（转写含逗号「99,999日元」）应覆盖为 99999 但没有 —— 可能 `_fillFormFromText` 里 `amount = _mergedAmount ?? data.amount`
取了**陈旧/错误**的值，或解析器对**逗号分组大数/连续数字**解析成 9。是 partial-fill 引入的回归（用户称「老问题又出现」）。
**改（先调查、repro 驱动）:**
- 加 repro 测试：transcript「今天买手机，一共用了99999日元」与「今天买手机，一共用了99,999日元」**必须填 99999**（非 9）。
- 定位真因并修：
  - **final 必须权威覆盖**：final 解析得到的金额一定覆盖之前 partial 填入的值（不被陈旧 `_mergedAmount` 或 `_lastFilledAmount` 挡住）。
  - 若解析器/merger 对「99999」「99,999」(逗号)解析错误 → 修解析（去逗号分组、正确解析多位连续数字）。
  - 若 partial 填表是元凶 → 可让 **partial 填分类/商家/日期，但金额以 final 权威值为准**（或 partial 金额也填但 final 必覆盖），
    取「实时」与「正确」的平衡；以 repro 测试为准绳。
- 不回归外币 triple / JPY-native / 满意度 / merger 其他用例。
