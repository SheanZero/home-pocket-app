# Quick Task 260622-nhs · 修复 R3 — 真机三个 bug

**日期:** 2026-06-22 · 前置 R2（`440afe73,19ee8f62,d9ad48d9`）真机发现 3 个问题。

## BUG 1 — 长条太高 + 键盘下方留白过大；语音记录键应融入键盘
**现状:** `VoiceRecordBar`（`hold_to_talk_bar.dart`）height 52 + `margin(16,0,16,8)`，是键盘上方一张独立卡片；
键盘整体包在 `SafeArea(top:false)` 里，底部留白过大。
**改:**
- **减小长条高度**（52 → ~36–40），并**融入键盘**：去掉与键盘之间的间距（或贴在键盘顶部成为其一部分），
  视觉上是键盘的一条窄顶条，而非悬在上方的独立大卡片。可降低边框/底色强度让它像键盘组成部分。
- **键盘下方留白恢复到加语音前**：当前 `SafeArea` 造成的底部空白过大 —— 收紧到只够避开 Home 指示条的最小间距
  （对照 R1 之前 SmartKeyboard 的原始底距；不要双重 padding）。键盘底排仍不被 Home 指示条压住即可。

## BUG 2 — 点击「重置」后不再接收语音
**根因:** `_onVoiceReset`（manual screen）→ `resetPttSessionState()` 只清空 transcript/merger/parse 缓冲，
**不重启识别器**。当识别器此时已自终止（pauseFor/done）且未处于 re-arm 周期时，重置后就不再聆听。
**改:** 重置后**保证继续聆听** —— `_onVoiceReset` 在清空缓冲后**主动 re-arm**（确保 `_continuousActive == true && _isRecording == true`，
并在识别器未在听时调用一次 `startListening`/`_reArmPttListening`）。在 mixin 暴露一个安全的 `restartPttListening()`（幂等：仅当未在听时启动）供重置调用。
加测试：重置后 session 仍 active 且能再次填表。

## BUG 3 — 语音弹窗改为「就地替换键盘」，去浮层、去背景灰
**现状:** `VoiceListeningModal`（`voice_listening_overlay.dart`）= `Positioned.fill` Stack + 0.34 alpha 遮罩(scrim) + 底部 sheet，
浮在全屏之上、表单被灰掉。
**改:**
- **就地替换键盘**：录音中（`pttIsRecording`）时，在键盘所在的底部槽位**用语音面板替换 [长条 + SmartKeyboard]**，
  占据同一footprint（表单上方不跳动/不 reflow）。
- **去掉 scrim**（不灰背景，表单保持清晰可见，便于看自动填表）。
- **去掉浮层**：不再用外层 Stack 的 `Positioned.fill` overlay；把 `VoiceListeningModal` 改为内联面板
  `VoiceRecordPanel`（无遮罩、无 bottom-sheet 圆角阴影 chrome —— 就是键盘容器里的内容：正在聆听 / 转写 / 波形 /
  录音红麦克风 / 「轻点空白处退出」/ 「重置·恢复账目」）。
- **退出**：无 scrim 后，「轻点空白处退出」= 点面板空白区（非重置键）触发 `onExit`；重置键点击不冒泡退出（保持现有 stopPropagation 语义）。
- 从外层 Stack 移除 `if (pttIsRecording) VoiceListeningModal(...)`；在 `AnimatedSlide/SafeArea` 内联三元切换：
  `pttIsRecording ? VoiceRecordPanel(...) : Column([slim VoiceRecordBar, SmartKeyboard])`。
