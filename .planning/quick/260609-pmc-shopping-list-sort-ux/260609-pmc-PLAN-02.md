<objective>
Two refinements to the sort-mode UX from on-device review of plan-01.

Purpose:
1. The move-to-top/bottom buttons used plain `keyboard_arrow_up/down` chevrons, which read
   ambiguously. Replace with the bar-line variants (`vertical_align_top` / `vertical_align_bottom`)
   so the "jump to the top/bottom edge" meaning is obvious.
2. The dragged-item proxyDecorator wrapped the tile in a TRANSPARENT Material with elevation.
   Because the tile's own Container paints no background, the elevation shadow bleeds through the
   transparent body and the whole row looks gray while dragging. The user wants only an edge shadow
   on a clean (opaque) card — see reference screenshot (white card, soft top/bottom shadow, no tint,
   no colored border).
</objective>

### shopping_list_screen.dart — proxyDecorator (lines ~173-201)
```dart
proxyDecorator: (child, index, animation) {
  return AnimatedBuilder(
    animation: animation,
    builder: (ctx, _) {
      final double elevation = Tween<double>(begin: 0, end: 6)
          .animate(CurvedAnimation(parent: animation, curve: Curves.easeOut)).value;
      return Material(
        elevation: elevation,
        color: Colors.transparent,          // <-- CAUSE OF GRAY BLEED-THROUGH
        child: DecoratedBox(
          decoration: BoxDecoration(
            border: Border(left: BorderSide(color: ctx.palette.borderInputActive, width: 6)), // <-- REMOVE
          ),
          child: child,
        ),
      );
    },
  );
},
```

The tile root (`shopping_item_tile.dart` line ~133) is a `Container` with ONLY a left
`BorderSide` accent (width 4) and NO background color — it relies on the scaffold/list
background showing through. So a transparent elevated Material drops a shadow under a
see-through body → the gray look.

### shopping_item_tile.dart — move buttons (lines ~393, ~426)
Currently `Icons.keyboard_arrow_up` (line ~393) and `Icons.keyboard_arrow_down` (line ~426),
size 20, `color: palette.textSecondary`, inside the `_buildTrailingCluster` move buttons.

`palette.card` is the established opaque surface token (already used in this file at lines
~78, ~287). Light mode = warm-white card; dark mode = the dark card color. ThemeExtension
accessor `context.palette` / `ctx.palette` resolves it.
