<objective>
Foreign-currency (non-JPY) amounts that are whole numbers currently render with a
useless ".00" suffix. The screenshot shows the transaction-detail edit headline as
"12,211.00 USD"; it must read "12,211 USD". The same defect exists in the foreign
keypad seed string. Fractional amounts (12.50) and JPY (0-decimal) amounts must be
unaffected.

Root cause: TWO call-sites duplicate identical minor→major formatting that always
uses `toStringAsFixed(decimals)`:
  1. `transaction_edit_screen.dart:95` `_minorToMajorString` → drives the headline
     string fed to `AmountDisplay` (which re-appends the literal decimal part).
  2. `amount_edit_bottom_sheet.dart:112` `_initialEditStr` → seeds the keypad in
     currency-aware mode.

`AmountDisplay._formatted` only adds comma grouping and re-emits whatever decimal
part it receives — so the fix must drop the all-zero decimal part at the SOURCE
string, not in the widget.

Purpose: surgically remove ".00" for whole-number foreign amounts at both the
display and edit entry points, via ONE shared helper, without touching JPY,
fractional amounts, the derived-JPY row, or base-currency displays elsewhere.

Output: a shared `formatMinorAsMajor` helper in `currency_conversion.dart`, both
call-sites delegating to it, and unit tests proving the trim + non-trim cases.
</objective>
