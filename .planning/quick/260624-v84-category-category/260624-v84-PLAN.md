# 260624-v84 — 进入「选择分类」自动滚动到已选一级类目

## 背景

用户在「添加账目」表单中已选分类（如 `水电燃气 > 燃气/煤油费`）。点击「分类」行进入 `CategorySelectionScreen`（标题「选择分类」）时，页面是一个整页的一级类目 `ListView`（点开 L1 展开 L2 chip）。当前 `_loadCategories` 已经会**自动展开**已选分类所属的一级类目（`_expandedL1Id`），但列表始终停在顶部，没有滚动到该一级类目 —— 当已选类目靠下时用户看不到当前选择。

## 目标行为

进入页面时，如果 `selectedCategoryId` 非空且能解析到所属一级类目，则在首帧布局完成后自动滚动，使该一级类目对齐到视口顶部（其展开的 L2 chip 随即可见）。无已选分类时保持原状。

## 实现方案

纯 UI 行为增强，单文件为主 + 一个 widget 测试。不改动分类数据模型、选择逻辑或 `categoryId` 写入路径。

- 复用 `_loadCategories` 中已解析出的「已选分类所属 L1」（即 `_expandedL1Id`），将其记入一次性字段 `_pendingScrollL1Id`。
- 给 `_buildReadBody` 中 `l1.id == _pendingScrollL1Id` 的那个 `_CategoryGroup` 挂一个 `GlobalKey`（给 `_CategoryGroup` 构造函数补 `super.key`）。
- `setState`（`_isLoading=false`）之后，若 `_pendingScrollL1Id != null`，用 `WidgetsBinding.instance.addPostFrameCallback` 在首帧后调用 `Scrollable.ensureVisible(key.currentContext, alignment: 0.0, duration: 300ms, curve: easeInOut)`，随后把 `_pendingScrollL1Id` 置空（一次性）。
- `Scrollable.ensureVisible` 自动向上查找 `ListView` 的 Scrollable，无需新增 `ScrollController`，也无需预估 item 高度（对展开后的高度天然正确）。
- 边界：`currentContext == null`（目标未构建/stale id）直接 return；列表过短自动 clamp 到 maxScrollExtent；已在顶部则偏移为 0，不抖动。
