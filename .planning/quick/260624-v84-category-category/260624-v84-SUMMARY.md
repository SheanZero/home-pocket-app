# 260624-v84 — 进入「选择分类」自动滚动到已选一级类目

## 成果

进入 `CategorySelectionScreen`（标题「选择分类」）时，如果表单已带有已选分类（`selectedCategoryId` 非空），列表会在首帧后自动滚动，使该分类所属的一级类目对齐到视口顶部，其已自动展开的二级 chip 也随即可见。无已选分类或 id 失效时保持原行为（停在顶部）。

## 变更

### `lib/features/accounting/presentation/screens/category_selection_screen.dart`
- 新增字段 `_pendingScrollL1Id`（一次性滚动目标）、`_selectedGroupKey`（GlobalKey）、`_scrollController`。
- `_loadCategories`：在解析出已选分类所属 L1（复用既有 `_expandedL1Id` 自动展开逻辑）后记入 `_pendingScrollL1Id`；`setState` 之后注册 `addPostFrameCallback` 触发滚动。
- 新增 `_scrollToSelectedGroup()`：两阶段滚动 —— 阶段 1 用估算行高 `jumpTo` 让 `ListView.builder` 把目标 group 布局出来；阶段 2 在下一帧用 `Scrollable.ensureVisible(alignment: 0.0, 250ms)` 精确对齐到顶部。完成后清空 `_pendingScrollL1Id`（一次性，不干扰用户后续手动滚动）。
- `_buildReadBody`：给 `ListView.builder` 挂 `controller: _scrollController`；给 `l1.id == _pendingScrollL1Id` 的 `_CategoryGroup` 挂 `_selectedGroupKey`。
- `_CategoryGroup` 构造函数补 `super.key`。
- `dispose()` 释放 `_scrollController`。

### `test/widget/.../category_selection_screen_test.dart`
- 新增 group「auto-scroll to selected category」：
  - 选中靠下的分类时，断言 ListView 滚动偏移 > 0 且目标一级类目可见、接近顶部。
  - 无已选分类时，断言滚动偏移为 0（不滚动）。

## 设计决策 / 踩坑

- **首选 `Scrollable.ensureVisible`**（无需预估高度、对展开后高度天然正确），但 `ListView.builder` 懒构建导致首帧时远处的目标 group 尚未构建，`GlobalKey.currentContext == null`，单纯 ensureVisible 不滚动（实测 offset 恒为 0）。
- **试过 `cacheExtent: double.infinity`**（强制全部布局）→ 被 viewport 布局断言拒绝，抛出数千异常并污染同文件其它测试。已弃用。
- **最终方案：`ScrollController` 两阶段 jump→ensureVisible**。估算行高仅用于把目标拉进构建范围，最终对齐由 ensureVisible 精确完成，对可变高度（目标展开 L2）无副作用。
