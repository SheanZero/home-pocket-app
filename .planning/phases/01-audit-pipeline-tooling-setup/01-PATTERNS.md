## Shared Patterns (cross-cutting)

### Generated-file exclusion

**Source:** `analysis_options.yaml` lines 4–6 (verbatim today):
```yaml
analyzer:
  exclude:
    - "**/*.g.dart"
    - "**/*.freezed.dart"
```

**Apply to:**
- All audit scanners (`scripts/audit/*.dart`) MUST skip these globs when enumerating files.
- `merge_findings.dart` MUST drop any shard finding whose `file_path` ends with `.g.dart`, `.freezed.dart`, or `.mocks.dart` as defense-in-depth (RESEARCH §"Code Examples — Example 3" `_generatedFileGlobs`, lines 634–638).
- AI subagent prompts MUST include a "Do NOT scan generated files (`*.g.dart`, `*.freezed.dart`, `*.mocks.dart`)" directive (RESEARCH §"Code Examples — Example 4").

### Process invocation pattern

**Source:** RESEARCH §"Architecture Patterns — Pattern 1" + §"Code Examples — Example 1" lines 400–421:
```dart
final result = await Process.run(
  'dart',
  ['run', 'custom_lint', '--format=json'],
  runInShell: true,
);
final lints = jsonDecode(result.stdout as String) as List<dynamic>;
```

**Apply to:** every `scripts/audit/*.dart` that wraps an external CLI tool. Note Pitfall P1-8: read `result.stdout` ONLY (custom_lint prints status to stderr).

### Plain-Dart-class immutability (no codegen)

**Source:** `lib/shared/utils/result.dart` (CONVENTIONS lines 152–162) + RESEARCH §"Code Examples — Example 2" callout "Avoid `@freezed` here":
```dart
class Result<T> {
  final T? data;
  final String? error;
  final bool isSuccess;
  bool get isError => !isSuccess;

  factory Result.success(T? data) => Result._(data: data, isSuccess: true);
  factory Result.error(String message) => Result._(error: message, isSuccess: false);
}
```

**Apply to:** `scripts/audit/finding.dart`. Plain Dart class with `final` fields, named constructor, `toJson()` / `fromJson` factory. Keeps `scripts/` build_runner-free (Pitfall P1-7 + RESEARCH §"Code Examples — Example 2" lines 612–613).

### Import organization

**Source:** CONVENTIONS lines 41–58:
1. `dart:` core libraries.
2. blank line, then `package:` external imports.
3. blank line, then **relative** imports (inside `lib/`) OR `package:home_pocket/...` (inside `test/`).

**Apply to:** every `scripts/*.dart` and `test/scripts/*.dart` Phase 1 creates. Scripts under `scripts/` use relative imports for sibling files (`import 'audit/finding.dart';` — RESEARCH Example 3 line 625). Tests under `test/scripts/` use relative imports back into `scripts/` (no package URI exists for scripts/).

### Shell script header

**Source:** `bash` standard pattern + RESEARCH Example 1:
```bash
#!/usr/bin/env bash
# scripts/<name>.sh
# <one-line description>
set -euo pipefail
```

**Apply to:** all 4 `scripts/audit_*.sh` wrappers + `scripts/test_audit_pipeline.sh` + `scripts/test_idempotency.sh`. Strict mode (`-euo pipefail`) is mandatory.

### Print-for-CLI vs `dev.log` for runtime

**Source:** CONVENTIONS lines 178–181 + analysis_options.yaml line 14 (`avoid_print: false`):
- Runtime code uses `dart:developer` `dev.log(...)`.
- CLI scripts use `print(...)` — `avoid_print: false` permits this in `scripts/`.

**Apply to:** all `scripts/*.dart`. Use `print(...)` for CLI summary output (matches `scripts/arb_to_csv.dart` line 34: `print('Exported ${sortedKeys.length} keys to docs/i18n/translations.csv');`).
