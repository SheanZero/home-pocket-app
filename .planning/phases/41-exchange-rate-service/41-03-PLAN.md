    Define five variants (names from RESEARCH.md Pattern 2):

    RateFetched — fresh from API:
    - `required String rate` (full-precision string, ADR-020)
    - `required String currency`
    - `required DateTime rateDate` (requested date)
    - `DateTime? actualDate` (non-null when API returned different date — RATE-05)
    - `required String source` ('frankfurter' | 'fawazahmed0')

    RateCached — from exact-date cache hit:
    - `required String rate`
    - `required String currency`
    - `required DateTime cachedDate`
    - `required String source`
    - `bool isManualOverride = false` (ADR-022 D-02 flag)

    RateFallback — offline fallback (most-recent API-cached row):
    - `required String rate`
    - `required String currency`
    - `required DateTime cachedDate` (staleness indicator for RATE-03)

    RateManual — manual-only fallback (D-07 lowest priority):
    - `required String rate`
    - `required String currency`
    - `required DateTime cachedDate`

    RateUnavailable — no rate at all (D-08):
    - `required String currency`

    Each variant has a `const` constructor. The sealed base has `const RateResult()`. No JSON serialization needed (this is a runtime-only discriminated union).

    Add a `rateSignal` concept: `GetExchangeRateUseCase` will attach signals via a wrapper. To keep RateResult variants simple, define a separate `RateSignal` sealed class in the same file:
    - `sealed class RateSignal { const RateSignal(); }`
    - `final class RateSignalDialog extends RateSignal` — fields: `required String currency`, `required String oldRate`, `required String newRate` (ADR-022 D-02)
    - `final class RateSignalToast extends RateSignal` — fields: `required int oldJpy`, `required int newJpy` (ADR-022 D-03)

    Also define a `RateResultWithSignal` wrapper class (not sealed, plain Dart):
    - `final RateResult result`
    - `final RateSignal? signal`
    - `const RateResultWithSignal({required this.result, this.signal})`

    This keeps RateResult variants clean while allowing GetExchangeRateUseCase to attach ADR-022 signals.

    4. Try fawazahmed0 Cloudflare: GET `https://$dateStr.currency-api.pages.dev/v1/currencies/jpy.min.json` with `.timeout(const Duration(milliseconds: 1000))`.

       - Any failure: throw `ExchangeRateApiException('All sources failed for $currency on $dateStr')`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ExchangeRateApiClient → external APIs | Outbound HTTPS GET; no user data in URL; response JSON must be validated before use |
| API JSON response → Dart double | rawRate is an untrusted number; must guard rawRate == 0 and non-finite before inversion |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-05 | Information Disclosure | ExchangeRateApiClient URL construction | mitigate | URL contains only YYYY-MM-DD date + ISO currency code; no user ID, book ID, or amount. Enforced by unit test asserting URL regex (SC-5). kDebugMode guard prevents full URL logging in release builds (PATTERNS.md §kDebugMode guard) |
| T-41-06 | Tampering | API JSON response parsing | mitigate | Null-safe cast `(rates?[currency] as num?)?.toDouble()`; guard `rawRate == null \|\| rawRate == 0` → throw before inversion (avoids division by zero → Infinity stored as rate string) |
| T-41-07 | Denial of Service | Network timeout per source | mitigate | Per-source `.timeout(const Duration(milliseconds: 1500))` caps wait per source; total chain ≤ ~3s (D-04 per CONTEXT.md) |
| T-41-08 | Spoofing | fawazahmed0 CDN response | accept | Free public API; no auth possible; rate is informational (user can override). Risk accepted as stated in RESEARCH.md Security Domain |
| T-41-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan (connectivity_plus already installed in Plan 02) |
