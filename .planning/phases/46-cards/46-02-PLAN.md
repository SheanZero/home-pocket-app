<objective>
Build the two JOY-side DATA paths round-5 B needs that no existing query provides: (a) per-L1-category joy AMOUNT (the segment weights for the 悦己花在哪 horizontal stacked bar, D-C2) and (b) per-day joy COUNT (the 小确幸 calendar heatmap depth, D-C1). Both are pure Dart transforms over the existing `findByBookIds(ledgerType: LedgerType.joy)` primitive + the locked L1 rollup helper — zero new DAO, zero migration (schema stays v21).

The joy AMOUNT rolls up through the SAME `l1AncestorOf` rule the donut uses (D-11) so joy segments are a verifiable strict subset of the donut's L1 amounts.

Purpose: Provide the application/data layer the Wave-2 joy cards consume (46-05). Keep this strictly data — no widget/registry work.
Output: `GetJoyCategoryAmountsUseCase`, `GetPerDayJoyCountsUseCase`, their models, and two auto-dispose providers.
