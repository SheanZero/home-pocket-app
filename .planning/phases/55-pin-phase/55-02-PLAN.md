<objective>
Fix the single highest-impact correctness bug in the phase (LOCK-10): `BiometricService._handlePlatformException` catches `on PlatformException` and switches on string codes (`'LockedOut'`, `'NotEnrolled'`…) that local_auth 3.0.1 NEVER emits. local_auth 3.x throws `LocalAuthException` with a `LocalAuthExceptionCode` enum; as written, every biometric error escapes uncaught and ejects the user out of their own data. Rewrite to catch `LocalAuthException`, map ALL enum codes plus a wildcard to a PIN-fallback outcome.

Purpose: Guarantee the lock UI can ALWAYS land on the PIN page (LOCK-05/10), never a dead end.
Output: Rewritten error handling + a regression test asserting the NEW exception model.
</objective>

- (Optional) `auth_result.dart` left structurally intact (6 variants retained for telemetry per RESEARCH Open Question 1) — the UI mapping for every non-success variant is PIN fallback.
- Rewritten `biometric_service_test.dart` asserting the `LocalAuthException` model (all 14 codes → fallbackToPIN; wildcard; residual net).
