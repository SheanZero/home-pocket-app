RESEARCH Pitfall #1 (load-bearing):
- Drift's `migrator.addColumn(transactions, transactions.entrySource)` emits ONLY the column itself with inline column constraints (`.text().withDefault(...)`). It does NOT apply the table-level `customConstraints` CHECK to existing rows in an upgraded database.
- The recommended workaround is `customStatement` with column-level inline CHECK in the same ALTER TABLE ADD COLUMN statement (SQLite supports this: https://sqlite.org/lang_altertable.html).
- ALSO add the entry to `customConstraints` so fresh-install path (Drift creates the table from scratch via `customConstraints`) produces equivalent CHECK on the table.
- Both paths MUST be exercised by the test (test the migrated-from-v16 path is harder — see Task 3 deferred sub-step).
