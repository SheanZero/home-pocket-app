       ```dart
       /// Extracts the monetary amount from a text string.
       ///
       /// Supports:
       /// 1. Arabic numerals: 「680円」「¥1280」「1,280」 — always tried first.
       /// 2. Locale-routed kanji/kana state machines:
       ///    - localeId starts with 'ja' → JapaneseNumeralStateMachine
       ///    - localeId starts with 'zh' → ChineseNumeralStateMachine
       ///    - null or other localeId → try ja then zh as defensive fallback
       ///
       /// Returns null if no amount is found.
       int? extractAmount(String text, {String? localeId}) {
         // Priority 1: Arabic numerals (locale-independent)
         final arabicAmount = _extractArabicAmount(text);
         if (arabicAmount != null) return arabicAmount;

         // Priority 2: locale-routed numeral state machines
         if (localeId != null && localeId.startsWith('ja')) {
           return _jaMachine.parse(text);
         }
         if (localeId != null && localeId.startsWith('zh')) {
           return _zhMachine.parse(text);
         }
         // Fallback (null locale or unsupported) — try ja then zh.
         return _jaMachine.parse(text) ?? _zhMachine.parse(text);
       }
       ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-05-T | Tampering | Locale spoofing (caller passes wrong localeId) | mitigate | Defensive fallback in `extractAmount` — if localeId is null or unmatched, falls back to ja-then-zh. Both machines return null on locale-mismatched input (per Plan 20-03 / 20-04 normalize: unrecognised chars → empty token stream). Final null is propagated cleanly; UI cannot misread a parser result as a valid amount. |
| T-20-05-E | Elevation of Privilege | Application → Infrastructure dependency direction | accept | Application layer importing Infrastructure is the canonical dependency direction per CLAUDE.md 5-layer Clean Architecture. `import_guard` custom_lint catches reverse imports automatically. |
| T-20-05-D | Denial of Service | extractAmount called repeatedly on long text | accept | Each call is O(N) (Plans 20-03 / 20-04 threat models). UseCase callsite is `execute(text)` once per voice-recognition result; merger (Plan 20-08) calls once per committed buffer. No unbounded loops. |

Threat surface intentionally minimal — application transfer station with no I/O.
</threat_model>
