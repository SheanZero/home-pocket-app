## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-06 | Tampering | transactions table migration | mitigate | Three nullable columns added via customStatement without DEFAULT; no data written to existing rows; verifyChain unaffected (ADR-021 confirmed: hash formula unchanged) |
| T-40-07 | Tampering | exchange_rates table | mitigate | Table lives in SQLCipher AES-256-CBC encrypted AppDatabase (inherited by architecture); no separate crypto needed |
| T-40-08 | Tampering | migration idempotency | mitigate | `CREATE INDEX IF NOT EXISTS` guard prevents double-execution errors on partial migration retries |
| T-40-09 | Tampering | hash chain integrity across migration | mitigate | ADR-021 decision (Plan 40-02): currency columns excluded from hash formula; verifyChain will pass on pre-migration rows with null currency fields |
</threat_model>
