<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sync wire → fromSyncMap | Adversarial peer may send payloads with partial-triple (1 or 2 currency fields) |
| CreateTransactionUseCase | Validates partial-triple BEFORE DB write — the domain tamper gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-13 | Tampering | TransactionSyncMapper.fromSyncMap | mitigate | all three new fields use `as T?` null-safe cast (Pitfall 4 guard); a v1.6 payload with no currency keys produces null on all three fields — no exception, no data loss |
| T-40-14 | Tampering | partial-triple sync manipulation | mitigate | CreateTransactionUseCase returns Result.error before DB write when hasCurrencyField && !hasAllCurrencyFields; prevents analytics SUM poisoning from a single-field foreign currency row |
| T-40-15 | Tampering | appliedRate FormatException | mitigate | convertToJpy calls double.parse(appliedRate); a non-parseable appliedRate string throws FormatException; CreateTransactionUseCase should wrap the convertToJpy call in try/catch and return Result.error on FormatException (add this in the execute() DB-write path when all three fields are non-null) |
| T-40-16 | Tampering | hash chain scope | mitigate | ADR-021 confirmed: calculateTransactionHash signature does NOT include originalCurrency/originalAmount/appliedRate; architecture test (domain_import_rules_test.dart) guards this |
</threat_model>
