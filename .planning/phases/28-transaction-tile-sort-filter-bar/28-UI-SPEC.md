## Spacing Scale

Same 8-point scale as Phase 27 (inherited; do not redefine per phase). Declared usages for Phase 28 components:

| Token | Value | Usage in This Phase |
|-------|-------|---------------------|
| xs | 4dp | Gap between tag and merchant text in tile; gap between category label and satisfactionIcon; gap between chip label icon and text |
| sm | 8dp | Gap between merchant and category text in tile (`SizedBox(height: 2)` already in `HomeTransactionTile` for category sub-line; `SizedBox(width: 8)` after tag); inter-chip gap in filter bar |
| md | 16dp | Tile horizontal padding (inherited from `HomeTransactionTile`: `horizontal: 10` existing — do not change); filter bar horizontal screen-edge padding; category sheet horizontal padding |
| lg | 24dp | Category sheet top padding above header; spacing between day-header text and first tile in group |
| xl | 32dp | n/a in this phase |
| 2xl | 48dp | Minimum touch target for all interactive bar controls (chips, direction arrow icon button, search icon button); delete confirmation dialog action buttons |
| 3xl | 64dp | n/a in this phase |

**Exceptions:**
- Tile row padding: `EdgeInsets.symmetric(horizontal: 10, vertical: 14)` is inherited from `HomeTransactionTile` verbatim. Do not change it — altering would break visual consistency with the Home screen tile.
- Day-header height: 32dp (`SizedBox(height: 32)`) — explicitly compact to not steal vertical space from tiles.
- Filter bar height: 44dp fixed (`SizedBox(height: 44)` wrapping the `SingleChildScrollView`) — enough for chip tap targets with `EdgeInsets.symmetric(horizontal: 8, vertical: 6)` chip padding.
- Chip vertical padding & bar scroll vertical padding: 6dp — required to fit 32dp chips inside the 44dp pinned bar (32 + 2×6 = 44) while preserving touch-target clearance; raising to 8dp would force bar height to 48dp.
- Swipe delete background: red zone right-side icon has `EdgeInsets.only(right: 16)` padding (confirmed from RESEARCH.md Pattern 2).
- Category sheet tristate checkbox row height: 48dp minimum (Material touch-target floor).

### C-01 `ListTransactionTile` (`list_transaction_tile.dart`)

Wraps `HomeTransactionTile` in a `Dismissible` and adds an `onTap` for navigation. Placed in `lib/features/list/presentation/widgets/`.

**Structure:**
```
Dismissible (key: ValueKey(tx.transaction.id), direction: endToStart)
├── background: Container(red, Alignment.centerRight, padding right 16)
│   └── Icon(Icons.delete, color: AppColors.card, size: 20)
└── child: HomeTransactionTile(
    tagText: ledger short label,
    tagBgColor: survivalLight / soulLight,
    tagTextColor: survival / soul,
    merchant: tx.transaction.merchant ?? tx.transaction.note ?? '—',
    category: CategoryLocalizationService.resolveFromId(categoryId, locale),
    categoryColor: survival / soul,
    formattedAmount: NumberFormatter.formatCurrency(amount, currencyCode, locale),
    amountColor: AppColors.textPrimary,
    satisfactionIcon: _satisfactionIcon(tx),   // optional; soul rows only; ADR-014
    onTap: () => _onTap(context, ref, tx),
  )
```

**Tile layout (inherited from `HomeTransactionTile` verbatim):**
```
Row(horizontal: 10, vertical: 14)
├── Tag badge (3dp radius, 6/1 padding) — micro 10dp w700
├── SizedBox(width: 8)
├── Expanded Column
│   ├── merchant — bodyMedium 14dp w500 textPrimary
│   └── Row(min)
│       ├── category — caption 12dp w500 ledger-color
│       └── [satisfactionIcon if soul] — 14dp AppColors.soul
└── amount — amountSmall 15dp w700 textPrimary (tabular)
```

**Row time display (D-09):** The tile does NOT show the date (date is in the day-header). The tile shows **transaction time** only, appended after the category label OR as a trailing sub-caption. Recommended placement: second sub-line below merchant, replacing or alongside category. See below for exact slot.

Revised tile layout for Phase 28 (adds time to `HomeTransactionTile` pattern):
```
Row(horizontal: 10, vertical: 14)
├── Tag badge
├── SizedBox(width: 8)
├── Expanded Column
│   ├── merchant — bodyMedium 14dp w500 textPrimary
│   └── Row(spaceBetween)
│       ├── Row(min)  — category + satisfactionIcon
│       │   ├── category label — caption 12dp w500 ledger-color
│       │   └── [satisfactionIcon 14dp soul, if applicable]
│       └── time — caption 12dp w500 textSecondary (e.g. "14:32")
└── amount — amountSmall 15dp w700 textPrimary
```

Time format: `DateFormat('HH:mm', locale.toString()).format(tx.transaction.timestamp)` — do NOT use `DateFormatter.formatDateTime` (that includes date). This is a direct `intl` call, acceptable for time-only display.

**Row divider:** `Divider(height: 1, thickness: 1, color: AppColors.borderList)` between tiles within a day group. No divider between the last tile of one group and the day-header of the next (the day-header's background provides visual separation).

**`Dismissible` interaction (ROW-02 / D confirmed in RESEARCH.md):**
- Direction: `DismissDirection.endToStart` (right-to-left swipe = left swipe on LTR screens). Right swipe = no-op.
- `confirmDismiss`: calls `showDialog<bool>` (AlertDialog — see C-04 below). Returns `Future<bool?>`. Null = cancel (user dismissed dialog via back button).
- `onDismissed`: calls `ref.read(deleteTransactionUseCaseProvider).execute(tx.transaction.id)` fire-and-forget + `ref.invalidate(listTransactionsProvider(bookId: bookId))`. SnackBar shown via `ScaffoldMessenger.of(context)` called BEFORE any await. Context access is safe here because the Scaffold is still alive.
- Key: `ValueKey(tx.transaction.id)` — required to prevent Flutter assertion errors on list rebuild.

**SC#1 reconciliation (D-09 note):** The tile does not display the full date — it shows time only. The day-group header (C-02) satisfies SC#1's "transaction date formatted via `DateFormatter`" requirement. If the verifier requires per-row date display, add a compact date badge at caption size as a fallback (do not change the day-header).

**satisfactionIcon mapping (soul rows, Claude's Discretion — follow `HomeTransactionTile` precedent):**
Apply the ADR-014 mapping. Show `satisfactionIcon` on soul-ledger rows only. Survival rows pass `satisfactionIcon: null`.

### C-02 `ListDayGroupHeader` (inline widget or separate file)

Sticky-style day section header rendered above each day's tile group in the `ListView`.

**Layout:**
```
Container(
  color: AppColors.backgroundMuted,      // #F5F4F2 — subtle strip
  height: 32,
  padding: EdgeInsets.symmetric(horizontal: 16),
  child: Align(
    alignment: Alignment.centerLeft,
    child: Text(
      DateFormatter.formatDate(dayDate, locale),  // e.g. "2026/05/30" ja → "2026年5月30日"
      style: AppTextStyles.caption.copyWith(color: AppColors.textSecondary),
    ),
  ),
)
```

**Date format per locale:**
- ja: `DateFormat('M月d日(E)', 'ja')` → "5月30日(金)" — day-of-week in parens for kakeibo convention
- zh: `DateFormat('MM月dd日', 'zh')` → "05月30日"
- en: `DateFormat('MMM d, EEEE', 'en')` → "May 30, Friday"

Implementation note: use `DateFormatter.formatDate(date, locale)` if the project's `DateFormatter` accepts a `DateTime` — otherwise construct `DateFormat` directly with locale. Do not call `DateFormatter.formatDateTime` (that includes time).

**Grouped-by-day assembly (D-09 / D-10):**

Group `List<TaggedTransaction>` → `List<_DayGroup>` → flatten to `List<_ListItem>` (sealed `_HeaderItem` / `_RowItem`) → `ListView.builder`. Day groups sorted by `filter.sortConfig.sortDirection` (desc = newest day first; asc = oldest first). Within-day order comes from `listTransactionsProvider` SQL ORDER BY.

```
ListScreen Column
├── CalendarHeaderWidget (Phase 27, fixed)
├── ListSortFilterBar (C-03, fixed 44dp)
└── Expanded
    └── ListView.builder
        ├── _HeaderItem → ListDayGroupHeader
        └── _RowItem → ListTransactionTile
```

#### C-03c Ledger Chips (FILTER-02)

Three chips: All / 生存 / 魂. Mutually exclusive single-select (All = no ledger filter; selecting 生存 or 魂 sets ledger filter; re-selecting active chip clears it = returns to All).

| Chip | Active state | Inactive state |
|------|-------------|----------------|
| All | `survivalLight` bg + no border (neutral) OR `backgroundMuted` bg | `card` bg + `borderDefault` border |
| 生存 | `survivalLight` bg + `survival` border + `survival` text | `card` bg + `borderDefault` border + `textSecondary` text |
| 魂 | `soulLight` bg + `soul` border + `soul` text | `card` bg + `borderDefault` border + `textSecondary` text |

"All" chip is active when `filter.ledgerType == null`. Tapping active 生存/魂 chip → `setLedgerFilter(null)` (back to All). Tapping inactive 生存/魂 → `setLedgerFilter(LedgerType.survival)` / `setLedgerFilter(LedgerType.soul)`.

Chip height inside bar: constrained to 32dp via `materialTapTargetSize: MaterialTapTargetSize.shrinkWrap` — the bar itself is 44dp so the tap target is extended by the outer container.

Labels: "すべて" / "生存" / "魂" (ja, used as placeholder; Phase 30 ARB keys).

## Interaction Contract

### Tile Tap (ROW-01 / SC#2)

| Trigger | Result | State mutation |
|---------|--------|----------------|
| Tap tile row | `Navigator.push<bool>(TransactionEditScreen(transaction: tx.transaction))` | None immediately |
| Edit screen `pop(true)` (save) | `ref.invalidate(listTransactionsProvider(bookId: bookId))` + `ref.invalidate(calendarDailyTotalsProvider(bookId, year, month))` | Provider invalidated → reactive rebuild |
| Edit screen `pop(false)` or back (no save) | No invalidate needed | None |

`TransactionEditScreen` constructor: `required Transaction transaction` (verified: `transaction_edit_screen.dart:28`). Pass `tx.transaction` (the decrypted domain model). Never serialize note/amount to route params.

### Swipe Delete (ROW-02 / SC#3)

| Trigger | Result | State mutation |
|---------|--------|----------------|
| Left swipe (endToStart) > threshold | Red delete zone revealed (full-row background) | None |
| Swipe release before threshold | Tile snaps back | None |
| Swipe confirmed → `confirmDismiss` fires | `showDialog<bool>` (AlertDialog) | None |
| Dialog "キャンセル" | Dialog closes, tile snaps back (`confirmDismiss` returns `false`) | None |
| Dialog "削除" | `confirmDismiss` returns `true`; `onDismissed` fires | `DeleteTransactionUseCase.execute(id)` + `ref.invalidate(listTransactions + calendarTotals)` |
| Right swipe (startToEnd) | No-op (direction locked to `endToStart`) | None |

### Sort Chip Interactions (SORT-01..04 / SC#4)

| Trigger | Result |
|---------|--------|
| Tap sort chip | `showMenu` with 3 field options (date/edit-time/amount) + checkmark on active |
| Select field in menu | `listFilterProvider.notifier.setSort(config.copyWith(sortField: selected))` |
| Tap direction arrow | `listFilterProvider.notifier.setSort(config.copyWith(sortDirection: toggled))` |

Sort chip label **always** reflects active field name (SC#4 hard requirement from D-05). Arrow **always** reflects active direction.

### Ledger Filter Chips (FILTER-02)

| Trigger | Precondition | Result |
|---------|-------------|--------|
| Tap "All" chip | Any ledger filter active | `setLedgerFilter(null)` |
| Tap "生存" chip | Not selected | `setLedgerFilter(LedgerType.survival)` |
| Tap "魂" chip | Not selected | `setLedgerFilter(LedgerType.soul)` |
| Tap active "生存" or "魂" chip | Already selected | `setLedgerFilter(null)` (back to All) |

### Category Filter (FILTER-03 / SC#5)

| Trigger | Result |
|---------|--------|
| Tap Categories chip | `showModalBottomSheet(CategoryFilterSheet)` |
| Tap "Apply" in sheet | `listFilterProvider.notifier.setCategories(localSelected)` + sheet closes |
| Tap "Cancel" in sheet | No provider change, sheet closes |
| Tap "Clear" in sheet header | Clears local selection (`_localSelected = {}`) — does NOT immediately update provider |
| Tap L1 row (none → all) | Select all L2 children in `_localSelected` |
| Tap L1 row (all → none) | Deselect all L2 children in `_localSelected` |
| Tap L1 row (partial → all) | Select all L2 children (partial → all) |
| Tap L2 checkbox | Toggle single L2 id in `_localSelected` |

### Search Expand (FILTER-01 / D-06)

| Trigger | Result |
|---------|--------|
| Tap search icon | `_searchExpanded = true`; `AnimatedContainer` expands to 160dp; `TextField` autofocuses |
| Type in search field | `listFilterProvider.notifier.setSearch(query)` on each change |
| Submit / dismiss keyboard | `_searchExpanded = false` if field empty; stay expanded if non-empty |
| Tap clear (×) in search field | `setSearch('')`; `_searchExpanded = false` |

### Clear All (FILTER-04 / D-07)

| Trigger | Result |
|---------|--------|
| Tap "Clear" chip (conditional) | `listFilterProvider.notifier.clearAll()` — resets `activeDayFilter`, `ledgerType`, `categoryIds`, `searchQuery`. Month selection preserved. |

`clearAll()` also collapses search if expanded: `setState(() { _searchExpanded = false; _searchController.clear(); })`.

### Day-Filter Active State (D-10)

When `activeDayFilter != null` (user tapped a day in the calendar — Phase 27 interaction), the list renders a single day group (one `_HeaderItem` + its tiles). The bar's "Clear" chip is visible (day filter counts as an active filter). The calendar highlight is handled by Phase 27's `CalendarHeaderWidget` — no additional tile-level indicator needed.
