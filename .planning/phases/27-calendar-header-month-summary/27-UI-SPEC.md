## Component Inventory

### C-01 CalendarHeaderWidget (`list_calendar_header.dart`)

Top-level widget placed above the transaction list in `ListScreen`. A `ConsumerWidget` that reads `calendarDailyTotalsProvider` and `listFilterProvider`.

**Structure (top to bottom):**

```
CalendarHeaderWidget
├── [Row] Month Nav Bar
│   ├── IconButton (chevron_left, 48dp touch target)
│   ├── GestureDetector / TextButton (month label → jump to today's month)
│   └── IconButton (chevron_right, 48dp touch target)
├── TableCalendar (full-month grid, always expanded)
│   └── calendarBuilders.defaultBuilder → DayCell (custom)
└── SummaryRow
    ├── Month total line (always visible)
    └── Day subline (visible only when activeDayFilter != null)
```

**Month Nav Bar:**
- Height: 48dp row
- Month label: `AppTextStyles.titleMedium`, centered, tappable (calls `listFilterProvider.notifier.selectMonth(today.year, today.month)`)
- Chevron icons: `Icons.chevron_left` / `Icons.chevron_right`, 24dp, `AppColors.textTertiary`, 48dp tap target
- Tap left chevron → `selectMonth(year, month - 1)` (use a DateTime to roll year boundaries: `DateTime(year, month - 1)` then read `.year` / `.month`, so Jan→prev correctly yields Dec of prior year)
- Tap right chevron → `selectMonth(year, month + 1)` (same DateTime rollover for Dec→Jan)
- Month label tap → `selectMonth(DateTime.now().year, DateTime.now().month)`

### C-02 DayCell (custom `CalendarBuilders.defaultBuilder`)

Replaces the default `table_calendar` cell. Renders inside each grid square.

**Layout:**
```
Column(mainAxisSize: min, mainAxisAlignment: center, crossAxisAlignment: center)
├── Text(day number) — bodySmall 13dp, w500, color per state
└── Text(compact amount) — micro 10dp, w700, color per state  [only if dayTotal > 0]
```

**States:**

| State | Container decoration | Number color | Amount color |
|-------|---------------------|--------------|--------------|
| Default (no amount, not today) | transparent | `AppColors.textPrimary` | — (hidden) |
| Has amount (not today, not selected) | transparent | `AppColors.textPrimary` | `AppColors.textSecondary` |
| Today (not selected) | `AppColors.accentPrimaryLight` fill + 1dp `AppColors.accentPrimaryBorder` border, radius 6dp | `AppColors.textPrimary` | `AppColors.textSecondary` |
| Selected (any day) | `AppColors.accentPrimary` fill, radius 6dp | `AppColors.card` (white) | `AppColors.card` (white) |
| Today + Selected | `AppColors.accentPrimary` fill (selection wins), radius 6dp | `AppColors.card` (white) | `AppColors.card` (white) |
| Out-of-month (prev/next month overflow) | transparent, opacity 0.35 | `AppColors.textTertiary` | — (hidden) |

**Cell sizing:** `rowHeight: 52dp` in `TableCalendar` configuration. Cell width is auto-distributed across 7 columns.

**Compact amount format:** `NumberFormatter.compact(amount, locale: currentLocale, currencyCode: currencyCode)` — produces "1.2万" (ja/zh) or "1.2k" (en). Never shows "¥0" for empty days (the `dayTotal > 0` guard suppresses the second Text entirely).

**Day-total lookup (executor precision):** Cells read their amount from the normalized map (see `## Data Normalization`). Look up `dailyMap[DateTime.utc(day.year, day.month, day.day)]` (or whatever normalization the provider commits to — the cell and the provider MUST agree on key normalization, otherwise every lookup misses and all cells render blank).

**Day tap interaction (toggle):**
- Tap any day where `!isSameDay(activeDayFilter, tappedDay)` → `listFilterProvider.notifier.selectDay(tappedDay)` (highlights day, filters list in Phase 28+)
- Tap the already-selected day → `listFilterProvider.notifier.selectDay(null)` (clears day filter, removes highlight)
- Canonical expression: `final f = ref.read(listFilterProvider); isSameDay(f.activeDayFilter, tappedDay) ? notifier.selectDay(null) : notifier.selectDay(tappedDay)` — use `table_calendar`'s `isSameDay` helper, never `==` on raw `DateTime` (time-of-day components would break equality).

### C-03 SummaryRow

Horizontal row below the calendar grid, separated by 1dp `AppColors.borderDivider` top border.

**Padding:** 16dp horizontal, 12dp vertical.

**Month total line (always visible):**
```
Row(mainAxisAlignment: spaceBetween)
├── Text(label) — AppTextStyles.caption, AppColors.textSecondary [placeholder; ARB Phase 30]
└── Text(amount) — AppTextStyles.amountSmall (15dp, w700, tabularFigures), AppColors.textPrimary
```
Amount: sum of all values in the normalized daily-totals map → `NumberFormatter.format(total, ...)` (full format, not compact: "¥12,345").

**Day subline (conditional, shown when `activeDayFilter != null`):**
```
Row(mainAxisAlignment: spaceBetween)
├── Text(date label) — AppTextStyles.caption, AppColors.textSecondary [DateFormatter output]
└── Text(day amount) — AppTextStyles.caption, AppColors.textSecondary
```
Day amount: `dailyMap[normalize(activeDayFilter)] ?? 0`, formatted via `NumberFormatter.format(...)`. Reuses the same map as the cells (D-07: no extra query).

**Animation:** `AnimatedSize` (height collapse) on the day subline when it appears/disappears. Duration: 200ms, curve: `Curves.easeInOut`.

### C-04 TableCalendar configuration

```dart
TableCalendar(
  firstDay: DateTime(2020, 1, 1),   // practical lower bound
  lastDay: DateTime(2030, 12, 31),  // practical upper bound
  focusedDay: focusedDay,           // driven by listFilterProvider.selectedYear/Month
  calendarFormat: CalendarFormat.month,    // always full month (D-05)
  availableCalendarFormats: const { CalendarFormat.month: '' },  // lock format — no toggle
  startingDayOfWeek: _localeStartingDay(currentLocale),  // locale-aware (see below)
  locale: currentLocale.toLanguageTag(),   // table_calendar locale param
  headerVisible: false,             // custom nav bar replaces built-in header (C-01)
  calendarBuilders: CalendarBuilders(
    defaultBuilder: _buildDayCell,
    todayBuilder: _buildDayCell,    // reuse same builder; state param distinguishes today
    selectedBuilder: _buildDayCell, // reuse same builder
    outsideBuilder: _buildDayCell,  // out-of-month days
  ),
  selectedDayPredicate: (day) => isSameDay(day, activeDayFilter),
  onDaySelected: _onDayTapped,
  onPageChanged: _onPageChanged,    // see de-duplication note below
  rowHeight: 52,
  daysOfWeekHeight: 20,
)
```

**`startingDayOfWeek` locale rule:**
- `ja` locale → `StartingDayOfWeek.sunday` (日曜始まり is conventional in Japan)
- `zh` locale → `StartingDayOfWeek.sunday` (中文 calendars also Sunday-start by convention)
- `en` locale → `StartingDayOfWeek.monday` (ISO 8601 week standard)

**Nav-callback de-duplication (executor precision):** Both the chevron handlers AND `onPageChanged` (native swipe) end at `selectMonth`. To avoid double-mutation / fighting `focusedDay`:
- Drive `focusedDay` *from* `listFilterProvider` (`DateTime(selectedYear, selectedMonth)`), making the provider the single source of truth.
- In `onPageChanged`, call `selectMonth(focusedDay.year, focusedDay.month)` — idempotent if the value already matches (no-op rebuild).
- Chevrons mutate the provider; the calendar re-renders from the new `focusedDay`. Do NOT additionally call `_pageController.animateToPage` from chevrons — let the rebuilt `focusedDay` move the page, so there is exactly one mutation path.

## Interaction Contract

### Month Navigation (CAL-01 / SC#1)

| Trigger | Result | State mutation |
|---------|--------|----------------|
| Tap left chevron | Calendar advances to previous month | `selectMonth(year, month-1)` → also clears `activeDayFilter` (existing `selectMonth` behavior, verified `state_list_filter.dart:26`) |
| Tap right chevron | Calendar advances to next month | `selectMonth(year, month+1)` → clears `activeDayFilter` |
| Horizontal swipe on calendar grid | Same as chevron (table_calendar native swipe; `onPageChanged` callback fires) | Same `selectMonth` call in `onPageChanged` (idempotent — see C-04 de-dup note) |
| Tap month label text | Jumps to current real month (today's year+month) | `selectMonth(DateTime.now().year, DateTime.now().month)` |

### Day Tap Filter (CAL-03 / SC#3)

| Trigger | Precondition | Result |
|---------|-------------|--------|
| Tap day cell | Day is not currently selected | Day is highlighted (accent fill), `selectDay(tappedDay)` called |
| Tap day cell | Day is currently selected (toggle clear) | Highlight removed, `selectDay(null)` called, month view restored |

`selectDay(null)` is the verified clear path (`state_list_filter.dart:35`).

### Data Flow (CAL-02 / SC#2, CAL-04 / SC#4)

- `calendarDailyTotalsProvider(bookId: bookId, year: selectedYear, month: selectedMonth)` → `AsyncValue<Map<DateTime, int>>`
- The provider internally calls `analyticsRepository.getDailyTotals(bookId:, startDate:, endDate:, type: 'expense')`. It converts `(year, month)` → `(startDate, endDate)` via `DateBoundaries.monthRange(year, month)` (Phase 24 util; localtime-safe boundaries), then folds the returned `List<DailyTotal>` into the normalized map.
- Provider only re-computes when `(bookId, year, month)` changes. It does NOT watch `listFilterProvider` filter state (Pitfall 3 — prevents keystroke-driven calendar re-renders in Phase 28+).
- `getDailyTotals` is expense-only by default (`type = 'expense'`, verified `analytics_dao.dart:230`), satisfying SC#2/SC#4 expense-only basis without extra filtering.
- Loading state: day cells show no amount (graceful degradation — cells still render date numerals); summary amount slot shows a shimmer placeholder (see States section).
- Error state: summary row shows error label (see Copywriting Contract); day cells show no amount.

## Data Normalization

`getDailyTotals` returns `List<DailyTotal>` where each `DailyTotal.date` is parsed from the SQL `DATE(..., 'localtime')` day string (i.e. midnight-local of that day) and `.totalAmount` is an `int` (minor units). The provider MUST fold this into a `Map<DateTime, int>` with a **single, documented key-normalization rule** that the day cells and the day-subline lookup both reuse:

- Normalize every key to a date-only DateTime (strip time-of-day): e.g. `DateTime(d.year, d.month, d.day)`.
- When the cell/subline looks up a day, apply the identical normalization to `tappedDay` / `activeDayFilter` before indexing.
- Rationale: `table_calendar` hands builder callbacks a `DateTime` that may carry a non-zero time component; a map keyed on raw DateTimes would miss on `==`. Normalization + a shared helper (`DateTime _dayKey(DateTime d)`) is the contract — if the provider and the cell disagree, all cells silently render blank (the highest-risk failure mode for this phase).

Month total (SC#4 / D-11) = `map.values.fold(0, (a, b) => a + b)` — single source of truth, guaranteed same expense-only basis as the cells.

## States and Loading Skeleton

### Calendar Loading State

While `calendarDailyTotalsProvider` is `AsyncLoading`:
- Calendar grid renders normally (date numerals visible)
- Day cells show no compact amount (empty slot — no spinner per cell)
- Summary row amount slot shows a `SizedBox(width: 60, height: 15)` shimmer placeholder (solid `AppColors.backgroundMuted` `#F5F4F2`, radius 4dp)

### Calendar Error State

When `calendarDailyTotalsProvider` is `AsyncError`:
- Day cells show no amount (silent degradation — calendar nav still works)
- Summary row shows error copy (see Copywriting Contract)
- No full-screen error overlay (calendar remains navigable)

### Empty Month State

When `calendarDailyTotalsProvider` returns an empty map (no expenses this month):
- All day cells show no amount (correct per SC#2)
- Summary row shows "¥0" formatted via `NumberFormatter` (not a separate empty state widget — zero is valid data). The `LIST-03` empty-state copy is a Phase 30 / Phase 28 list concern, not this widget.

## Amount Formatting Rules

Per CLAUDE.md mandate — `NumberFormatter` must be used for all monetary values. Never use raw `intl.NumberFormat` directly in widgets.

| Slot | Format | Example (JPY) | Example (USD) |
|------|--------|---------------|---------------|
| Day cell compact amount | `NumberFormatter.compact(amount, locale, currencyCode)` | 1.2万 | 1.2k |
| Summary month total | `NumberFormatter.format(amount, locale, currencyCode)` | ¥12,345 | $123.45 |
| Summary day subline | `NumberFormatter.format(amount, locale, currencyCode)` | ¥3,400 | $34.00 |

**Amount style:** `AppTextStyles.amountSmall` (15dp, w700, `FontFeature.tabularFigures()` baked in) for month total only. Day compact uses `AppTextStyles.micro` (10dp, w700, color-overridden to `textSecondary`) without the `amountSmall` token — cells are too narrow for the amountSmall font size.
