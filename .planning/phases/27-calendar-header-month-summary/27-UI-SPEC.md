| Property | Value |

## Spacing Scale

Declared values (multiples of 4 only; Flutter `dp` = logical pixels):

| Token | Value | Usage in This Phase |
|-------|-------|---------------------|
| xs | 4dp | Inline gaps between day-cell date numeral and compact amount; chevron icon padding |
| sm | 8dp | Summary row label-to-amount gap; calendar top/bottom internal padding |
| md | 16dp | Horizontal screen padding (calendar edges flush to 16dp inset); summary row vertical padding |
| lg | 24dp | Section gap between calendar grid bottom and summary row container; gap above calendar |
| xl | 32dp | n/a in this phase |
| 2xl | 48dp | Touch target floor for prev/next chevron buttons (iOS/Android minimum) |
| 3xl | 64dp | n/a in this phase |

Exceptions:
- Chevron icon buttons: minimum 48dp touch target (`SizedBox(width: 48, height: 48)`) per iOS HIG / Material touch-target guidelines, regardless of visual icon size (24dp icon centered in 48dp target).
- Day cell height: driven by `table_calendar`'s `rowHeight` parameter; set to **52dp** to accommodate date numeral (20dp font) + compact amount (10dp font) + 4dp internal gap without overflow in a ~40dp-wide cell.

## Component Inventory

### C-01 CalendarHeaderWidget (`list_calendar_header.dart`)

Top-level widget placed above the transaction list in `ListScreen`. A `ConsumerWidget` that reads `calendarDailyTotalsProvider` and `listFilterProvider`.

**Structure (top to bottom):**

```
CalendarHeaderWidget
├── [Row] Month Nav Bar
│   ├── IconButton (chevron_left, 48dp touch target)
│   ├── GestureDetector / TextButton (month label → jump to today's month)
│   └── IconButton (chevron_right, 48dp touch target)
├── TableCalendar (full-month grid, always expanded)
│   └── calendarBuilders.defaultBuilder → DayCell (custom)
└── SummaryRow
    ├── Month total line (always visible)
    └── Day subline (visible only when activeDayFilter != null)
```

**Month Nav Bar:**
- Height: 48dp row
- Month label: `AppTextStyles.titleMedium`, centered, tappable (calls `listFilterProvider.selectMonth(today.year, today.month)`)
- Chevron icons: `Icons.chevron_left` / `Icons.chevron_right`, 24dp, `AppColors.textTertiary`, 48dp tap target
- Tap left chevron → `listFilterProvider.selectMonth(year, month - 1)` (TableCalendar `focusedDay` updates)
- Tap right chevron → `listFilterProvider.selectMonth(year, month + 1)`
- Month label tap → `listFilterProvider.selectMonth(DateTime.now().year, DateTime.now().month)` + reset `focusedDay` to today

### C-02 DayCell (custom `CalendarBuilders.defaultBuilder`)

Replaces the default `table_calendar` cell. Renders inside each grid square.

**Layout:**
```
Column(mainAxisSize: min, crossAxisAlignment: center)
├── Text(day number) — 13dp, w500
└── Text(compact amount) — 10dp, w700, AppTextStyles.micro  [only if amount > 0]
```

**States:**

| State | Container decoration | Number color | Amount color |
|-------|---------------------|--------------|--------------|
| Default (no amount, not today) | transparent | `AppColors.textPrimary` | — (hidden) |
| Has amount (not today, not selected) | transparent | `AppColors.textPrimary` | `AppColors.textSecondary` |
| Today (not selected) | `AppColors.accentPrimaryLight` fill + 1dp `AppColors.accentPrimaryBorder` border, radius 6dp | `AppColors.textPrimary` | `AppColors.textSecondary` |
| Selected (any day) | `AppColors.accentPrimary` fill, radius 6dp | `AppColors.card` (white) | `AppColors.card` (white) |
| Today + Selected | `AppColors.accentPrimary` fill (selection wins), radius 6dp | `AppColors.card` (white) | `AppColors.card` (white) |
| Out-of-month (prev/next month overflow) | transparent, opacity 0.35 | `AppColors.textTertiary` | — (hidden) |

**Cell sizing:** `rowHeight: 52dp` in `TableCalendar` configuration. Cell width is auto-distributed across 7 columns.

**Compact amount format:** `NumberFormatter.compact(amount, locale: currentLocale, currencyCode: currencyCode)` — produces "1.2万" (ja/zh) or "1.2k" (en). Never shows "¥0" for empty days.

**Day tap interaction:**
- Tap any day → if not already selected: `listFilterProvider.selectDay(tappedDay)` (highlights day, filters list in Phase 28+)
- Tap already-selected day → `listFilterProvider.selectDay(null)` (clears day filter, removes highlight)
- Toggle logic: `ref.read(listFilterProvider).activeDayFilter == tappedDay ? selectDay(null) : selectDay(tappedDay)`

### C-03 SummaryRow

Horizontal row below the calendar grid, separated by 1dp `AppColors.borderDivider` top border.

**Padding:** 16dp horizontal, 12dp vertical.

**Month total line (always visible):**
```
Row
├── Text(label) — AppTextStyles.caption, AppColors.textSecondary [placeholder; ARB Phase 30]
└── Text(amount) — AppTextStyles.amountSmall (15dp, w700, tabularFigures), AppColors.textPrimary
```
Amount: sum of all values in `calendarDailyTotalsProvider` map → `NumberFormatter.format(total, ...)` (full format, not compact: "¥12,345").

**Day subline (conditional, shown when `activeDayFilter != null`):**
```
Row
├── Text(date label) — AppTextStyles.caption, AppColors.textSecondary [DateFormatter output]
└── Text(day amount) — AppTextStyles.caption, AppColors.textSecondary
```
Day amount: `calendarDailyTotalsProvider.value?[activeDayFilter] ?? 0`, formatted via `NumberFormatter.format(...)`.

**Animation:** `AnimatedSize` or `AnimatedSwitcher` on the day subline — cross-fade or height collapse when subline appears/disappears. Duration: 200ms, curve: `Curves.easeInOut`.

### C-04 TableCalendar configuration

```dart
TableCalendar(
  firstDay: DateTime(2020, 1, 1),   // practical lower bound
  lastDay: DateTime(2030, 12, 31),  // practical upper bound
  focusedDay: focusedDay,           // driven by listFilterProvider.selectedYear/Month
  calendarFormat: CalendarFormat.month,    // always full month (D-05)
  availableCalendarFormats: const { CalendarFormat.month: '' },  // lock format — no toggle
  startingDayOfWeek: _localeStartingDay(currentLocale),  // locale-aware (see below)
  locale: currentLocale.toLanguageTag(),   // table_calendar locale param
  headerVisible: false,             // custom nav bar replaces built-in header (C-01)
  calendarBuilders: CalendarBuilders(
    defaultBuilder: _buildDayCell,
    todayBuilder: _buildDayCell,    // reuse same builder; state param distinguishes today
    selectedBuilder: _buildDayCell, // reuse same builder
    outsideBuilder: _buildDayCell,  // out-of-month days
  ),
  selectedDayPredicate: (day) => isSameDay(day, activeDayFilter),
  onDaySelected: _onDayTapped,
  onPageChanged: (focusedDay) { /* update listFilterProvider.selectMonth */ },
  rowHeight: 52,
  daysOfWeekHeight: 20,
)
```

**`startingDayOfWeek` locale rule:**
- `ja` locale → `StartingDayOfWeek.sunday` (日曜始まり is conventional in Japan)
- `zh` locale → `StartingDayOfWeek.sunday` (中文 calendars also Sunday-start by convention)
- `en` locale → `StartingDayOfWeek.monday` (ISO 8601 week standard)

## Interaction Contract

### Month Navigation (CAL-01 / SC#1)

| Trigger | Result | State mutation |
|---------|--------|----------------|
| Tap left chevron | Calendar advances to previous month | `listFilterProvider.selectMonth(year, month-1)` → also clears `activeDayFilter` (existing `selectMonth` behavior) |
| Tap right chevron | Calendar advances to next month | `listFilterProvider.selectMonth(year, month+1)` → clears `activeDayFilter` |
| Horizontal swipe on calendar grid | Same as chevron (table_calendar native swipe; `onPageChanged` callback fires) | Same selectMonth call in `onPageChanged` |
| Tap month label text | Jumps to current real month (today's year+month) | `listFilterProvider.selectMonth(DateTime.now().year, DateTime.now().month)` |

### Day Tap Filter (CAL-03 / SC#3)

| Trigger | Precondition | Result |
|---------|-------------|--------|
| Tap day cell | Day is not currently selected | Day is highlighted (accent ring), `listFilterProvider.selectDay(tappedDay)` called |
| Tap day cell | Day is currently selected (toggle clear) | Highlight removed, `listFilterProvider.selectDay(null)` called, month view restored |

### Data Flow (CAL-02 / SC#2, CAL-04 / SC#4)

- `calendarDailyTotalsProvider(bookId: bookId, year: selectedYear, month: selectedMonth)` → `AsyncValue<Map<DateTime, int>>`
- Provider only re-computes when `(bookId, year, month)` changes. It does NOT watch `listFilterProvider` filter state (Pitfall 3 — prevents keystroke-driven calendar re-renders in Phase 28+).
- Loading state: day cells show no amount (graceful degradation — cells still render with date numerals only; `CircularProgressIndicator` in summary row amount slot, 16dp size).
- Error state: summary row shows error label (see Copywriting Contract below); day cells show no amount.

## States and Loading Skeleton

### Calendar Loading State

While `calendarDailyTotalsProvider` is `AsyncLoading`:
- Calendar grid renders normally (date numerals visible)
- Day cells show no compact amount (empty slot — no spinner per cell)
- Summary row amount slot shows a `SizedBox(width: 60, height: 15)` shimmer placeholder (solid `AppColors.backgroundMuted`, radius 4dp)

### Calendar Error State

When `calendarDailyTotalsProvider` is `AsyncError`:
- Day cells show no amount (silent degradation — calendar nav still works)
- Summary row shows error copy (see Copywriting Contract)
- No full-screen error overlay (calendar remains navigable)

### Empty Month State

When `calendarDailyTotalsProvider` returns an empty map (no expenses this month):
- All day cells show no amount (correct per SC#2)
- Summary row shows "¥0" formatted via `NumberFormatter` (not a separate empty state widget — zero is valid data)

## Copywriting Contract

All copy is delivered via `S.of(context)` (ARB keys). Phase 30 finalizes tri-lingual copy (ja/zh/en). Phase 27 uses placeholder ARB keys with English defaults where new keys are needed.

| Element | ARB key | ja placeholder | zh placeholder | en placeholder |
|---------|---------|----------------|----------------|----------------|
| Month summary label | `calMonthTotal` | 今月の支出 | 本月支出 | Monthly Spend |
| Day subline label | `calDayTotal` | {date}の支出 | {date}支出 | {date} Spend |
| Data error (summary row) | `calLoadError` | データを読み込めません | 无法加载数据 | Unable to load data |

**Primary CTA:** None — this phase has no submit/save action. The interactive elements are navigation-only (chevrons, day tap).

**Empty state:** Zero-spend month shows "¥0" in summary row — no separate empty state copy in this phase. Phase 30 handles `LIST-03` empty state.

**Error state copy:** `calLoadError` appears in the summary row amount slot as `AppTextStyles.caption` / `AppColors.textSecondary` text when the provider returns an error. No retry button — stale calendar data is non-critical; user can navigate away and back.

**Destructive actions:** None in this phase.

**Month label format:**
- ja: `DateFormatter` → "2026年5月" (use `DateFormat('yyyy年M月', 'ja')`)
- zh: `DateFormatter` → "2026年05月" (use `DateFormat('yyyy年MM月', 'zh')`)
- en: `DateFormatter` → "May 2026" (use `DateFormat('MMMM yyyy', 'en')`)

**Day subline date format:**
- ja: "5月3日" (`DateFormat('M月d日', 'ja')`)
- zh: "05月03日" (`DateFormat('MM月dd日', 'zh')`)
- en: "May 3" (`DateFormat('MMM d', 'en')`)

## Amount Formatting Rules

Per CLAUDE.md mandate — `NumberFormatter` must be used for all monetary values. Never use raw `intl.NumberFormat` directly in widgets.

| Slot | Format | Example (JPY) | Example (USD) |
|------|--------|---------------|---------------|
| Day cell compact amount | `NumberFormatter.compact(amount, locale, currencyCode)` | 1.2万 | 1.2k |
| Summary month total | `NumberFormatter.format(amount, locale, currencyCode)` | ¥12,345 | $123.45 |
| Summary day subline | `NumberFormatter.format(amount, locale, currencyCode)` | ¥3,400 | $34.00 |

**Amount style:** `AppTextStyles.amountSmall` (15dp, w700, `FontFeature.tabularFigures()`) for month total only. Day compact uses `AppTextStyles.micro` (10dp, w700) without the `amountSmall` token — cells are too narrow for amountSmall font size.
