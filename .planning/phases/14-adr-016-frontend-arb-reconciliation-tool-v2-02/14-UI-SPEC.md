# Phase 14 — UI Design Contract

> Visual and interaction contract for ADR-016 Frontend + ARB Reconciliation.
> Generated by gsd-ui-researcher. Consumed by gsd-planner, gsd-executor, and gsd-ui-checker.

---

## 1. Product Boundary

Phase 14 makes the ADR-016 Joy migration visible and coherent. It changes HomeHeroCard, AnalyticsScreen Variant epsilon, Settings target configuration, and ARB vocabulary. It does not introduce charts, trend windows, family target semantics, gamification, or cross-period comparisons.

### In scope

| Surface | Contract |
|---------|----------|
| HomeHeroCard single mode | Keep the existing three-ring structure. Outer ring becomes monthly Joy target progress. Center shows cumulative Joy plus active target. Supporting metrics stay lower hierarchy. |
| HomeHero ring color | Outer ring and center main number transition smoothly from sage green `#47B88A` to gold across 0%-100%; at and beyond 100% they freeze at gold. |
| AnalyticsScreen | Variant epsilon places Joy Index / cumulative `joyContribution` as the first mini-hero KPI. Density, Joy/yen, ROI, and density-trend UI remain removed. |
| Settings | Add a nullable monthly Joy target numeric control with neutral recommendation display. Blank means use recommendation or fallback. |
| ARB | ja/zh/en parity. User-facing copy says Joy Index / 悦己指数 / ときめき指数. Dart/ARB key names use JoyContribution semantics, not JoyPerYen or ROI. |

### Out of scope

| Item | Reason |
|------|--------|
| Joy trend chart or custom time windows | Phase 15 owns time-window redesign. |
| Family-mode target semantics | Phase 16 or later owns family Joy expansion. |
| Any badge, milestone, streak, toast, snackbar, notification, haptic, or celebration animation | Blocked by ADR-012 and ADR-016. |
| Formula copy in product UI | Formula correctness belongs in ADRs, code, and tests. |
| Cross-period comparison or target delta copy | Blocked by ADR-012. |

## 3. Spacing Scale

Use the existing 4px-based scale. Do not introduce one-off spacing unless required by current widget structure.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Info-icon gaps, label/value micro gaps |
| sm | 8px | ListTile secondary spacing, KPI tile internal rhythm |
| md | 16px | Screen horizontal padding, HomeHero section rhythm, Settings section padding |
| lg | 24px | Lower-density screen bottom and section breathing room |
| xl | 32px | Analytics group separation |
| 2xl | 48px | Major visual separation when a section needs more air |
| 3xl | 64px | Analytics scroll bottom inset |

Exceptions:
- HomeHeroCard root padding should normalize to `16px` when this surface is touched, with goldens regenerated.
- HomeHero ring remains `120x120` unless a golden proves overflow in ja/zh/en. If adjusted, keep a fixed square dimension and update all 0/50/100/>100 goldens.
- Settings uses Material `ListTile` rhythm for consistency with existing sections.

## 4. Typography

Use existing `AppTextStyles`; do not create a new text scale.

| Role | Style | Size | Weight | Line Height |
|------|-------|------|--------|-------------|
| Label | `AppTextStyles.caption` | 12px | 500 | 1.3 |
| Body | `AppTextStyles.bodyMedium` | 14px | 500 | 1.4 |
| Section heading | existing section heading style | 18px | 700 | 1.25 |
| KPI display | `AppTextStyles.amountLarge` | 30px | 700 | 0.9 |

Numeric Joy values must use tabular figures via existing amount styles. Do not scale font size with viewport width. If the center needs two lines, use fixed line styles and `FittedBox` only around the numeric cluster, not around whole copy blocks.

## 5. Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Existing `context.wmBackground` / `context.wmCard` | Screen backgrounds and cards |
| Secondary (30%) | Existing muted backgrounds, `context.wmBackgroundDivider`, `context.wmSoulTagBg` | Tracks, dividers, KPI tile backgrounds |
| Accent (10%) | `#47B88A` to `#D9A441` | Joy progress affordance only |
| Supporting metrics | Existing olive/accent/shared palette | Middle/inner rings, Best Joy, family mode |
| Destructive | Existing Material red / project destructive styling | Settings destructive actions only; not part of target UI |

Accent reserved for:
- Outer HomeHero single-mode Joy target ring.
- Center main Joy number color.
- Joy Index KPI tile border/title accent.

Do not flood the HomeHero card background, supporting rows, icons, or Settings recommendation copy with gold. Gold is an ambient state color on the progress affordance, not a celebration treatment.

### Joy progress color function

| Progress | Outer ring color | Center main number |
|----------|------------------|--------------------|
| 0% | `#47B88A` | `#47B88A` |
| 50% | linear or perceptually smooth interpolation between `#47B88A` and `#D9A441` | same interpolated color |
| 100% | `#D9A441` | `#D9A441` |
| >100% | `#D9A441`; no oscillation, second cycle, glow, or pulse | `#D9A441`; displayed Joy value remains uncapped |

Implementation may extract a pure helper for color interpolation. If it does, unit-test 0%, 50%, 100%, and >100%.

## 6. HomeHero Contract

### 6.1 Single-mode ring semantics

Single mode keeps exactly three rings:

| Ring | Metric | Fill contract | Color |
|------|--------|---------------|-------|
| Outer | Monthly Joy target progress | `null` when Joy is empty; otherwise `min(joyContribution / activeTarget, 1.0)` | Smooth `#47B88A` to `#D9A441` |
| Middle | Average satisfaction | Existing `avgSatisfaction / 10` | Existing olive palette |
| Inner | Highlights count | Existing `highlightsCount / totalSoulTx` when total > 0 | Existing accent palette |

The outer ring uses capped progress. The displayed cumulative Joy number is not capped.

### 6.2 Active target resolution

Resolve the target before the pure HomeHero presentation widget receives data:

1. `AppSettings.monthlyJoyTarget` when non-null and valid.
2. `monthlyJoyTargetRecommendationProvider(bookId, currencyCode)` when it returns `Value<int>`.
3. Phase 13 fallback baseline when recommendation is `Empty`.

The planner may either extend `HomeHeroCard` constructor parameters with resolved target data or introduce a thin `ConsumerWidget` wrapper. The inner visual widget must remain testable with plain values.

### 6.3 Center content

Center content in single mode must show:

| Line | Content | Style |
|------|---------|-------|
| Primary | Cumulative Joy via `formatJoyCumulative(joyContribution, currencyCode)` | Numeric, tabular, colored by progress |
| Secondary | Active target reference, localized as compact "of target" copy | Caption/body small, neutral text |

Do not show a percentage. Do not show `>100%`. Do not show formula text. Empty state shows a neutral dash or localized no-data text already used by the card.

### 6.4 Legend and tooltip copy

Rename live single-mode copy from Joy/yen density to JoyContribution semantics:

| Current | Required |
|---------|----------|
| `homeJoyPerYenLegend` | `homeJoyContributionLegend` or equivalent |
| `homeJoyPerYenTooltip` | `homeJoyContributionTooltip` or equivalent |
| `homeJoyIndexTooltip` density text | Rewrite to ring semantics: outer target progress, middle average satisfaction, inner highlights |

Tooltip copy explains the concept in natural language: "monthly accumulated Joy Index toward your active target". It must not show the formula, base, exponent, Kahneman-Tversky citation, Joy/¥, ROI, or density language.

### 6.5 100% behavior contract

At and beyond 100%, HomeHero must have:

| Forbidden behavior | Contract |
|--------------------|----------|
| Toast/snackbar/dialog/notification | Must not be emitted |
| Haptic feedback | Must not be emitted |
| Celebration animation, glow, pulse, badge | Must not be emitted |
| Achievement copy | Must not appear |
| `>100%`, `120%`, or equivalent percentage | Must not appear |

Allowed behavior is only continuous value-change animation already used for ring repaint or number/color interpolation. There is no threshold branch that triggers a special effect.

### 6.6 Group mode

Group mode is not redesigned. Preserve current group ring semantics and layout unless an ARB rename is required for vocabulary consistency. Do not add family target copy, family progress, or per-member Joy target semantics.

## 7. Analytics Variant Epsilon Contract

### 7.1 KPI mini-hero ordering

`KpiMiniHeroStrip` order changes to:

1. Joy Index / cumulative `joyContribution`.
2. Total spending.

The Joy tile is the first scan target and remains the same height as the spending tile. Do not add a separate large Joy card.

### 7.2 Joy tile content

| Element | Content |
|---------|---------|
| Label | Joy Index / 悦己指数 / ときめき指数 |
| Primary value | `formatJoyCumulative(report.joyContribution, currencyCode)` |
| Supporting line | Average satisfaction, median, and/or coverage may remain as secondary context |
| Semantics | Localized sentence naming Joy Index, value, rated count, and total soul transaction count |

Empty state uses localized neutral copy. It must not mention density, Joy/¥, ROI, or formula details.

### 7.3 Deleted density surfaces stay deleted

Do not rebuild:
- Joy/yen trend chart.
- Density KPI strip item.
- Density distribution/story copy.
- ROI or happiness-density labels.

The AnalyticsScreen section rhythm remains AppBar + MonthChipPicker + KPI mini-hero + Time + Distribution + Stories. Phase 15 owns any future time-window or trend redesign.

## 8. Settings Target UI Contract

### 8.1 Placement

Add a new `JoyTargetSection` rather than overloading `AppearanceSection`. Place it after `VoiceSection` and before `DataManagementSection`, because the target is personal metric configuration rather than backup/security/family infrastructure.

### 8.2 Section structure

| Element | Contract |
|---------|----------|
| Section heading | Localized "Joy target" / "悦己目标" / "ときめき目標" equivalent |
| Current target row | Shows configured value when set; otherwise shows recommendation/fallback as active target |
| Numeric input | Nullable integer input; digits only; no decimals; no currency symbol |
| Clear action | Allows returning to recommendation-driven target |
| Recommendation copy | Neutral reference: "Based on the past 3 months, recommended X" when available; fallback copy when not enough data |
| Validation | Reject zero, negative, non-integer, and obviously invalid values with localized problem + solution copy |

The UI may use a ListTile that opens a dialog, or an inline compact input row. Prefer dialog if inline keyboard/focus behavior would make the Settings list unstable.

### 8.3 Forbidden target framing

Settings recommendation copy must not include:
- higher/lower/above/below comparison.
- `+N`, `-N`, arrows, red/green comparison.
- target-raising prompt.
- achievement, milestone, or "beat target" language.
- cross-month delta language.

Configured target and recommendation may both be visible, but only as two neutral facts.

### 8.4 Persistence and refresh

Saving target calls `SettingsRepository.setMonthlyJoyTarget(int?)`, then invalidates `appSettingsProvider` and any affected target/recommendation consumers needed for HomeHero refresh. Do not introduce a Drift table. Do not persist fallback as if the user configured it.

## 9. Copywriting Contract

All user-facing text must go through `S.of(context)` and ja/zh/en ARB in lockstep.

| Element | en | zh | ja |
|---------|----|----|----|
| Product term | Joy Index | 悦己指数 | ときめき指数 |
| HomeHero target reference | of target | 目标 | 目標 |
| Settings recommendation | Based on the past 3 months, recommended {value} | 基于过去 3 个月，建议 {value} | 過去3か月をもとに、目安は {value} |
| Settings fallback | Recommendation appears after 3 months of soul records | 记录满 3 个月后会显示建议值 | 3か月分の悦己記録があると目安が表示されます |
| Invalid input | Enter a whole number greater than 0 | 请输入大于 0 的整数 | 0 より大きい整数を入力してください |
| Clear target action | Use recommendation | 使用建议值 | 目安を使う |

Exact translation may be refined during ARB implementation, but the meaning and forbidden framing are locked.

### Copy removals

Remove or rename live ARB keys and call sites containing:
- `homeHappinessROI`
- `homeJoyPerYen*`
- `analyticsCardTitleJoyTrend`
- `analyticsCardCaptionJoyTrendGap`
- "Joy/¥"
- "density"
- "ROI"
- `幸福密度`
- `ハピネス密度`
- `快乐ROI`
- `幸せROI`

If a generated localization member remains after removal, rerun `flutter gen-l10n`; do not hand-edit generated files.

## 10. Accessibility and Semantics

| Surface | Contract |
|---------|----------|
| HomeHero ring | Semantics must name Joy Index progress as target progress without exposing percentage beyond 100. If percentage is needed for screen readers, clamp to 100 and pair with absolute Joy value. |
| Info icons | Existing info-icon tap behavior stays isolated from card tap. Tooltip text is localized and formula-free. |
| Settings input | Dialog/input has label, current value, clear action, action-specific confirmation/discard labels, and localized validation. Numeric keyboard where supported. |
| Analytics KPI | Joy KPI semantics updated from average satisfaction to cumulative Joy Index. |
| Color | Gold progress cannot be the only signal. Numeric value + target text must communicate state. |

---

## 11. Animation and Motion

Allowed:
- Existing ring repaint/value-change motion.
- Smooth interpolation of ring sweep and color over ordinary Flutter rebuild animation if already present or implemented as a small, deterministic animation under 600ms.

Forbidden:
- Threshold-only animation at 100%.
- Pulse, glow, scale bounce, confetti, badge, snackbar, haptic, or notification.
- Any motion that starts only because the target was reached.

## 12. Testing and Golden Contract

### Required HomeHero states

| State | Fixture | Expected |
|-------|---------|----------|
| 0% | Joy 0, target 50 | Empty or zero sweep; start color; no percent |
| 50% | Joy 25, target 50 | Half outer ring; interpolated color; center shows 25 and target 50 |
| 100% | Joy 50, target 50 | Full gold outer ring; center shows 50 and target 50; no event/copy |
| >100% | Joy 80, target 50 | Full gold outer ring; center shows 80 and target 50; no `>100%` |

Regenerate HomeHero goldens for all four states and review manually for ja/zh/en overflow risk.

### Required automated checks

| Area | Check |
|------|-------|
| HomeHero math | Widget/unit tests for target resolution and `min(current / activeTarget, 1.0)` |
| HomeHero 100% | Widget/source checks for no snackbar, haptic, notification, threshold copy, pulse, `>100%` |
| Ring color | Unit test helper or golden states 0/50/100/>100 |
| Analytics | Joy Index first in `KpiMiniHeroStrip`; primary value uses `joyContribution`; density strings absent |
| Settings | Blank clears target; valid integer persists; invalid input does not persist; recommendation copy is neutral |
| ARB | ja/zh/en parity; `flutter gen-l10n`; stale density grep gates |

### Grep gates

These must return no live UI hits after implementation:

```bash
rg -n "joyPerYen|homeHappinessROI" lib/ --glob "*.dart"
rg -n "Joy/¥|density|ROI|幸福密度|ハピネス密度|快乐ROI|幸せROI" lib/l10n/app_*.arb
```

Planner may allow generated-file hits only if they disappear after ARB removal and `flutter gen-l10n`.
