> Visual and interaction contract for AnalyticsScreen full rebuild as a unified 2-region dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker; revised 2026-05-03 after SCOPE simplification (3-region → 2-region) and IA selection (Variant δ).

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage in Phase 11 |
|-------|-------|-------------------|
| xs | 4px | Icon gaps, tooltip inner padding, KPI tile sub-line gap, card title ↔ body gap |
| sm | 8px | Compact element spacing, themed-group inner card gap, chart axis label gap |
| md | 16px | Default widget padding, card inner padding, KPI mini-hero horizontal padding |
| lg | 24px | AnalyticsScreen content top padding |
| xl | 32px | Inter-themed-group vertical break (時間 ↔ 分布 ↔ 物語) |
| 2xl | 48px | Empty-state region (thin-sample text fallback Card vertical padding) |
| 3xl | 64px | Page-level top safe-area + AppBar offset |

Exceptions:
- **Chart canvas heights:** `LineChart` height 180px, `BarChart` height 200px, `PieChart`/`Donut` height 180px — non-multiple-of-4 dictated by fl_chart tick density and label readability. Justified by chart-specific UX, not a general spacing exception.
- **KPI tile height:** 92px (gap=4 × content padding=14 × text-line-heights) — nominally derived from content; planner may set fixed height via `AppDimensions` if needed.
- **Touch targets:** all interactive controls (month chip, info `ⓘ` icons, CTA buttons) ≥ 44 × 44px hit area, achieved via `Padding` outside the icon glyph.

## Typography

Phase 11 declares **4 sizes × 2 effective weights** sourced from existing `AppTextStyles` (no new token introduction). Reuse-only contract:

| Role | Token | Size | Weight | Line Height | Usage in Phase 11 |
|------|-------|------|--------|-------------|-------------------|
| Body | `bodyMedium` | 14px | 500 | 1.5 (default) | Chart axis labels, card body copy, FamilyInsightCard sentence, Best Joy story body |
| Label | `caption` | 12px | 500 | 1.4 | Coverage caption ("n=k rated"), chart legend, gap-vs-zero policy text, histogram `5`-bar trilingual annotation, KPI tile labels |
| Heading | `titleLarge` | 16px | 700 | 1.2 | Themed-group headers (時間 / 分布 / 物語), card titles |
| Display (amounts) | `amountLarge` | 30px | 700 | 0.9 | KPI mini-hero values (総支出 ¥amount, 悦己平均 mean) |
| Display (sub-amounts) | `amountMedium` | 18px | 700 | 1.2 | Best Joy strip ¥ amount, chart tooltip values, secondary KPIs |
| Display (chart axis) | `amountSmall` | 15px | 700 | 1.2 | Y-axis numeric labels on `LineChart`, `BarChart`, `PieChart` legend (forces tabular figures → aligned ticks) |

**Effective weights:** `500` (body / labels) and `700` (titles / amounts / micro). `titleSmall` (600) and `bodyLarge` (600) are NOT used in Phase 11 — keeps the surface to a strict 2-weight rhythm.

**Forbidden in Phase 11:**
- `headlineLarge / headlineMedium / headlineSmall` (Phase 10 hero territory; Analytics dashboard uses `titleLarge` instead)
- `dividerLabel` (no all-caps section dividers — group structure uses spacing + soft "━ Title ━" header, not eyebrow labels)
- Hand-rolled `TextStyle(...)` literals — every text element MUST cite an `AppTextStyles.*` token

| Role | Value (light) | Value (dark) | Usage |
|------|---------------|--------------|-------|

| Secondary (30%) | `AppColors.card` `#FFFFFF` | `AppColorsDark.card` `#252836` | Card frames neutral fill (themed group containers, AppBar) |
| Accent — **悦己 cards** | `AppColors.soul` `#47B88A` + soul-tint surface `#F0F8F4` | `AppColors.soul` + `AppColorsDark.tagGreen` `#1E3028` | 悦己 KPI tile fill, 悦己 chart-card fill, Joy/¥ trend line stroke, satisfaction histogram default-bar fill, Best Joy strip emphasis, FamilyInsightCard sentence highlight |
| Accent — **総 cards** | `AppColors.survival` `#5A9CC8` + 総-tint surface `#F0F6FB` | `AppColors.survival` + `AppColorsDark.tagBlue` `#1E2D3D` | 総 KPI tile fill, 総 chart-card fill (6 か月推移 / 類別支出 / 今月の最大支出), 総 card title color |
| Accent — **家族 card** (FamilyInsight only) | `AppColors.olive` `#8A9178` + family-tint surface `#FFF7E6` (warm ochre wash) + stroke `#F0E0BC` | `AppColors.olive` + dark equivalent `#3D3528` | FamilyInsightCard fill (only widget that uses ochre — semantically distinct from per-ledger cards) |
| Destructive | not used | not used | **No destructive actions in Phase 11** — read-only dashboard |

**Accent reserved for (悦己 cards — `AppColors.soul`):**
- `LineChart` Joy/¥ trend stroke and dot fill (only when MetricResult.Value)
- Satisfaction histogram default fill (per-bar gradient overrides per D-10 below)
- Best Joy strip "category · date" emphasis underline
- 悦己 KPI tile label tint
- FamilyInsightCard sentence highlight tokens (category name + count)
- Card titles prefixed 「悦己 · ...」 use this color

**Accent reserved for (総 cards — `AppColors.survival`):**
- 6 か月支出推移 `BarChart` strokes and current-month bar fill
- 類別支出分布 `PieChart`/`Donut` legend swatches (cycled with category-specific tokens; the chart's primary tint anchor)
- 総 KPI tile label tint (総支出 label color)
- Card titles prefixed 「総 · ...」 use this color

**Accent reserved for (家族 card — `AppColors.olive`):**
- FamilyInsightCard title prefix「家族 · ...」 color
- Card stroke + ochre fill identifies it visually as "anti-leaderboard family aggregate" — distinct from per-ledger 総/悦己 cards

**Histogram per-bar gradient (D-10 — cool→warm dual-hue, 1 cool → 10 warm):**

| Bar (sat value) | Color (light) | Source |
|-----------------|---------------|--------|
| 1 (cool start) | `AppColors.survival` `#5A9CC8` | reuse existing survival blue — semantically NEUTRAL ordinal, NOT "negative" (see ADR-014 guard below) |
| 2 | interpolate(survival 70%, soul 30%) | linear blend (computed in code) |
| 3 | interpolate(survival 50%, soul 50%) | linear blend |
| 4 | interpolate(survival 30%, soul 70%) | linear blend |
| 5 | `AppColors.soul` `#47B88A` (soul green, **median** band) | the median position — receives the trilingual annotation (see Copywriting) |
| 6 | interpolate(soul 70%, olive 30%) | linear blend (`AppColors.olive` `#8A9178`) |
| 7 | interpolate(soul 50%, olive 50%) | linear blend |
| 8 | interpolate(soul 30%, accent 70%) | toward `AppColors.accentPrimary` `#E85A4F` (warm coral) |
| 9 | interpolate(soul 10%, accent 90%) | linear blend |
| 10 (warm end) | `AppColors.accentPrimary` `#E85A4F` | reuse existing primary coral — semantically TOP-end of unipolar positive scale |

**Critical guard (ADR-014 unipolar positive scale, D-10 binding tension):**
- The cool→warm gradient is **purely ordinal visual differentiation**. Bar 1 represents "low intensity of positive feeling," NOT a negative emotion.
- The starting hue is `AppColors.survival` (a neutral cool blue used elsewhere as a CATEGORY token), explicitly **NOT** red/orange or any hue conventionally read as "bad / sad / dissatisfied."
- A persistent caption below the histogram MUST display: **「色は ordinal 表現で 1 は「低強度のポジティブ」を示します。否定的な感情ではありません」 / 「色彩仅为 ordinal 视觉区分，1 代表"低强度的积极感受"，不是负面情绪 (per ADR-014)」 / "Bar colors are ordinal only. Bar 1 means a low-intensity positive feeling, not a negative emotion (per ADR-014)."**
- `accessibilityLabel` / `Semantics` on each bar MUST read the satisfaction value + count + total only — **forbidden** to include words like "差/悪い/bad/不好/低/不满/dissatisfied/sad" in any language. Phase 11 code reviewer is the final gate.

**Forbidden in Phase 11:**
- Inline hex literals in widget code (`Color(0xFF...)`) — must reference `AppColors.*` or `context.wm*`
- New color tokens added to `app_colors.dart` (Phase 12 ARB rename pass also touches no colors per milestone-start decision "不动色调")
- Multi-color overlay charts where each color encodes a member identity (D-11/D-12 binding ban)
- Any color use suggesting Joy ROI / happiness share / soul% framing (Phase 10 D-02 binding)
- Red-orange or yellow-red gradient on histogram bar 1-3 (cultural "bad" reading; ADR-014 tension)

## Interaction Contracts

### Month chip
- Range: from earliest `transactions.timestamp` month through current month (inclusive). Future months disabled.
- Tap → opens month-picker bottom sheet (`showModalBottomSheet`); selection writes via `selectedMonthProvider.notifier.setMonth(...)`; all chart/headline providers automatically invalidate (already keyed by `(bookId, year, month)`).
- Chip displays current month + small `▼` glyph; tap target ≥ 44px via outer Padding.

### Charts (`fl_chart`)
- **`LineChart` (Joy/¥ trend):** baseline-anchored y-axis (min = 0, no auto-zoom into negative); dotData visible only on days with data; line segments break across no-data days (segmented `LineChartBarData` per D-06). Tap or long-press shows tooltip with `{date} · ¥{amount} · sat {sat}/10`. Tooltip dismisses on tap-outside.
- **`BarChart` (histogram):** y-axis baseline 0; bars 1-10 always rendered (zero-count bars show as 1px stub at baseline for visual continuity). Tap shows tooltip with `sat {sat}/10 · {count}`. Bar 5 annotation rendered as text label above the bar (NOT inside tooltip — must be permanently visible).
- **`BarChart` (6-month trend, 総 card):** 6 bars (M-5 through M); current month highlighted via stroke + bolder fill. Tap on any bar shows month total tooltip.
- **`PieChart`/`Donut` (category distribution, 総 card):** top-N categories by Yen amount + "その他" bucket; legend below chart with category name + amount + percent. Tap on slice highlights it (visual ring around the slice) and dims others.
- All charts: `≥ 44 × 44px` total touch target enforced via outer `Padding` + `GestureDetector`. Color contrast verified against WCAG AA in both light + dark themes.

### Cards
- All cards support pull-to-refresh on the AnalyticsScreen Scaffold level (single `RefreshIndicator`); refresh invalidates all consumed providers.
- Group-mode FamilyInsightCard appears in 物語 group as the THIRD card (after 総 今月の最大支出 and 悦己 ベスト ジョイ); does NOT appear in single mode.
- Empty/error states render PER-CARD (not per-group): if 6 か月推移 errors but 類別支出 is fine, only the 6 か月 card shows the error; the rest of the dashboard renders normally.

### Best Joy strip (悦己 card in 物語 group)
- Tap → navigate to transaction detail screen (`Navigator.push(... TransactionDetailScreen(transactionId))`). Long-press → no action (avoid quick-action surfaces around what should be a savored moment).

### 今月の最大支出 strip (総 card in 物語 group)
- Tap → navigate to transaction detail screen for the largest single transaction. Same gesture contract as Best Joy.

### Phase 10 hero card → AnalyticsScreen integration
- `Navigator.push(... AnalyticsScreen(bookId))` — **no `initialSection` / `initialTab` / `scrollTo` parameter**. Variant δ scroll-top default places 悦己平均 KPI tile + 時間 group's 悦己 card immediately visible after the mini-hero strip; the "悦己 first impression" is achieved by structure, not navigation logic.
- The `home_screen.dart` onTap call site is updated by the planner — if Phase 10 widget exposes an `initialSection` param, it can be removed.

### Theme switching
- Light ↔ dark is system-driven via `Theme.of(context).brightness`. All colors resolved through `context.wm*` extensions or `AppColors.*` constants. **No widget hardcodes a brightness check.** Charts honor `context.wmTextPrimary` for axis labels and ledger-tinted fills resolved via theme-aware extensions.

- All chart bars / line points / pie slices expose `Semantics(label: ...)` reading the data point in the user's locale.

- Touch targets ≥ 44 × 44px (per CLAUDE.md mobile pitfall).
- Histogram bar-color gradient meets WCAG AA contrast against `context.wmCard` background in both themes (verified visually in Pencil before code; spot-checked at executor stage).
- ADR-014 guard caption is announced by screen readers as a region-level descriptor on the histogram chart.
- KPI mini-hero tile colors (総 light blue + 悦己 light green) meet WCAG AA against `context.wmTextPrimary` for value/label text.
- Empty/error/thin-sample states render a Semantics-friendly heading + body + CTA chain (not icon-only).
