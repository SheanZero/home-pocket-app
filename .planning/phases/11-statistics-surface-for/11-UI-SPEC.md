**Histogram per-bar gradient (D-10 — cool→warm dual-hue, 1 cool → 10 warm):**

| Bar (sat value) | Color (light) | Source |
|-----------------|---------------|--------|
| 1 (cool start) | `AppColors.survival` `#5A9CC8` | reuse existing survival blue — semantically NEUTRAL ordinal, NOT "negative" (see ADR-014 guard below) |
| 2 | interpolate(survival 70%, soul 30%) | linear blend (computed in code) |
| 3 | interpolate(survival 50%, soul 50%) | linear blend |
| 4 | interpolate(survival 30%, soul 70%) | linear blend |
| 5 | `AppColors.soul` `#47B88A` (soul green, **median** band) | the median position — receives the trilingual annotation (see Copywriting) |
| 6 | interpolate(soul 70%, olive 30%) | linear blend (`AppColors.olive` `#8A9178`) |
| 7 | interpolate(soul 50%, olive 50%) | linear blend |
| 8 | interpolate(soul 30%, accent 70%) | toward `AppColors.accentPrimary` `#E85A4F` (warm coral) |
| 9 | interpolate(soul 10%, accent 90%) | linear blend |
| 10 (warm end) | `AppColors.accentPrimary` `#E85A4F` | reuse existing primary coral — semantically TOP-end of unipolar positive scale |

**Critical guard (ADR-014 unipolar positive scale, D-10 binding tension):**
- The cool→warm gradient is **purely ordinal visual differentiation**. Bar 1 represents "low intensity of positive feeling," NOT a negative emotion.
- The starting hue is `AppColors.survival` (a neutral cool blue used elsewhere as a CATEGORY token), explicitly **NOT** red/orange or any hue conventionally read as "bad / sad / dissatisfied."
- A persistent caption below the histogram MUST display: **「色は ordinal 表現で 1 は「低強度のポジティブ」を示します。否定的な感情ではありません」 / 「色彩仅为 ordinal 视觉区分，1 代表"低强度的积极感受"，不是负面情绪 (per ADR-014)」 / "Bar colors are ordinal only. Bar 1 means a low-intensity positive feeling, not a negative emotion (per ADR-014)."**
- `accessibilityLabel` / `Semantics` on each bar MUST read the satisfaction value + count + total only — **forbidden** to include words like "差/悪い/bad/不好/低/不满/dissatisfied/sad" in any language. Phase 11 code reviewer is the final gate.

**Forbidden in Phase 11:**
- Inline hex literals in widget code (`Color(0xFF...)`) — must reference `AppColors.*` or `context.wm*`
- New color tokens added to `app_colors.dart` (Phase 12 ARB rename pass also touches no colors per milestone-start decision "不动色调")
- Multi-color overlay charts where each color encodes a member identity (D-11/D-12 binding ban)
- Any color use suggesting Joy ROI / happiness share / soul% framing (Phase 10 D-02 binding)
- Red-orange or yellow-red gradient on histogram bar 1-3 (cultural "bad" reading; ADR-014 tension)

## Interaction Contracts

### Month selector
- Range: from earliest `transactions.timestamp` month through current month (inclusive). Future months disabled.
- Tap chevron → `selectedMonthProvider.notifier.setMonth(...)`; all chart/headline providers automatically invalidate (already keyed by `(bookId, year, month)`).
- Forward chevron disabled when `selectedMonth == currentMonth` (visual: opacity 0.4, no tap response).

### Charts (`fl_chart`)
- **`LineChart` (Joy/¥ trend):** baseline-anchored y-axis (min = 0, no auto-zoom into negative); dotData visible only on days with data; line segments break across no-data days (segmented `LineChartBarData` per D-06). Tap or long-press shows tooltip with `{date} · ¥{amount} · sat {sat}/10`. Tooltip dismisses on tap-outside.
- **`BarChart` (histogram):** y-axis baseline 0; bars 1-10 always rendered (zero-count bars show as 1px stub at baseline for visual continuity). Tap shows tooltip with `sat {sat}/10 · {count}`. Bar 5 annotation rendered as text label above the bar (NOT inside tooltip — must be permanently visible).
- Both charts: `≥ 44 × 44px` total touch target enforced via outer `Padding` + `GestureDetector`. Color contrast verified against WCAG AA in both light + dark themes.

### Region cards
- IA-dependent (Tab swipe / scroll-to / push), but all regions support pull-to-refresh that invalidates all consumed providers.
- Group-mode FamilyInsightCard appears at the TOP of the 悦己 region (above Joy/¥ trend), not at the bottom.

### Best Joy strip
- Tap → navigate to transaction detail screen (`Navigator.push(... TransactionDetailScreen(transactionId))`). Long-press → no action (avoid quick-action surfaces around what should be a savored moment).

### Phase 10 hero card → AnalyticsScreen integration
- IA-A (Tab): `Navigator.push(... AnalyticsScreen(bookId, initialTab: AnalyticsTab.joy))`
- IA-B (Single-Page-Folded): `Navigator.push(... AnalyticsScreen(bookId, scrollToJoyLedger: true))`
- IA-C (Entry-Points): `Navigator.push(... AnalyticsScreen(bookId))` — main view IS 悦己
- The `home_screen.dart` onTap call site is updated by the planner once IA is locked.

### Theme switching
- Light ↔ dark is system-driven via `Theme.of(context).brightness`. All colors resolved through `context.wm*` extensions or `AppColors.*` constants. **No widget hardcodes a brightness check.** Charts honor `context.wmTextPrimary` for axis labels and `context.wmCard` for chart backgrounds.
