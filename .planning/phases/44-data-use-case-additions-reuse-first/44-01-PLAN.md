    - Given an L2-grain `CategoryBreakdown` list where two L2 leaves share one L1 parent, the helper returns ONE L1 entry whose amount = sum of the two leaves' amounts and whose transactionCount = sum of the two counts.
    - Given a `CategoryBreakdown` whose `categoryId` is itself an L1 category (level==1, parentId==null — a transaction filed directly on the L1 per Pitfall 2), the helper rolls it up to that same L1 id (ancestor of a level-1 id is itself).
    - The returned L1 entries are sorted amount-descending.
    - A `topN` parameter (default 10) truncates to the top-N L1 entries for the donut; without truncation the full L1 list is returned.
    - A subtotal/count lookup by L1 id returns the rolled-up amount + count for that L1 (the drill summary's source-of-truth in Plan 03); a missing L1 id returns zero/absent, not an error.
    - Empty input → empty output.
  </behavior>
