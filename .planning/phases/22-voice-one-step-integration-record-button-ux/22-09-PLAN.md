Wave 1 / Plan 09 — surgical code fix for the two BLOCKER gaps. Touches exactly ONE file (`lib/features/accounting/presentation/screens/voice_input_screen.dart`) and exactly four methods (`_onStatus`, `_onError`, `_onLongPressStart`) plus one new helper (`_showRecognitionErrorToast`) plus one new field (`_hasPermanentError`).

**Gap G-01 fix (recognizer self-termination):** When the platform speech recognizer self-terminates via `_onStatus('done')` or `_onStatus('notListening')` while the user is still holding the mic (`_pressStart != null`), `_onStatus` currently flips `_isRecording = false` and the eventual finger release falls through both guards in `_onLongPressEnd` — transcript is silently dropped. The fix: route `_onStatus` into the same commit path as `_onLongPressEnd` when the user is still holding. Idempotency is preserved by clearing `_pressStart` BEFORE invoking the commit path, so the subsequent `_onLongPressEnd` short-circuits at its existing `start == null` guard.

**Gap G-02 fix (silent error swallow + WR-05 non-localized message):** `_onError` currently ignores `errorMsg` and `permanent`. The fix: surface a localized SoftToast via a new helper `_showRecognitionErrorToast(errorMsg, permanent)` that switches on the platform error code (`error_network` / `error_no_match` / `error_audio` / else) and renders the matching `voiceRecognitionError*` ARB string (added in Plan 22-08). For `permanent == true`, the fix sets a new `_hasPermanentError` flag that `_onLongPressStart` checks at its top guard, disabling new presses until the screen is rebuilt (next `_initSpeechService` call clears the flag).

**Scope discipline (gap closure):**
- DO NOT touch any other method (`_startRecording`, `_stopRecordingAndCommit` body, `_cancelRecordingAndDiscard`, `_onResult`, `_parseVoiceInput`, `_parseFinalResult`, `_onSavePressed`, `build`, etc.).

```dart
/// G-02 / WR-05: surface a localized SoftToast for speech-recognition
/// failures reported via _onError. Maps the platform's English-only error
/// code (errorMsg) to one of the 4 voiceRecognitionError* ARB strings
/// added in Plan 22-08. Mirrors _showPermissionError's overlay pattern
/// verbatim (same positioning, same dismissal lifecycle).
void _showRecognitionErrorToast(String errorMsg) {
  final l10n = S.of(context);
  final String message;
  // Platform speech_to_text error codes per 22-REVIEW.md WR-05.
  // error_permission is intentionally not mapped here — _initSpeechService
  // surfaces voiceMicrophonePermissionRequired for the cold-start denial;
  // if a permission error reaches _onError mid-session it falls through
  // to Unknown, which is acceptable copy.
  switch (errorMsg) {
    case 'error_network':
    case 'error_network_timeout':
      message = l10n.voiceRecognitionErrorNetwork;
      break;
    case 'error_no_match':
      message = l10n.voiceRecognitionErrorNoMatch;
      break;
    case 'error_audio':
      message = l10n.voiceRecognitionErrorAudio;
      break;
    default:
      message = l10n.voiceRecognitionErrorUnknown;
      break;
  }

  final overlay = Overlay.of(context);
  late OverlayEntry entry;
  entry = OverlayEntry(
    builder: (_) => Positioned(
      top: MediaQuery.of(context).padding.top + 16,
      left: 0,
      right: 0,
      child: SoftToast(
        message: message,
        icon: Icons.error_outline,
        onDismissed: () => entry.remove(),
      ),
    ),
  );
  overlay.insert(entry);
}
```
