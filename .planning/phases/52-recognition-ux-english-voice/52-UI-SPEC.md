## Spacing Scale

The project has **no `app_spacing.dart` token file**; spacing is expressed as inline EdgeInsets/SizedBox multiples of 4. New affordances MUST stay on this 4-point grid and match the established voice-entry rhythm (260623-0cj fixed a **12dp** keyboard-row gap as the canonical inter-row spacing).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Iconв†”label gap inside a chip; intra-chip padding |
| sm | 8px | Chip-to-chip horizontal gap; compact vertical gaps |
| md | 12px | Canonical inter-row gap in the voice/keyboard column (260623-0cj); band/chip row top padding |
| lg | 16px | Form field vertical spacing; section padding |
| xl | 24px | Major block separation within the form |

**Exceptions (touch-target floors вЂ” these OVERRIDE the grid, do not shrink them):**
- Tappable alternate chips: **min 44px** touch height (HIG floor, 260623-0cj precedent вЂ” chips are interactive).
- Voice record pill / central square / action-row keys: existing **44dp** floor (260623-0cj R2), number keys **48dp** floor (v1.3). Phase 52 does not change these; the re-mocked `VoiceRecordPanel` inherits them.
- The "е…¶д»–/ж›ґе¤љ" exit chip is a chip в†’ same 44px floor.

## Typography

All new text reuses `AppTextStyles` (`lib/core/theme/app_text_styles.dart`). Family is `Outfit`; only **two weights** appear on the new recognition surface: **w500/w600 (label/medium)** and **w700 (emphasis/amount)** вЂ” consistent with the existing entry screen.

| Role | Style token | Size | Weight | Line height |
|------|-------------|------|--------|-------------|
| Chip label (category name on alternate chips) | `AppTextStyles.labelMedium` | 12px | w600 | default |
| Body / field text (merchant name, form fields) | `AppTextStyles.bodyMedium` | 14px | w500 | default |
| Selected-category label (the resolved category in the form) | `AppTextStyles.titleSmall` | 14px | w600 | default |
| Amount (header / display) | `AppTextStyles.amountMedium` / `amountSmall` | 18 / 15px | w700 | 0.9 / default (tabular figures) |

**Hard rules:**
- The confidence **band carries NO visible text** (CONTEXT D-03) вЂ” there is no band typography role. Band conveys confidence by color/icon-intensity only; the only "text" is an **a11y-hidden `Semantics` label** (screen-reader only, never painted).
- Correction sheet / exit-chip-opened full category selector reuses the **existing category-selector typography** вЂ” do not introduce new heading sizes.
- Amounts always use `AppTextStyles.amount*` (tabular figures) вЂ” never a generic body style for monetary values (CLAUDE.md rule).

## Color

Reuse ADR-019 `AppPalette` tokens only (verified hex below from `lib/core/theme/app_palette.dart`). **No new color literals** вЂ” anti-toxicity + `color_literal_scan` will reject raw hex. Light values shown; dark variants resolve automatically via `context.palette` on `AppPalette.dark`.

| Role | Token (light hex) | Usage |
|------|-------------------|-------|
| Dominant (60%) | `background` `#FBF7F4` (warm cream) | Form/screen surface |
| Secondary (30%) | `card` `#FFFFFF` / `backgroundMuted` `#F3EDE8` | Form card, default (unselected) chip fill |
| Accent (10%) вЂ” daily | `daily` `#5FAE72`, `dailyText` `#2E6B3A`, `dailyLight` `#EEF6EC` | Band/chip when resolved category is a **ж—Ґеёё** category (ledger-derived intensity) |
| Accent (10%) вЂ” joy | `joy` `#D98CA0`, `joyText` `#A53D5E`, `joyLight` `#FBEAEF` | Band/chip when resolved category is a **ж‚¦е·±** category |
| Primary (selection emphasis) | `accentPrimary` / `borderInputActive` `#6FA36F` | Selected-category border / active chip outline (the leaf-green primary) |
| Borders / dividers | `borderDefault` `#E6DDD8`, `borderDivider` `#EAE1DC` | Chip outline (unselected), correction-sheet dividers |
| Muted text | `textSecondary` `#71877A`, `textTertiary` `#A8BCB2` | Weak-band de-emphasis, exit-chip label |
| Destructive | `error` `#E5484D` | **Voice-record "stopped/reset" state only** (`Icons.restore` on red вЂ” existing `_CentralSquare`, 260622-nhs R7). NO destructive *data* action is introduced this phase. |

**Accent reserved for (explicit list вЂ” NOT "all interactive elements"):**
1. The confidence band's intensity expression (border depth / a small dot), keyed to the resolved category's ledger (daily-green vs joy-pink family).
2. The currently-selected alternate chip's active outline (`accentPrimary` leaf-green) vs unselected chips (`borderDefault` outline on `backgroundMuted` fill).
3. Existing voice-record pill (FAB sakura gradient, unchanged from 260623-0cj) and the red stopped-state square (`error`).

**Confidence band в†’ color mapping (CONTEXT D-03, ADR-012-safe, NO numbers):**
- `ConfidenceBand.strong` в†’ full-intensity category-family accent (e.g. solid 2px `daily`/`joy` border or a filled dot).
- `ConfidenceBand.medium` в†’ reduced intensity (e.g. `dailyLight`/`joyLight` fill or 1px border).
- `ConfidenceBand.weak` в†’ minimal / `textTertiary`-level de-emphasis, paired with prominently-shown alternate chips (low-confidence path в†’ user confirms/corrects, never auto-stamped).
- Band is **cleared the instant the user selects any category** (CONTEXT D-09 в†’ becomes user-authoritative, no "guess" marker).

**Destructive actions:** none in this phase. Merchant name is editable/clearable plain text (CONTEXT D-16) but clearing it is **not** a destructive-confirmation flow and teaches no table. The only "reset" affordance is the voice panel's snapshot-restore (`onReset` в†’ `ManualEntrySnapshot`), already-built, no new confirmation dialog.

## Phase-Specific Interaction Contract

(Non-template section вЂ” load-bearing interaction rules the executor must honor. Sourced from CONTEXT.md decisions.)

| Rule | Behavior | Source |
|------|----------|--------|
| Band appears on resolve-on-final | Band + chips render at the first end-of-speech `isFinal` category resolution; visible alongside the record panel if still on-screen. | D-08 |
| Chip limit | Top **~3** alternates by reconciler rank (conflict-demoted merchant default category included, L2-deduped) **+ 1 exit chip**. Never overflow; always an escape to the full selector. | D-04 |
| Tap = instant swap, teaching deferred | Tapping a chip / selector immediately swaps category + re-derives ledger (instant UI feedback) but writing `category_keyword_preferences` is **deferred to actual transaction save** (avoids polluting the learning table from reset / abandoned drafts). | D-05 |
| Both chip and full selector count as a correction | Any change away from the recognized original category вЂ” via chip OR full selector вЂ” records a correction at save. Form state must remember the "recognized original category" to detect change. | D-06 |
| Write key = `resolvedKeyword` verbatim; null в†’ don't teach | Learning write key == read key (260526-pg6). If `resolvedKeyword` is null (bare merchant / both-weak / no keyword), the entry writes **no** learning row and **never** the merchant table. | D-07 |
| Band clears on user selection | Once the user picks any category, band clears (user-authoritative); chips may collapse. | D-09 |
| Manual entry в†’ no affordance | No recognition outcome в†’ no band/chips. | D-10 |
| Voice-recognition language decoupled | Session speech locale (zh/ja/en) is independent of `currentLocaleProvider`; threaded end-to-end to STT + both recognizers + number/currency parsing. Goldens must override the correct locale provider (guard v1.8 WR-04 regression). | D-15 |
