<objective>
Add findByBookIds (one-shot query) and watchByBookIds (reactive stream) to TransactionDao. The watch variant is the backbone of LIST-02 — it eliminates ref.invalidate by having Drift auto-push new results whenever any write touches the transactions table.

Purpose: Without these DAO methods the multi-book list screen (Phase 26+) cannot exist. SC#1 verifies single-SQL correctness; SC#2 verifies reactivity; SC#4 verifies soft-delete-only hash-safety contract.

Output: Two new methods in transaction_dao.dart; test file covering SC#1 (multi-book, filters, ORDER BY), SC#2 (watch emits on 3 write types), SC#4 (softDelete does not touch hash fields; full-chain verifyChain still valid).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caller → findByBookIds / watchByBookIds | bookIds list is caller-supplied; DAO trusts its input — callers pass only the bookIds they are authorized for (enforced in Phase 26 provider). DAO does not validate authorization. |
| ORDER BY column | SortField enum values map to compile-time string constants in a switch statement; user input never reaches the ORDER BY clause. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-02-01 | Tampering | findByBookIds IN (?) clause | mitigate | All bookId values are passed as `Variable.withString(bookId)` — SQLite parameterized binding; no string concatenation of user values into SQL. |
| T-24-02-02 | Tampering | ORDER BY column name | mitigate | Column name derived from `switch (sortField)` with compile-time string literals; enum input can only select from the three pre-approved column names. SQL injection not possible via enum. |
| T-24-02-03 | Information Disclosure | bookId cross-leak | accept | DAO receives an authorized bookIds list from caller; no authorization check at DAO level — trust boundary is at the provider/use-case tier (Phase 26). Document: "DAO trusts bookIds input; caller is responsible for authorization." |
| T-24-02-04 | Tampering | softDelete / hash chain | mitigate | softDelete() writes only isDeleted=true + updatedAt via TransactionsCompanion (no currentHash/prevHash fields set). Physical row preserved. SC#4 test proves hash fields are unchanged post-soft-delete. |
| T-24-02-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan — extends existing Drift DAO pattern only. |
