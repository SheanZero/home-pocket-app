# Phase 45 — UI Design Contract

> Visual and interaction contract for the presentation-shell rebuild. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## ⚠️ Scope Banner — ZERO VISUAL CHANGE (locked by D-A1)

**Phase 45 is a PURE STRUCTURAL, BEHAVIOR-PRESERVING refactor.** The analytics screen MUST render **pixel-identically** before and after. `golden 保绿` (goldens stay green) is an explicit success criterion (REDES-01). There is **no new visual direction to decide** — the design system already exists and is live.

The work is: extract 7 inline `_*Card` ConsumerWidgets from the 739-LOC `analytics_screen.dart` into `presentation/widgets/cards/*.dart`, promote the shared `_AnalyticsDataCard` shell, and introduce a typed card registry driving both render order and `_refresh()` invalidation. **No new cards, no IA reorder, no chart polish, no animation, no color change, no copy change.**

This UI-SPEC's job is therefore to **codify the EXISTING visual/interaction contract as the invariant to preserve** — not to design anything. Every value below is "current state, preserve verbatim." Each is sourced from the live codebase or a locked ADR.

**Explicitly DEFERRED to Phase 46 (do NOT specify here):** round-5 B IA reorder (trend-to-top + donut hero), card add/remove (drop KPI mini-hero / 值得 headline / 记忆故事; add 悦己花在哪 / 小确幸日历), fl_chart 1.2.0 native per-rod labels / donut `cornerRadius`, warm-color motion, drill-down pushed-route host. Golden re-baseline is deferred to Phase 47.

If a checker dimension below appears "thin," that is **correct** — the design contract for this phase is "render exactly as today."

## Spacing Scale

> **Preserve-as-is.** The analytics shell body today uses the values below (sourced from `analytics_screen.dart:91-206`). The project baseline is an 8-point scale, but the existing shell carries two real exceptions (`12px` inter-stat gap, `14px` card padding). Because D-A1 forbids any visible change, these exceptions **MUST be preserved verbatim** — "fixing" them to the nearest 8-multiple would change rendering and break goldens. The extraction must move these spacers/paddings unchanged.

Declared values (current shell + card chrome):

| Token | Value | Usage (current, preserve verbatim) |
|-------|-------|-------|
| xs | 4px | Title→caption gap inside `_AnalyticsDataCard` (`:730`) |
| sm | 8px | Section-header → card gap; inter-card gaps within a section (`:107,118,125,137,145,160,177,186,196`) |
| inner | 12px | Caption → child gap inside `_AnalyticsDataCard` (`:732`) — **8-pt exception, preserve** |
| card-pad | 14px | `_AnalyticsDataCard` inner `EdgeInsets.all(14)` (`:725`); Card `borderRadius` also 14 (`app_theme.dart:23`) — **8-pt exception, preserve** |
| md | 16px | `SingleChildScrollView` horizontal + top padding `EdgeInsets.fromLTRB(16,16,16,24)` (`:91`) |
| body-btm | 24px | Scroll body bottom padding (`:91`) |
| xl | 32px | Pre-section-header gaps (Time / Distribution / Stories) (`:103,114,173`) |
| 3xl | 64px | Trailing bottom spacer (`:206`) |

Per-card `.when(loading)` placeholder heights (preserve verbatim — they hold layout during async): 110, 120, 260, 280px depending on card (`:373,386,434,477,538,551,600,648,696`).

Exceptions: `12px` inner gap and `14px` card padding/radius are intentional carry-overs from the live design — **preserved, not corrected**, because Phase 45 is behavior-preserving. Any change to a spacer or padding value is out of scope (→ Phase 46) and would fail the golden gate.

## Typography

> **Preserve-as-is.** All analytics text already flows through `AppTextStyles` (`lib/core/theme/app_text_styles.dart`) and theme text styles. Monetary values use the `amount*` family (tabular figures). No size/weight changes this phase. The "3-4 sizes / 2 weights" guideline does not apply to a behavior-preserving refactor of an existing live design system — the established type ramp below is the invariant to preserve.

Amount type ramp (current, preserve verbatim — `app_text_styles.dart`):

| Role | Size | Weight | Notes |
|------|------|--------|-------------|
| `amountLarge` | 30px | w700 (bold) | Tabular figures; hero/large monetary values |
| `amountMedium` | 24px | w700 (bold) | Tabular figures; card monetary values |
| `amountSmall` | 18px | w700 (bold) | Tabular figures; inline monetary values |

Title / caption / label styles inside cards come from the existing `_AnalyticsDataCard` shell and theme text styles (titles ~w600/w700, captions muted). These are moved verbatim during extraction; no new styles introduced.

**Hard rule:** Never substitute a generic text style for a monetary value (CLAUDE.md "Amount Display Style"). Extraction must keep every amount on its current `AppTextStyles.amount*` style.

## Interaction Contract (behavior to preserve)

> Phase 45 preserves all observable interaction; the only mechanism change is internal (registry-driven refresh). Recorded so the checker/auditor can confirm no observable behavior drifted.

| Interaction | Current behavior (preserve verbatim) |
|-------------|--------------------------------------|
| Scroll | `RefreshIndicator` → `SingleChildScrollView(AlwaysScrollableScrollPhysics)` → `Column(crossAxisAlignment: stretch)`. Keep `SingleChildScrollView` (existing `analytics_screen_test.dart` flings it via `find.byType`). |
| Pull-to-refresh | Invalidates the visible cards' providers. New mechanism: `registry.where(isVisible).expand(refreshTargets)` (D-B2) — observably identical to today's hand-written `_refresh()`. |
| Time window | `TimeWindowChip` in AppBar (unchanged). |
| Joy metric variant | `JoyMetricVariantChip` in AppBar (unchanged); toggling it MUST NOT touch HomeHero (GUARD-01 / `home_screen_isolation_test`). |
| Conditional cards | `_FamilyCard` + family `PerCategoryBreakdownCard` show only in group mode — now expressed as registry `isVisible: (ctx) => ctx.isGroupMode` (D-B4); solo-mode visible output unchanged. |
| Per-card loading | Fixed-height placeholders (see Spacing) hold layout during async; preserve heights. |
| Drill-down | **Not built this phase.** Pushed-route host (D-C1) lands in Phase 46; Phase 45 reserves nothing. |
