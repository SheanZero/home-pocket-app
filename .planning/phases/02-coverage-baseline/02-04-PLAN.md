## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test runtime -> lcov.info | A flaky or partially-failing test suite produces a misleading lcov; the baseline becomes the "before" image of a broken suite |
| Local environment -> committed artifacts | The artifacts committed reflect the developer's local machine state; if the lcov was generated on a different SDK version than CI, drift can appear |
| FROZEN baseline (D-08) -> fix-phase planners | Phases 3–6 trust this baseline as the canonical "before"; if it is wrong, every fix-phase scoping decision is downstream wrong |
