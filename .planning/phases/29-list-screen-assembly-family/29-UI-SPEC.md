## Spacing Scale

The list feature uses a compact 2/4/6/8/16 step set already established in
`list_transaction_tile.dart` + `list_sort_filter_bar.dart`. Declared values for THIS phase:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xxs | 2px | Tag chip vertical padding (`vertical: 1`ŌåÆ visual 2); micro inter-row gap |
| xs | 4px | Inter-chip gap inside the same group (member chips adjacent) |
| sm | 6px | Tag chip horizontal padding (matches ledger tag `horizontal: 6`) |
| md | 8px | Gap between chip groups; gap before member chip on tile trailing edge |
| lg | 16px | Bar horizontal padding (`EdgeInsets.symmetric(horizontal: 16)`) |

Exceptions:
- Filter bar fixed height **44px** (existing `ListSortFilterBar` container height ŌĆö member +
  Mine-only chips MUST fit this row; do NOT grow the bar height).
- Touch targets: chips use `MaterialTapTargetSize.shrinkWrap` (matches existing bar chips);
  the bar is horizontally scrollable so added family chips extend the scroll extent, not height.
- 8-point scale not applicable ŌĆö this is a dense data row + chip bar conforming to Phase 28
  precedent, not a greenfield layout.

## Color

This phase does NOT alter the 60/30/10 base. It introduces ONE new role: the **member
attribution accent**, drawn from the EXISTING `AppColors.shared*` family (the established
group-mode / family accent ŌĆö terracotta), which is semantically correct (family/group) and
visually distinct from survival blue and soul green.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `AppColors.background` | `#FCFBF9` warm ivory | List + bar background |
| Secondary (30%) | `AppColors.card` | `#FFFFFF` | Tiles, unselected chips |
| Survival accent | `AppColors.survival` / `survivalLight` | `#5A9CC8` / `#E8F0F8` | Ledger tag (survival) ŌĆö UNCHANGED |
| Soul accent | `AppColors.soul` / `soulLight` | `#47B88A` / `#E5F5ED` | Ledger tag (soul) ŌĆö UNCHANGED |
| **Member accent (NEW role)** | `AppColors.shared` / `sharedLight` / `sharedBorder` | `#D4845A` / `#FFF0E0` / `#F0DCC8` | **Member attribution chip + member filter chip selected state** |
| Selected chip surface | `AppColors.backgroundMuted` | `#F5F4F2` | "All" chip selected neutral state (matches existing ledger-All chip) |
| Destructive | `Colors.red` | ŌĆö | Swipe-delete background + confirm button (existing tile, UNCHANGED) |

**Member accent reserved for (explicit list ŌĆö never "all family UI"):**
1. The per-row member attribution chip (`tagBgColor = AppColors.sharedLight`,
   `tagTextColor = AppColors.shared`).
2. The SELECTED state of a per-member filter chip in the bar.
3. NOT used for: Mine-only chip selected state (see Copywriting/interaction ŌĆö Mine-only uses
   the neutral selected treatment `backgroundMuted` to read as "scope toggle", not "a member"),
   ledger tags, calendar, or amounts.

**Member color strategy (D-discretion resolved):** use a SINGLE member-neutral accent
(`AppColors.shared*`) for ALL members ŌĆö do NOT derive a per-member hue. Rationale: (a) avoids
inventing N hardcoded hex (CLAUDE.md forbids), (b) the emoji avatar already disambiguates
*which* member, (c) one accent keeps the chip's job = "this is a family row" unambiguous and
distinct from the two ledger colors. Per-member derived color is explicitly DEFERRED.

## Component Contracts (this phase's load-bearing visual decisions)

### CC-1 ŌĆö Member attribution chip (FAM-02 / D-01)
- **Container:** reuse the EXISTING ledger-tag container in `list_transaction_tile.dart`
  (lines 134ŌĆō145): `BorderRadius.circular(3)`, padding `EdgeInsets.symmetric(horizontal: 6, vertical: 1)`, text `AppTextStyles.micro`.
- **Color:** `bgColor = AppColors.sharedLight`, `textColor = AppColors.shared` (distinct from
  survival/soul). Driven the SAME way as the ledger tag (caller-provided color params).
- **Content:** `"{emoji} {shortName}"` e.g. `­¤É╗ Õż¬ķāÄ` ŌĆö emoji from `ShadowBookInfo.memberAvatarEmoji`, name from `memberDisplayName`.
- **Position:** SECOND visual element on the tile **trailing edge**, placed BETWEEN the info
  column and the amount. Layout: `[ledger tag] ŌĆ” merchant/category col (Expanded) ŌĆ” [member chip] [8px gap] [amount]`. The amount stays rightmost (alignment anchor preserved).
- **Truncation (discretion resolved):** chip text is single-line with `maxLines: 1,
  overflow: TextOverflow.ellipsis`, constrained to `maxWidth Ōēł 72px`. Short-name rule:
  display the FULL `memberDisplayName` and let ellipsis clip; do NOT pre-truncate to N chars.
  Emoji always renders (it is the primary disambiguator); name clips first.
- **Own rows:** render NOTHING. Driven by data: `taggedTx.memberTag == null` Ō¤║ own/solo row
  Ō¤║ no chip. No `isOwn` branch in UI.

### CC-2 ŌĆö Member filter chips + Mine-only chip (FAM-03 / FAM-04 / D-02, D-03)
- **Host:** appended into the EXISTING single-row horizontally-scrolling `ListSortFilterBar`
  (44px pinned bar). Family chips render ONLY when `isGroupMode == true` (D-04).
- **Chip widget:** `ActionChip`, `AppTextStyles.caption` label, `MaterialTapTargetSize.shrinkWrap` ŌĆö identical construction to existing ledger/category chips.
- **Order within the bar (discretion resolved):** family segment goes at the **END** of the
  scrolling row, after the category chip and before the conditional clear chip:
  `Sort ŌåÆ Direction ŌåÆ All(ledger) ŌåÆ ńö¤ÕŁś ŌåÆ ķŁé ŌåÆ Category ŌåÆ Search ŌåÆ ŌĆ¢ [Mine only] ŌåÆ [memberŌéü] [memberŌéé] ŌĆ” ŌĆ¢ ŌåÆ [Clear]`.
  An `8px` group gap separates the search/ledger segment from the family segment.
- **Single-select mutex group** (member chips + Mine-only + the implicit "All members" state):
  - Tap member chip ŌåÆ `setMemberFilter(thatShadowBookId)`.
  - Tap **Mine only** ŌåÆ `setMemberFilter(ownBookId)`.
  - Tap the currently-selected chip again ŌåÆ `setMemberFilter(null)` (back to All members).
  - "All members" is the `memberBookId == null` default (D-03) ŌĆö represented by NO chip
    selected (do not add a separate "All members" chip unless needed for discoverability; if
    added, it uses the neutral `backgroundMuted` selected treatment like the ledger-All chip).
- **Selected vs unselected states:**
  | Chip | Unselected | Selected |
  |------|-----------|----------|
  | Member chip | `backgroundColor: AppColors.card`, `side: AppColors.borderDefault`, label `AppColors.textSecondary` | `backgroundColor: AppColors.sharedLight`, `side: AppColors.sharedBorder` (or `AppColors.shared`), label `AppColors.shared` |
  | Mine-only chip | see prominence below | `backgroundColor: AppColors.backgroundMuted`, `side: AppColors.borderDefault`, label `AppColors.textPrimary` |
- **Mine-only prominence (discretion resolved):** Mine-only is visually distinguished from
  member chips by being **FILLED/emphasized even when UNSELECTED** ŌĆö unselected Mine-only uses
  `backgroundColor: AppColors.backgroundMuted` + leading `Icons.person_outline` (14px,
  `AppColors.textSecondary`), label `AppColors.textPrimary`. Member chips are outlined-on-card
  when unselected. This makes Mine-only read as a persistent "scope shortcut" (always visible
  in group mode per SC#5), not as a 4th member. Mine-only uses the NEUTRAL selected treatment
  (not `shared`/terracotta) precisely so it does not masquerade as a member-accent chip.
- **Member emoji in chip:** member filter chip may show `"{emoji} {shortName}"` as its label
  (reuses the attribution-chip text recipe) so the bar chip visually matches the row chip.

### CC-3 ŌĆö Pull-to-refresh (LIST-04 / D-05)
- **Host (discretion resolved):** wrap the list's scrollable in `_buildList`
  (`list_screen.dart`) with a Material `RefreshIndicator`, mirroring the established precedent
  in `analytics_screen.dart:80`. The `ListView.builder` (data state) becomes the
  `RefreshIndicator.child` and MUST get `physics: const AlwaysScrollableScrollPhysics()` so
  the gesture fires even when content is short.
- **Spinner look (discretion resolved):** default Material `RefreshIndicator` styling with
  `color: AppColors.accentPrimary` (coral `#E85A4F` ŌĆö the app's primary accent, matching the
  existing loading `CircularProgressIndicator` at `list_screen.dart:66`). No custom displacement
  or edge-offset; default Material placement under the (non-scrolling) calendar header + bar.
- **`onRefresh` behavior:** local-DB reload only ŌĆö `ref.invalidate(listTransactionsProvider)`
  **and** `ref.invalidate(calendarDailyTotalsProvider(...))` (D-05 planner note: calendar must
  also refresh so family-merged daily totals reflect any newly-landed rows). Does NOT trigger a
  P2P sync round. `onRefresh` must return a `Future` that completes when the invalidated
  providers settle (re-read via `.future` or a short settle), so the spinner dismisses honestly.
- The empty/error states are NOT scrollable by default ŌĆö wrap them in an
  `AlwaysScrollableScrollPhysics` scroll view (or `LayoutBuilder` + `SingleChildScrollView`) so
  pull-to-refresh still works when the list is empty (SC#1 must work in solo + empty).

### CC-4 ŌĆö Group-mode gating (D-04)
- The ENTIRE family cluster (member attribution chips on tiles + member filter chips +
  Mine-only chip) renders ONLY when `isGroupModeProvider == true`. Solo mode = byte-for-byte
  Phase 28 (no family controls, no dead chips ŌĆö consistent with Phase 28 D-07 "no dead control"
  philosophy). Pull-to-refresh works in BOTH modes.
