<objective>
Fix TD-1: member-filtered category-donut pull-to-refresh serves stale cached data
because `categoryDonutRefreshTargets` never lists the
`memberFilteredCategoryBreakdownProvider` that `category_donut_card.dart` watches
when a member filter is active. Thread the donut's member filter through
`AnalyticsCardContext` so the registry-derived `_refresh` union
(`registry.where(isVisible).expand(refreshTargets)`) invalidates the displayed
filtered breakdown, add the new analytics family to the registry-test whitelist,
and add a durable completeness guard so this regression class cannot silently
recur.

Purpose: a pull-to-refresh on the donut while filtered to one member must
re-fetch that member's breakdown, not serve the auto-dispose-kept cached value.
Output: threaded context field + conditional refresh target + extended isolation
test (whitelist + completeness assertion).
</objective>
