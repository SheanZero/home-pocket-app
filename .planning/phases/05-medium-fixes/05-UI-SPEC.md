# Phase 05 — UI Design Contract

> Visual and interaction contract for frontend cleanup in Phase 05. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Flutter app, shadcn not applicable |
| Preset | not applicable |
| Component library | Flutter Material 3 with project components in `lib/features/*/presentation/widgets/` |
| Icon library | Flutter `Icons`; keep existing icon choices unless a touched widget already uses another project-approved icon source |
| Font | `Outfit` for app UI; `DM Sans` only for existing bottom navigation labels |

Source: `CLAUDE.md`, `lib/core/theme/app_theme.dart`, `lib/core/theme/app_text_styles.dart`, `pubspec.yaml`.

Phase 05 is a cleanup phase. It must not introduce new visual direction, new component abstractions, new interaction patterns, or new animation behavior. Existing widget hierarchy, layout density, tap targets, chart geometry, and navigation flow must remain visually equivalent while hardcoded user-visible strings and monetary formatting are routed through the approved localization/formatting surfaces.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, chart label gutters |
| sm | 8px | Compact element spacing, list item internal gaps |
| md | 16px | Default widget spacing and horizontal screen padding |
| lg | 24px | Section padding and card internal breathing room |
| xl | 32px | Major vertical grouping between screen sections |
| 2xl | 48px | Large empty-state or hero spacing already present in touched screens |
| 3xl | 64px | Page-level spacing only where already present |

Exceptions: existing fixed chart dimensions, existing keyboard/amount input dimensions, existing bottom-navigation/FAB geometry, and platform-minimum interactive targets must be preserved exactly. Localization must not change padding, margins, radius, chart reserved label sizes, or list row heights unless an existing translated string clips in a widget test; if clipping is found, prefer wrapping or existing text-scale behavior over changing surrounding layout.

Source: default 4/8 scale aligned to existing Flutter layout conventions; behavior-preservation constraint from `REQUIREMENTS.md` and `05-CONTEXT.md`.

## Typography

Use existing `AppTextStyles` only. Phase 05 does not define new text styles.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`AppTextStyles.bodyMedium`) | 500 | existing style default |
| Label | 12px (`AppTextStyles.caption`) | 500 | existing style default |
| Heading | 18px (`AppTextStyles.headlineSmall`) | 700 | existing style default |
| Amount | 15px / 18px / 30px (`amountSmall` / `amountMedium` / `amountLarge`) | 700 | existing style default; `amountLarge` height 0.9 |

Declared weights for Phase 05 decisions: 500 and 700. Existing 600-weight widgets may remain unchanged, but Phase 05 must not introduce new text styles or new 600-weight copy treatments.

Money typography contract:

- Real monetary values must use `AppTextStyles.amountLarge`, `AppTextStyles.amountMedium`, or `AppTextStyles.amountSmall`.
- `copyWith` is allowed only when based on one of the amount styles and preserving `fontFeatures: [FontFeature.tabularFigures()]`.
- Bare `TextStyle`, generic `body*`, or chart-label-only styles must not render transaction amounts, monthly expenses, ledger spending, budget/category amounts, or amount input previews.
- Percentages, satisfaction scores, status labels, and non-money legends are not money values and must not be forced into amount styles.

Source: `05-CONTEXT.md` D-17 through D-20, `lib/core/theme/app_text_styles.dart`, `05-RESEARCH.md`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | No new phase-level CTA. Preserve each touched screen's existing command label meaning; route through `S.of(context).<key>`. |
| Empty state heading | Home recent transactions empty state: preserve Japanese meaning "取引がまだありません"; use equivalent localized ARB values for ja/zh/en with no added explanatory body unless one already exists. |
| Empty state body | Do not introduce new empty-state body copy during Phase 05. If a touched widget already has body copy, localize the existing meaning only. |
| Error state | Voice microphone permission error: preserve "マイクへのアクセスを許可してください" meaning; localized copies must describe the permission problem and tell the user to allow microphone access. |
| Destructive confirmation | None. Phase 05 must not add destructive user actions or confirmation dialogs. |

ARB and localization copy constraints:

- All user-visible copy in touched UI must use `S.of(context).<key>` and capture `final l10n = S.of(context);` inside build methods where practical.
- ARB keys must be domain-scoped and readable: use prefixes such as `home`, `analytics`, `budget`, `voice`, `chart`, or the existing feature prefix. Avoid generic keys such as `title`, `label`, `message`, or `amount`.
- Relevant home/analytics/accounting keys may be normalized in this phase, but key renames must update every generated getter call site and must be followed by `flutter gen-l10n`.
- `app_ja.arb`, `app_zh.arb`, and `app_en.arb` must have identical normal key sets and metadata key sets.
- Preserve `ocrScan`, `ocrScanTitle`, and `ocrHint` in all three ARB files, with `@key` description metadata documenting that they are future OCR/MOD-005 stubs.
- Chart labels and short units must be either ARB-backed user-visible copy or `FormatterService` output. Formatter-internal locale outputs such as compact-number `万`, relative-date labels, and localized date pieces remain allowed inside formatter tests and formatter implementation.
- Intentional CJK language data remains whitelisted: voice/NLP dictionaries, merchant seed data, date/number formatter outputs, static category localization map, and language examples in comments.

Source: `REQUIREMENTS.md` MED-03 through MED-05, `05-CONTEXT.md` D-05 through D-12, `05-RESEARCH.md`.

## Interaction Contract

Phase 05 must preserve existing behavior.

| Surface | Contract |
|---------|----------|
| Home screen copy replacement | Replacing hardcoded CJK text must not change navigation, scrolling, card order, section visibility, empty-state visibility, or transaction tap behavior. |
| Soul fullness card | Localized labels must keep the same visual hierarchy, amount alignment, satisfaction/ROI semantics, and recent soul transaction rendering. |
| Voice input screen | The microphone permission message must localize without changing permission request flow, recording controls, parsed transaction preview, or confirmation navigation. |
| Analytics charts | Axis labels, tooltip/rod labels, legend labels, and short units must be localized/formatted while preserving chart series, ranges, colors, touch behavior, and reserved label space. |
| Budget/category/summary money widgets | Currency strings must use `FormatterService.formatCurrency(amount, currencyCode, locale)` and amount typography must use `AppTextStyles.amount*`; list sorting, progress thresholds, and status copy remain unchanged. |
| Category localization helper rename | `CategoryLocaleService` rename must not change category display values, locale fallback behavior, or accounting `CategoryService` business behavior. |

Currency/formatter boundary:

- Presentation code must not import infrastructure formatters directly. Use `FormatterService` from the application layer.
- Use `FormatterService.formatCurrency(...)` for every real money string; do not concatenate `¥`, `$`, or raw numeric values in widgets.
- Use `FormatterService.formatCompact(...)` for chart axis compact numbers and `FormatterService.formatDate/formatMonthYear/formatRelative(...)` for date labels when labels are date-derived.
- Always pass the current locale from the existing locale provider or localized widget context pattern already used in the touched file.

Source: `05-CONTEXT.md` D-08, D-17 through D-20; `05-RESEARCH.md` money display hotspots and formatter evidence.

## Visual Regression Contract

The expected visual outcome is byte-identical behavior from the user's perspective, except that localized strings appear through the active locale instead of hardcoded CJK.

Regression risks the executor must test:

- Longer English labels in chart axes, cards, empty states, and budget rows must not clip, overlap, or shift chart geometry.
- Japanese and Chinese compact labels must remain legible and must not be replaced by raw formatter-internal strings outside approved formatter boundaries.
- Monetary columns must retain tabular numeric alignment after `copyWith`.
- ARB key normalization must not silently fall back to missing generated getters or stale generated localization files.
- `CategoryLocaleService` rename must not alter visible category labels.
- Scanner whitelists must not hide user-visible CJK in presentation files.

Required test expectations:

- Add or update widget tests for touched localized UI surfaces using localized wrappers from existing test helpers.
- Add focused money widget/style tests that inspect rendered `Text.style.fontFeatures` and require `FontFeature.tabularFigures()` for real money values.
- Add ARB parity test/script covering normal keys and metadata keys across ja/zh/en.
- Add CJK scanner/architecture test over `lib/**/*.dart` with explicit whitelist paths/patterns from `05-CONTEXT.md`.
- Add lib-only MOD-009 scan assertion excluding `docs/` and `doc/`.
- Run `flutter gen-l10n` after ARB changes, then verify generated files are clean and current.
