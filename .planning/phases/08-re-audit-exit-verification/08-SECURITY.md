# Phase 08 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.
>
> Phase 08 is a planning/governance phase — it produces verification scripts, CI hardening, golden tests, a re-audit pipeline, coverage gates, a smoke-test checklist, and an ADR-011 amendment. No runtime user-facing surface, no auth/identity flow, no network endpoints, no schema changes at trust boundaries.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| Filesystem reads of `.planning/audit/` JSON catalogues | reaudit_diff.dart and merge_findings.dart consume JSON shards/issues files | repo-relative paths, finding metadata (no PII) |
| `.github/workflows/audit.yml` CI gate | The 4 permanent guardrails (analyze, coverage, per-file coverage, build_runner) decide if changes can land on `main` | exit codes |
| `.claude/commands/audit/*.md` AI agent prompts | Locked per Phase 1 D-01 — modification silently changes scan semantics | prompt text |
| Golden PNG snapshots (`test/golden/goldens/*.png`) | Binary regression baseline for amount/summary/soul widgets | rendered pixels (synthetic fixtures only) |
| ADR-011 body integrity | Canonical decision record for the cleanup initiative | markdown text (append-only convention) |
| `08-06-GATES-LOG.md` audit trail | Records actual exit codes of all 8 EXIT-04 gates for ADR-011 evidence | gate verdicts + commit hash |
| Smoke test sign-off (`08-SMOKE-TEST.md`) | Human attestation that load-bearing UAT items pass | tester name + date + commit hash (audit-trail intent) |
| `cleanup-touched-files.txt` per-file coverage list | Single-line audit.yml swap from `phase6-touched-files.txt`; widens 80% gate to 50+ files | repo-relative `lib/` paths |
