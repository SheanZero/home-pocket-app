| Boundary | Description |
|----------|-------------|
| User input → Form internal state | TextField inputs (merchant, note) plus numeric keypad input cross from UI to form state; trimmed before passing to use case |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-04-01 | Tampering | Edit-mode seed mutation | mitigate | Form builds UpdateTransactionParams(seed: ..., overrides...) and delegates to UpdateTransactionUseCase; CLAUDE.md Pitfall #4 — never mutate seed; use case enforces copyWith path. |
| T-18-04-02 | Information Disclosure | Note encryption preservation through edit | mitigate | Form reads decrypted seed.note from the repo (already decrypted at read time) and re-passes the value through UpdateTransactionParams.note; UpdateTransactionUseCase passes plaintext to repo.update which re-encrypts. No plaintext logging in the form (CLAUDE.md crypto rules). |
| T-18-04-03 | Tampering | Hash chain on edit | mitigate | Form invokes UpdateTransactionUseCase which does NOT recompute hash (D-08, enforced by Plan 02). Form never touches prevHash/currentHash directly. |
| T-18-04-04 | Repudiation | Edit without updatedAt stamp | mitigate | UpdateTransactionUseCase stamps updatedAt = DateTime.now() on every execute (D-07, Plan 02). Form does NOT set updatedAt — relies on the use case. |
| T-18-04-05 | Spoofing | entry_source flip on edit | mitigate | Form's .edit branch builds UpdateTransactionParams WITHOUT entrySource. seed.copyWith default (no entrySource: arg) preserves seed.entrySource verbatim (SC-3). DAO test in Plan 08 exercises all three EntrySource literals. |
| T-18-04-06 | Elevation of Privilege | Voice learning triggered by retroactive edit | accept | D-09 — .edit-mode category change does NOT trigger recordCategoryCorrectionUseCase.
