## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| voice transcript (untrusted) → recognizer | The (already-decrypted) user query string is untrusted external input; `normalizeMerchantKey` + bounded anchored scoring are the validation boundary (ASVS V5). |
| recognizer → in-memory cache | Public non-sensitive merchant seed data; no PII. |

A local pure-Dart scorer over a pre-loaded in-memory list. No SQL is built from the transcript (the repo read in Plan 01 is parameterized; the recognizer only reads the warm cache). No network, no crypto. Most ASVS L1 web categories (auth/session/access/XSS/CSRF) are **N/A**.
