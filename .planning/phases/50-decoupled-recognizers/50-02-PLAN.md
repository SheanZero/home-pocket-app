<objective>
Expand the category-only keyword seed (`default_synonyms.dart`) from the current ~16-L2 coverage to **full coverage of speakable L2** (~103 L2, every one with ≥1 zh + ≥1 ja seed), add the `seed-keyword-categoryId-是真L2` hard gate that blocks orphan-categoryId rows, AND add a **machine set-completeness gate** that proves every speakable L2 actually has ≥1 zh + ≥1 ja seed (D-04's headline deliverable). This is the Phase-49-scale authored-DATA deliverable (mirrors how Phase 49 isolated `DefaultMerchants`).

Purpose: DECOUP-02 requires `CategoryRecognizer` to resolve L2 from activity/item keywords even with no merchant — but the current seed only covers ~16 L2, so SC4's「加油用了400块」→ `cat_car_fuel` currently misfires (only `ガス`→`cat_utilities_gas` exists). Full speakable-L2 coverage is what makes the category-only path usable for everyday utterances. Two distinct gates protect D-04: the **categoryId hard gate** catches the silent-null bug class (a typo in ~206 hand-authored rows pointing at a non-existent id), and the **speakable-L2 coverage gate** catches the OTHER failure mode — a seed that is internally valid but covers only 40 of 103 speakable L2 (the categoryId gate AND a `cat_car_fuel` grep both pass on a 40-of-103 seed, so set-completeness needs its own machine assertion, not just the human spot-check).
Output: Expanded Dart-literal seed (VOICE-06 contract — adding literals, no resolver code change) + a cloned-and-extended categoryId integrity gate + a speakable-L2 set-completeness gate. Includes a blocking checkpoint for the user spot-check (D-04 mandate, word-quality on top of the machine gates).
</objective>
