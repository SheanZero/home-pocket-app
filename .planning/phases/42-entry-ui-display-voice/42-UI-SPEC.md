# Phase 42 — UI Design Contract

> Visual and interaction contract for the multi-currency entry / display / voice phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Stack note:** This is a Flutter app, not React/Next.js/Vite — the shadcn gate is **not applicable**. The project already ships a mature design system: `AppPalette` ThemeExtension (ADR-019 "Sakura Mochi × Wakaba"), `AppTextStyles`, 48dp touch-target floor, and a golden-baseline workflow. This phase is brownfield: it **extends** existing widgets (`SmartKeyboard`, `TransactionDetailsForm`, `NumberFormatter`, transaction list rows) and **must not** introduce a new visual language. The JPY-only path is a hard regression-protection invariant (CURR-04).

## Spacing Scale

The codebase has **no dedicated spacing-token file** — spacing is expressed as ad-hoc `EdgeInsets` per widget. New widgets in this phase MUST follow the 4px-multiple scale below and align to whatever the adjacent widget already uses (e.g. `SmartKeyboard` key gaps, `TransactionDetailsForm` row padding) to stay visually consistent.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps inside a selector row, inline chip padding |
| sm | 8px | Compact spacing (preview main-row → sub-row gap, list annotation → main amount gap) |
| md | 16px | Default element spacing, sheet content padding, form row vertical rhythm |
| lg | 24px | Section padding inside `CurrencySelectorSheet`, gap above preview panel |
| xl | 32px | Major block separation in detail/edit view |

Exceptions:

- **48dp minimum touch target (NON-NEGOTIABLE):** every interactive element in `SmartKeyboard` (currency key, dot key, digit keys) and every tappable row in `CurrencySelectorSheet` MUST be ≥48dp. This is a hard floor inherited from v1.3 Phase 19 and enforced by golden baselines. The dot-key gating (D-06) must preserve this floor — if the dot key is hidden/replaced for 0-decimal currencies, the replacement key (or layout) must still satisfy 48dp and not introduce mis-tap risk.

## Typography

Roles below map to existing `AppTextStyles` constants (do not invent new sizes). 4 declared sizes for this phase's new surfaces, 2 weight tiers (semibold w600 / bold w700) consistent with the existing scale. Body/secondary text uses w500–w600.

| Role | Size | Weight | Line Height | Maps to |
|------|------|--------|-------------|---------|
| Display (JPY preview main result `≈ ¥7,415`) | 30px | 700 | 0.9 | `AppTextStyles.amountLarge` (`.copyWith(color: palette.textPrimary)`) |
| Heading (sheet title, edit-section title) | 18px | 700 | default | `AppTextStyles.headlineSmall` |
| Body / selector row label (`USD 美元`) | 15px | 600 | default | `AppTextStyles.bodyLarge` / `titleMedium` |
| Caption / sub-rows (preview rate sub-row, list annotation, staleness label) | 12px | 500 | default | `AppTextStyles.caption` |

Amount-bearing values everywhere (preview main row, list foreign annotation, edit-page read-only JPY) MUST use the `amount*` styles (tabular figures) — never plain `body*`.

## Color

Project palette is the 60/30/10 source of truth (ADR-019). All values resolved via `context.palette`; both light and dark are defined.

| Role | Value (light) | Value (dark) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | `#FBF7F4` warm cream (`palette.background`) | `#171210` | App background, sheet scrim base |
| Secondary (30%) | `#FFFFFF` (`palette.card`) | `#231E1B` | Cards, `CurrencySelectorSheet` surface, preview panel surface, form rows |
| Accent (10%) | `#6FA36F` leaf green (`palette.borderInputActive` / primary) | `#8DC68D` | Active currency selection highlight, active input border, primary CTA |
| Warning (staleness) | `#C98A00` (`palette.warning`) | `#E5B53A` | Fallback / weekend-rate staleness label ONLY (D-05) |
| Error | `#E5484D` (`palette.error`) | `#F0676B` | Validation errors only (e.g. unparseable manual rate, mandatory-rate-required) |
| Text primary | `#20352B` (`palette.textPrimary`) | `#F0EBE6` | Amounts, labels, primary copy |
| Text secondary | `#71877A` (`palette.textSecondary`) | `#9A8E87` | Sub-rows, captions, list annotation, rate-date suffix |

Accent (leaf green `#6FA36F`) reserved for:

- Active/selected currency row in `CurrencySelectorSheet` (selection check / highlight)
- Active amount-input border (`borderInputActive`) during foreign entry — existing behavior, unchanged
- Primary save/confirm affordance (existing CTA color, unchanged)

**Reserved-color discipline (hard rules):**

- **Warning amber (`#C98A00`) is reserved EXCLUSIVELY for the staleness label** (D-05: `RateResult.fallback` cached-rate, or `fetched.actualDate ≠ transaction date`). It is non-blocking/informational and MUST NOT be used for the normal rate sub-row.
- **Sakura pink (`#D98CA0`) is OFF-LIMITS here** — it belongs to the FAB and the 悦己 Joy ledger identity only. Currency UI must not borrow it.
- **Ledger colors (daily green / joy pink) keep their existing meaning** — the currency layer is orthogonal to the dual-ledger and must not recolor ledger affordances.
- **No raw hex literals** — every color via `context.palette` (ADR-019 / CLAUDE.md pitfall #).

## Copywriting Contract

All copy MUST exist in ja/zh/en ARB files via `S.of(context)`; ja is default. Examples below are intent specs — exact ARB strings + key names are the planner/executor's to finalize (CONTEXT.md "Claude's Discretion"), but must cover these cases. Rate dates render via `DateFormatter` (locale-aware: ja `06/09`, zh `06月09日`, en `06/09`).

| Element | Copy (intent — ja primary) |
|---------|------|
| Primary CTA | Existing save affordance — unchanged label (`保存` / 保存 / Save). No new CTA introduced. |
| Currency selector entry | Currency key on `SmartKeyboard` shows active `currencySymbol` + ISO code (e.g. `$ USD`); JPY default shows `¥`. |
| Selector row format (D-01) | `🇺🇸 $ USD 美元` — flag + symbol + ISO code + localized currency name. Search hits on code OR name. |
| Selector "more" affordance (D-02) | 「もっと見る」/「更多」/「More」 — expands full ISO 4217 list with real-time search. |
| Preview main row (D-03) | `≈ ¥7,415` (amountLarge, tabular). `≈` signals derived/approximate-of-original. |
| Preview rate sub-row (D-03) | `USD 1 = ¥148.30 · {rate date}` (caption, textSecondary). Same wording as detail/edit rate line. |
| Preview loading (D-04) | In-place skeleton/dimmed placeholder at the preview row — NO text jump, NO keyboard occlusion. Within P41 ~3s budget. |
| Staleness — cached fallback (D-05) | warning color: 「{date} のレート（キャッシュ）を使用」/「使用 {date} 缓存汇率」/「Using cached rate from {date}」 |
| Staleness — weekend/holiday (D-05, RATE-05) | warning color: 「{date}（直近の営業日）のレート」/「周末，采用 {date} 汇率」/「{date} (most recent business day)」 |
| Mandatory manual rate (P41 D-08) | When `RateResult.unavailable` (new currency + offline + empty cache): inline prompt requiring the user to enter a rate before save is possible. Non-blocking elsewhere — save is gated only on a present, parseable rate. |
| List foreign annotation (DISP-02) | Secondary small text on the row: `USD 50.00` (caption, textSecondary). JPY rows unchanged. |
| Detail/edit fields (DISP-03 / D-10/D-11) | Three rows always visible: original amount (editable), rate (editable, never collapsed), JPY (read-only derived). "原币是事实，日元是结果" mental model. |
| Date-change >1% toast (D-13 / ADR-022 D-03) | Non-blocking toast + 「元に戻す」/「撤销」/「Undo」 with a 5s window; rate auto-recalculated and saved immediately. |
| Date-change after manual override (D-13 / ADR-022 D-02) | Two-choice dialog, NO default: 「手動レートを保持」 vs 「新しい日付で再取得」 (keep manual rate / re-fetch for new date). |
| Empty state | N/A — this phase adds fields to existing populated entry/list/detail surfaces; no new empty-state screen. The full-ISO selector list with active search yielding no match should show a neutral "no results" line (planner to localize). |
| Error state | Validation: unparseable/non-positive manual rate → inline `palette.error` message at the rate field; save blocked until corrected. (Network failure is NOT an error here — it degrades to cached fallback per RATE-03.) |

Destructive actions: **none in this phase.** No deletes/irreversible actions are introduced. The date-change re-fetch (D-13) is reversible (Undo / explicit dialog) and is governed by ADR-022, not a destructive-confirmation pattern.

## Hard Invariants (regression protection)

1. **CURR-04 — JPY path untouched:** when JPY is active, NO rate fetch, NO preview panel, NO list annotation, NO dot key. JPY entry/list/detail goldens must remain byte-identical.
2. **48dp touch floor:** all `SmartKeyboard` keys and selector rows ≥48dp, including the dot-gated layouts.
3. **ADR-022 D-01:** edit page = two inputs (original amount + rate) + one read-only derived JPY. No three-field bidirectional loop.
4. **Single conversion site:** `convertToJpy()` (ADR-020, `.round()`) is the ONLY place preview / list annotation / edit JPY are computed — guarantees identical figures across all three surfaces.
5. **Color discipline:** `context.palette` only; warning amber reserved for staleness; sakura pink off-limits.
6. **i18n:** all new copy in ja/zh/en ARB + `flutter gen-l10n`; dates via `DateFormatter`, amounts via `NumberFormatter`.
