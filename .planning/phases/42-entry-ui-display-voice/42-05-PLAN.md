<objective>
Build the genuinely-novel pure-logic piece of this phase: a host-owned `AmountInputController` state machine for per-currency decimal-input gating (CURR-05), and apply D-06 dot-key gating to `SmartKeyboard` while preserving the 48dp floor and layout stability.

Per RESEARCH Q3, the controller is immutable-state `{digits, decimals}`: `onDigit` caps at the currency minor unit (D-07), `onDot` only valid when `decimals>0`, `onCurrencyChange` TRUNCATES the fractional substring as a STRING op (D-08 — never `.round()`, RESEARCH Pitfall 3). JPY dot behavior stays byte-identical (CURR-04).

This plan turns `amount_input_controller_test.dart` (from 42-01) GREEN and adds a dot-gating golden/widget test. Host wiring of the controller into `manual_one_step_screen` happens in 42-08.
