From coverage/lcov.info (input format the parser must handle):
```
SF:lib/core/theme/app_theme.dart
DA:6,2
DA:17,1
DA:27,0
LF:9
LH:4
end_of_record
```
- Records separated by `end_of_record`
- `SF:<path>` source file (one per record)
- `DA:<line>,<hits>` per-line hit count (zero hits = uncovered)
- `LF:<n>` total instrumented lines, `LH:<n>` lines hit
- Per-file percentage = `LH / LF * 100` (defense: if LF/LH absent or LF=0, recompute from DA lines)

From CONTEXT.md D-11 (artifact schemas):
- coverage-baseline.txt: `path\tlines_covered/lines_total\tpercentage` per line, lex-sorted by path, no header, trailing newline, percentage formatted with `.toStringAsFixed(2)`
- coverage-baseline.json: top-level `{generated_at, flutter_test_command, lcov_source, threshold, total_files, files_below_threshold, entries: [...]}`; per-record `{file_path, lines_covered, lines_total, percentage, threshold_met}`
- files-needing-tests.txt: bare `<path>` per line, lex-sorted, only files where `percentage < threshold`, trailing newline
- files-needing-tests.json: same top-level metadata; per-record `{file_path, percentage, lines_below_threshold}`; `lines_below_threshold = lines_total - lines_covered`

From CONTEXT.md D-01..D-04 (coverage_gate CLI contract):
- Defaults: `--threshold 80`, `--lcov coverage/lcov_clean.info`
- Inputs: positional `<file>...`, `--list <file>` (newline-delimited), fallback `.planning/audit/files-needing-tests.txt`
- Output: per-file `path | covered/total | % | PASS|FAIL` table to stdout
- `--json` flag emits `{checked: [...], failures: [...], threshold: N, lcov_source: "..."}` to stdout
- Exit codes: 0=pass, 1=any file below threshold, 2=invocation error (missing lcov, unknown flag, no files supplied, missing fallback)
