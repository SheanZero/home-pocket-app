## Typography

The hero card uses **3 sizes** for body content + the existing `AppTextStyles.amount*` styles (which are MANDATORY for all monetary values per CLAUDE.md "Amount Display Style"). All amount text MUST use `AppTextStyles.amountLarge/Medium/Small` — these include `FontFeature.tabularFigures()` for column alignment. Generic body styles MUST NOT be used for amounts.

### Declared Typography Tokens

| Role | Style token | Size | Weight | Line height | Usage in HomeHeroCard |
|------|-------------|------|--------|-------------|------------------------|
| Display amount | `AppTextStyles.amountLarge` | 30 | 700 | 0.9 | Hero header monthly total `¥123,456` |
| Section amount | `AppTextStyles.amountMedium` | 18 | 700 | 1.2 (default) | Member-row spending amounts; Best Joy small `¥X,XXX` ONLY where amount must be readable; ring center text (e.g. "7.8" / "27") |
| Body large (titles) | `AppTextStyles.bodyLarge` | 15 | 600 | 1.5 (default) | Ring section title "悦己充盈 / Joy Index" / "家族の小確幸 / Family Joy" + member name |
| Body medium | `AppTextStyles.bodyMedium` | 14 | 500 | 1.5 | Hero header label "今月の支出" / "家族の支出"; legend label text; "先月 ¥amount" sub-line |
| Body small | `AppTextStyles.bodySmall` | 13 | 500 | 1.5 | Coverage caption "n=23/31 rated"; legend sub-text rows; trend chip "+4%" text (existing pattern) |
| Caption | `AppTextStyles.caption` | 12 | 500 | 1.5 | "群组成员" subheader (ja: メンバー / zh: 群组成员 / en: Members) |
| Best Joy BIG | inline TextStyle | 14 | 700 | 1.2 | Best Joy strip BIG line `category · date` — fontSize 14 / fontWeight 700 / fill `context.wmTextPrimary` (D-04) |
| Best Joy tag | inline TextStyle | 9 | 600 | 1.0 | Best Joy strip tag "本月最爱 / 今月の最愛" — fontSize 9 / fontWeight 600 / **letterSpacing 1** / fill warm-orange (D-04) |
| Best Joy small | inline TextStyle | 9 | 500 | 1.2 | Best Joy strip de-emphasized line `¥X,XXX · 满足 X/10 ✨` — fontSize 9 / fontWeight 500 / fill warm-orange (D-04) |

### Hard typography constraints (from D-04, CLAUDE.md, project rules)

- **All ¥ amounts MUST use `AppTextStyles.amountLarge` / `amountMedium` / `amountSmall`.** Hero total = `amountLarge`; member spending + Best Joy embedded amount + ring center text = `amountMedium`. Never use `bodyMedium` for monetary values (loses tabular figure alignment).
- **Best Joy strip 3-level typography is locked verbatim** (D-04 with no flexibility):
  - Tag: `fontSize: 9, fontWeight: FontWeight.w600, letterSpacing: 1` filled warm-orange.
  - BIG: `fontSize: 14, fontWeight: FontWeight.w700` filled primary text color (`context.wmTextPrimary`).
  - Small: `fontSize: 9, fontWeight: FontWeight.w500` filled warm-orange.
  - The `✨` glyph in the small line is **intentional** (Spotify Wrapped argmax pattern, research line 47) — DO NOT remove.
- **Currency formatting:** `FormatterService().formatCurrency(amount, book.currency, locale)` — currency code resolves from `Book.currency` per D-12; **NEVER** hardcode `'JPY'` (CLAUDE.md Pitfall #9).
- **Date formatting:** `DateFormatter` short-month form per locale (ja `4月15日` / en `Apr 15` / zh `4月15日`); resolves locale from `currentLocaleProvider`.
- **Number formatting:** `NumberFormat('#,##0').format(value)` for integer Yen amounts (matches existing `home_screen.dart:_formatInt`).

### Hierarchy rationale

The 3 typography sizes (14 / 15 / 18 plus 30 for hero) honor the project's existing `AppTextStyles` scale. The Best Joy strip's `9 / 14 / 9` typography (D-04) is intentionally **outside** the standard scale — this is a deliberate design choice to de-emphasize amount/satisfaction relative to "what + when" (research line 47, anti-`¥10 candy` framing). The 9px font is small but intelligible on iOS 14+/Android 7+ targets at standard pixel density.

### Tabular figures gate

`FontFeature.tabularFigures()` is mandatory for all numeric columns. Already encoded in `AppTextStyles.amountLarge/Medium/Small`. Any inline `TextStyle()` for numeric content MUST add `fontFeatures: const [FontFeature.tabularFigures()]`. Code reviewer's gate.
