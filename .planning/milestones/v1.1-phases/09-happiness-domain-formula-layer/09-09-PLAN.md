    ```dart
    /// Joy density (Joy per ¥) PTVF base + locale-aware formatting.
    ///
    /// HAPPY-02 / D-04 / D-19 / D-20: this file is the single source of truth for
    /// both:
    ///   - PTVF base by currency (used by GetHappinessReportUseCase math)
    ///   - Display unit (multiplier + label) by currency (used by UI)
    ///
    /// Co-locating both maps means future currency additions (EUR, GBP, KRW)
    /// are one-file edits.
    ///
    /// Currency code lookups are case-sensitive (matching Book.currency convention).
    /// Unknown codes fall back to JPY base (500.0) — does NOT throw.

    const Map<String, double> _PTVF_BASE_BY_CURRENCY = {
      'JPY': 500.0,
      'CNY': 25.0,
      'USD': 5.0,
    };

    const Map<String, ({double multiplier, String label})> _DISPLAY_UNIT_BY_CURRENCY = {
      'JPY': (multiplier: 1000.0, label: '/ ¥1k'),
      'CNY': (multiplier: 100.0, label: '/ ¥100'),
      'USD': (multiplier: 1.0, label: r'/ $1'),
    };

    /// PTVF base for the given currency code; defaults to 500.0 (JPY) when unknown.
    double ptvfBaseFor(String currencyCode) =>
        _PTVF_BASE_BY_CURRENCY[currencyCode] ?? 500.0;

    /// Formats a raw Joy/¥ density value for display.
    ///
    /// Multiplies the raw density by the currency-specific multiplier and appends
    /// the unit label (e.g., "5.0 / ¥1k" for JPY).
    ///
    /// Display unit fallback: unknown currency → JPY display unit (multiplier 1000, '/ ¥1k').
    String formatJoyDensity(double rawDensity, String currencyCode) {
      final unit = _DISPLAY_UNIT_BY_CURRENCY[currencyCode] ??
          _DISPLAY_UNIT_BY_CURRENCY['JPY']!;
      final scaled = rawDensity * unit.multiplier;
      // Display 1 decimal place — adjust if presentation team requests different precision.
      return '${scaled.toStringAsFixed(1)} ${unit.label}';
    }
    ```

    No `intl` package needed for this formatter (the multiplier + label semantics is currency-driven, not locale-driven). If the team later wants thousand-separators on `scaled`, they can pipe through `NumberFormatter`.
