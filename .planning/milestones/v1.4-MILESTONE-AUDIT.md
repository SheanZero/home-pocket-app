## 4. Tech Debt (non-blocking)

### GAP-1 — Calendar totals stale after family-sync / FAB quick-entry (CAL-02, CAL-04)
Shell invalidates `listTransactionsProvider` after sync (`main_shell_screen.dart:93`) and FAB return (`:172`) but not `calendarDailyTotalsProvider`. List refreshes; calendar per-day totals + month summary lag until pull-to-refresh or an in-list edit/delete. Small, real Phase 26↔27 boundary bug. Fix: add `ref.invalidate(calendarDailyTotalsProvider(...))` (current month) at both sites.

### GAP-2 — LIST-02 reactive stream is dead code (LIST-02)

Works for all verified flows but the stated mechanism is unwired and shell comments (`main_shell_screen.dart:92,171`) are stale/misleading. Either consume `useCase.watch()` or delete the 3-layer chain + fix comments.
