Per-month sampling pattern (D-04):
- M-1 = DateTime(asOf.year, asOf.month - 1, 1) through DateTime(asOf.year, asOf.month, 0, 23, 59, 59)
- M-2 = DateTime(asOf.year, asOf.month - 2, 1) through end of M-2
- M-3 = DateTime(asOf.year, asOf.month - 3, 1) through end of M-3
- Dart's DateTime normalizes negative months across year boundaries automatically (RESEARCH section Sampling Strategy #4 verified)

Empty fast-path:
- If any of the 3 months returns [] (no soul rows), that month does not contribute
- If <3 months have non-empty rows, return const Empty()
- If >=3 months have non-empty rows, compute median = sorted[1] (always index 1 for length 3), then Value(median.ceil(), 3)

Fallback baseline note: the `_fallbackBaseline` const is documented in code as spike-decided per 13-SPIKE.md per D-06. It is NOT currently exposed via any execute() return path -- execute() returns Empty when <3 months. The fallback is for the eventual UI consumer (Phase 14 Settings) which renders the fallback when Empty.
