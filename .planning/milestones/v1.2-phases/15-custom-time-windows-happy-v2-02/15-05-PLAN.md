Build the user-facing selector surface: `TimeWindowChip` (AppBar action replacing `MonthChipPicker`) and `TimeWindowPickerSheet` (bottom-sheet with type-row → body-chooser flow per D-02). NO new helpers are added to `DateFormatter` or `FormatterService` — all chip labels are composed via:
- ARB placeholder substitution (analyticsTimeWindowChipLabelYear/Quarter/Week/Custom from Plan 01)
- Existing `FormatterService.formatMonthYear` for the Month variant
- Existing `FormatterService.formatShortMonthDay` for the Week variant's `{monday}` placeholder and the Custom variant's `{start}`/`{end}` placeholders
- Raw `year.toString()` / `quarter.toString()` passed directly to ARB placeholders for Year and Quarter (locale suffixes like `年` live entirely in the ARB ja/zh values)

This plan does NOT yet wire the chip into `AnalyticsScreen` — that integration + `MonthChipPicker` deletion + `_refresh` re-key live in Plan 06 alongside the locking/no-delta tests. Splitting here keeps Plan 06's task budget focused on integration + verification.

Planner-locked decisions implemented here:
- **Week label format (Pitfall #4 / Open Q3):** Date-anchored "Week of {monday}" per UI-SPEC §Copywriting Contract. Use `FormatterService.formatShortMonthDay(mondayStart, locale)` for the placeholder substitution.
- **Custom-range error UX (Open Q5):** SnackBar (NOT dialog). Light, recoverable, leaves the sheet either closed (if invalid range was confirmed via system picker) or reopens automatically on the next chip tap.
- **Earliest-data awareness (Open Q6):** Preset lists derive from `earliestTransactionMonthProvider`. Week lists show all weeks back to the earliest month; quarter lists show all quarters back to the earliest quarter; year lists show all years back to the earliest year. Future dates always capped at today (D-07).
- **No helper additions to DateFormatter/FormatterService:** The existing methods cover every chip-label case. The `年` suffix for ja/zh year labels lives in the ARB key value (`"{year}年"`), not in a Dart helper — this keeps the formatter surface stable and avoids a redundant pass-through that would duplicate localization logic.
