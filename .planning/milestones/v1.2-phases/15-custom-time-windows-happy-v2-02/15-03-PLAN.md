<objective>
Migrate the 6 month-bound analytics use cases from `(year, month)` to `(startDate, endDate)` signatures with `TimeWindowValidation.assertValid` defense-in-depth gates. Lock Option A (display-anchor convention) for the three Freezed report models that carry `int year, int month` fields, with explicit dartdoc documenting the post-Phase-15 semantics. Retire the latent ADR-012 §4 cross-period delta UI surface in `total_spending_kpi_tile.dart`.

Planner-locked decisions:

- **Pitfall #1 (year/month Freezed fields) → Option A (display-anchor).** Verified via `grep -rn '\.year\|\.month' lib/features/analytics/presentation/widgets/`: no widget consumes `report.year` / `report.month` for display. Tests reference them only as factory-input round-trips. Option A is lower risk than Option B (parallel WindowReport types) and keeps the model surface stable for HomeHero's `previousMonthComparison` consumer.

- **UI-SPEC Open Decision #2 (MoM delta retirement) → RETIRE in this phase.** `_MomDeltaSubLine` in `total_spending_kpi_tile.dart:65-83` renders `"↑ +X% MoM"` text — a latent ADR-012 §4 cross-period delta surface. Per UI-SPEC default recommendation, drop in lockstep with the ARB key retirement (Plan 01 already removed `analyticsKpiTotalDeltaIncreased/Decreased`). `MonthComparison? previousMonthComparison` field stays on `MonthlyReport` because HomeHero `home_hero_card.dart:128` and `state_shadow_books.dart:87` still consume it for current-month trend percent (NOT cross-period delta on AnalyticsScreen).

Output:
- 6 use cases with new signature + validation gate.
- 3 report models with display-anchor dartdoc.
- 6 use-case test files rewritten for new signature + ArgumentError cases.
- `total_spending_kpi_tile.dart` with `_MomDeltaSubLine` + `if (delta != null) ...` block removed.
- 1 domain-model test file (`monthly_report_test.dart`) updated for display-anchor convention.
</objective>
