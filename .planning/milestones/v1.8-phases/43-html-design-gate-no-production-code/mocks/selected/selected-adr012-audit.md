## 判定框架

把每个元素归到两类之一：

- **OK（环境化 / 庆祝过去 ambient · celebrate-past）** —— 描述「已经发生的事实」或「构成占比」，不设目标、不打分、不催促、不比较、不排名。
- **FORBIDDEN（目标 / 跨期 / 成就 goal · cross-period · achievement）** —— ADR-012 §「Forbidden Features」明令禁止：streaks、badges、daily targets、cross-period delta、public sharing、per-member breakdown、历史趋势对比。

## 跨期例外 —— 记录在案（CRITICAL）

> **本设计包含一处对 ADR-012 §4「cross-period delta」的 documented, user-approved 例外。**

- **范围:** 跨期（本月 vs 上月）**仅**出现在 `支出趋势` 的 **总支出** 与 **日常** 两个**支出侧** tab（元素 #1、#2）。
- **语义:** 实用预算 parity —— 与首页 `支出趋势` 红框对齐，帮助用户判断「这个月花得比上个月多/少」。标签全程中性（本月/上月），无「超过/落后/目标/达标/+X%」等评判或 delta 措辞。
- **悦己侧绝对清洁:** `悦己` tab（#3）与**所有悦己元素**（横条 #6、日历 #7、直方图 #8）**zero 跨期 / zero 目标 / zero 进度环 / zero 排名 / zero 连续打卡 / zero 成就框定**。悦己跨期 = 成就压力，才是 ADR-012 真正守护的毒性红线，本设计严守。
- **来源与去向:** 此例外在 `mocks/round2/ROUND2-DECISION.md`（三处设计岔口表 · 第 1 行）中由用户在被明确告知约束后选定，并记录为「须在 GATE-04 ADR go/no-go 中作为一条记录在案的决定纳入，而非静默违规」。**本自审将该例外继续向 GATE-04 carry-forward。**

## 结论

**判定：✅ PASS** —— 选定方向（`selected-light.html` / `selected-dark.html`）符合 ADR-012 反游戏化约束。

**唯一例外（已记录、非违规）:** `支出趋势` 的 **总支出** 与 **日常** 两个支出侧 tab 含本月 vs 上月跨期对比，属 ROUND2-DECISION.md 记录的 **用户批准的 ADR-012 §4 支出侧例外**，标签中性、范围严格限于支出侧，须带入 GATE-04 ADR go/no-go 在案确认。**悦己侧全部元素保持 cross-period-free / target-free / progress-ring-free / ranking-free / streak-free / achievement-free。**
