## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed ProviderException test for Riverpod 3.1.0 async behavior**
- **Found during:** GREEN phase test run — 4/5 tests passed, Test 5 "ProviderException wrapping" failed
- **Issue:** The plan spec used `throwsA(isA<ProviderException>().having(...))` on a `waitForFirstValue` result, based on CLAUDE.md note "errors thrown by providers are wrapped in `ProviderException`". This wrapping behavior only applies to synchronous provider reads. For `@riverpod Future<T>` providers in Riverpod 3.1.0, the error is stored directly in `AsyncValue.error` as the raw exception (not wrapped).
- **Fix:** Test 5 rewritten to assert `result.hasError` is true and `result.error` is `isA<StateError>()`. Also removed unused `package:flutter_riverpod/misc.dart` import.
- **Files modified:** `test/unit/features/list/presentation/providers/calendar_totals_provider_test.dart`
- **Commit:** a837db8
- **Decision logged:** Riverpod 3.1.0 async Future provider errors are raw in AsyncValue.error
