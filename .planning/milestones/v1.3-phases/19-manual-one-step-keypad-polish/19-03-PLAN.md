P19-W1 default-category async race (RESEARCH §Pitfall 5):
- `_initializeDefaultCategory()` runs in `initState` and is async (reads `categoryRepositoryProvider.findActive()`). Between mount and the future's completion, `_selectedCategory == null`.
- Tapping Save during that window would invoke `submit()` → `CreateTransactionUseCase.execute(...)` with `category: null` → use case throws or returns `validationError`. Either way: a stray Drift transaction attempt for a phantom save.
- Mitigation (P19-W1 fix): BOTH save entry points (SmartKeyboard's onNext + KeyboardToolbar's onSave) check `_selectedCategory != null` before invoking `_save()`. If null, short-circuit with a toast / SnackBar AND render the Save buttons as visually disabled (greyed gradient / lower opacity).
- Implementation: introduce a `bool get _canSave => _selectedCategory != null && !_isSubmitting;`. SmartKeyboard's `onNext: _canSave ? _save : null` — but SmartKeyboard's `onNext` is `required VoidCallback` (no null). Workaround: pass `onNext: _canSave ? _save : _showCategoryLoadingToast` AND have `_GradientKey` accept a `bool enabled` prop OR communicate disabled state via opacity. Simplest path: in ManualOneStepScreen, write `void _trySave() { if (!_canSave) { _showToast(S.of(context).preparingCategory); return; } _save(); }` and pass `onNext: _trySave`. The KeyboardToolbar accepts `isSubmitting` already; extend with `isDisabled` (or repurpose `isSubmitting` to include the "category-not-ready" state by setting `_isSubmitting = !_canSave || _isSubmitting`).

FocusNode plumbing (P19-W3 fix — RESEARCH §Pattern 3 Option 1):
- Plan 01 Task 3 extended `TransactionDetailsFormConfig.$new` with optional `merchantFocusNode` + `noteFocusNode`. This plan PASSES per-host FocusNodes into the form via that config.
- ManualOneStepScreen's state holds `late final FocusNode _merchantFocus = FocusNode()..addListener(_handleFocusChange);` and `late final FocusNode _noteFocus = FocusNode()..addListener(_handleFocusChange);`.
- `_handleFocusChange`: `final hasTextFocus = _merchantFocus.hasFocus || _noteFocus.hasFocus; if (hasTextFocus == _isTextFieldFocused) return; setState(() { _isTextFieldFocused = hasTextFocus; if (hasTextFocus) _amountFocused = false; });`. The equality guard prevents rebuild storms during soft-keyboard animation (RESEARCH §Anti-Patterns).

- Dispose: `_merchantFocus.dispose(); _noteFocus.dispose();` in `dispose()`.
