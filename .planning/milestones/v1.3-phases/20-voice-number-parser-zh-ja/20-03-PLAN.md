       ```dart
       class ChineseNumeralStateMachine extends NumeralStateMachine {
         const ChineseNumeralStateMachine();

         static const _kanjiDigits = <String, int>{
           '零': 0, '〇': 0,
           '一': 1, '壱': 1, '壹': 1,
           '二': 2, '弐': 2, '贰': 2,
           '三': 3, '参': 3, '叁': 3,
           '四': 4, '五': 5, '伍': 5,
           '六': 6, '七': 7, '八': 8, '九': 9,
         };
         static const _kanjiUnits = <String, int>{
           '十': 10, '百': 100, '千': 1000, '仟': 1000,
           '万': 10000, '萬': 10000,
         };
         // Skip set: whitespace + zh/jp currency symbols + 元/块/塊 colloquial currency words
         static final _skipPattern = RegExp(r'[\s¥￥円えんyen元块塊]');

         @override
         int? parse(String text) => scan(normalize(text));

         @override
         List<NumeralToken> normalize(String text) {
           final tokens = <NumeralToken>[];
           for (final ch in text.characters) {
             if (ch == '零' || ch == '〇') {
               tokens.add(const ZeroPlaceholder());
             } else if (_kanjiDigits.containsKey(ch)) {
               tokens.add(Digit(_kanjiDigits[ch]!));
             } else if (_kanjiUnits.containsKey(ch)) {
               tokens.add(Unit(_kanjiUnits[ch]!));
             } else if (RegExp(r'^[0-9]$').hasMatch(ch)) {
               tokens.add(Digit(int.parse(ch)));
             }
             // Anything else (whitespace, currency suffix, random text): silently drop
             // The scanner doesn't see Skip-class tokens for zh — we drop at normalize
             // rather than emit Skip and ignore later. Either approach works; we prefer
             // drop-here so the token stream stays minimal for inspection.
           }
           return tokens;
         }
       }
       ```

    IMPORTANT — `零` precedence: the dispatch order MUST check `零`/`〇` BEFORE `_kanjiDigits.containsKey` (even though `零` is in `_kanjiDigits` for completeness, the explicit ZeroPlaceholder dispatch must win). This is the fix per PATTERNS.md §Pattern Assignments — the legacy code had `'零':0` in `kanjiDigits`, which silently overwrote `currentDigit` to 0 and then the next unit applied `currentDigit==0 ? 1 : digit` fallback, double-counting.

    Per D-08: no pinyin tables, no defensive romaji handling. If recognizer ever returns pinyin (extremely unlikely), the result is null and the Arabic-numeral path in `_extractArabicAmount` is the safety net.

    Layer-direction sanity check: imports only `package:characters/characters.dart` (already in flutter SDK transitively) + sibling `numeral_state_machine.dart`. No imports from features/application/data.
