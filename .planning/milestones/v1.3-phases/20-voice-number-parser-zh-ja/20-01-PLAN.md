    Algorithm invariants the `scan` body MUST encode (Pattern 1):
    - Variables: `var total = 0, section = 0, digit = 0; var sawAny = false;`
    - Dart 3 pattern switch over each token after `_expandPacked` flattens nested PackedToken.
    - `Unit(power == 10000)` case: `total += (section + (digit == 0 ? 1 : digit)) * 10000; section = 0; digit = 0;`
    - `Unit(other power)` case: `section += (digit == 0 ? 1 : digit) * power; digit = 0;`
    - `ZeroPlaceholder()` case: `digit = 0;` (no-op effect on section/total; required so `2千2百零4` doesn't apply implicit digit=1 fallback at the next Unit)
    - `Skip()` case: `continue;`
    - `PackedToken()` case: `continue;` (already expanded)
    - Tail flush after loop: `section += digit; total += section;`
    - Return: `sawAny && total > 0 ? total : null;`

    No concrete normalize/parse here — those live in the zh/ja machines (Wave 2). Do NOT add a default normalize that returns `[]`. Use `abstract` instead.

    Layer-direction sanity check: this file imports only `package:meta/meta.dart`. No imports from `lib/features/`, `lib/application/`, or `lib/data/`.
