    2. `void main()` top-level. Use file-scope mutable counters as the suite-state pattern (flutter_test runs tests in declaration order within a group; counters update naturally).

       ```dart
       void main() {
         final machine = const ChineseNumeralStateMachine();

         // Suite-wide pass/fail counters for the aggregate reporter.
         var passCount = 0;
         var totalCount = 0;

         // ─── Anchor cases — strict, individual test() blocks ───
         group('zh anchor cases (VOICE-01 / VOICE-02 / VOICE-03)', () {
           final anchors = voiceCorpusZh.where((c) => c.note?.startsWith('anchor:') ?? false).toList();
           // Sanity: ensure all 5 expected anchors are present in the fixture.
           setUpAll(() {
             expect(anchors.length, greaterThanOrEqualTo(5),
                 reason: 'Fixture must contain ≥5 anchor cases (Plan 20-01 contract)');
           });

           for (final c in anchors) {
             test('${c.input} -> ${c.expected}  [${c.note}]', () {
               totalCount++;
               final actual = machine.parse(c.input);
               try {
                 expect(actual, c.expected,
                     reason: 'anchor case must pass strictly: input="${c.input}" expected=${c.expected} actual=$actual');
                 passCount++;
               } catch (_) {
                 rethrow;  // anchor failures are HARD failures
               }
             });
           }
         });

         // ─── Statistical bucket — non-anchor cases ───
         group('zh statistical corpus (≥95% accuracy gate)', () {
           final nonAnchors = voiceCorpusZh.where((c) => !(c.note?.startsWith('anchor:') ?? false)).toList();

           for (final c in nonAnchors) {
             test(c.input, () {
               totalCount++;
               final actual = machine.parse(c.input);
               if (actual == c.expected) {
                 passCount++;
               }
               // Per-case soft expect with inspectable failure (does not abort suite)
               // We log the failure detail via reason but allow individual mismatches
               // to count toward the ≥95% gate rather than fail-fast.
               expect(actual, c.expected,
                   reason: 'input="${c.input}" expected=${c.expected} actual=$actual note=${c.note ?? ""}');
             });
           }
         });

         tearDownAll(() {
           final pct = totalCount == 0 ? 0.0 : (passCount / totalCount * 100);
           // Print is the deliberate test reporter output per RESEARCH §Validation Architecture
           // ignore: avoid_print
           print('═══════════════════════════════════════════');
           // ignore: avoid_print
           print('zh corpus: $passCount/$totalCount (${pct.toStringAsFixed(1)}%)');
           // ignore: avoid_print
           print('═══════════════════════════════════════════');
           expect(passCount / totalCount, greaterThanOrEqualTo(0.95),
               reason: 'VOICE-03: zh corpus accuracy ${pct.toStringAsFixed(1)}% < 95%');
         });
       }
       ```

    Subtlety: the statistical bucket uses `expect` which throws on mismatch — that means each mismatched case shows up as a red test in the output AND contributes to the count via the pre-throw `if (actual == c.expected) passCount++`. The tearDownAll accuracy gate is the actual VOICE-03 contract check. If the fixture is well-designed (≥95% of cases pass), the per-case fails are inspectable diagnostics; if accuracy < 95%, the tearDownAll gate fails loudly with the aggregate %.

    Alternative simpler shape: instead of `try/expect/catch` to bump passCount, use `if (actual == expected) passCount++` BEFORE the expect — that way passCount is accurate regardless of expect throwing. Then expect provides the per-case visibility. This is what the code above does.

    Implementation note on `print`: linter may flag `avoid_print` — the inline `// ignore: avoid_print` suppression is acceptable here because this IS a test reporter (the precedent for legitimate suppression). If the project's analyzer config forbids `print` entirely, switch to `debugPrint` (already imported with `package:flutter/foundation.dart` if needed).
