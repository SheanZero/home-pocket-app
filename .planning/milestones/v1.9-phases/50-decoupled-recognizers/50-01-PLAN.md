<objective>
Build the data foundation for `MerchantRecognizer`: the domain verdict model (`MerchantCandidate`), the flat in-memory matching record (`MerchantMatchEntry`), and the `MerchantRepository.loadAllForMatching()` read path that joins Phase-49's `merchants` + `merchant_match_keys` tables in a single read transaction.

Purpose: The recognizer (Plan 03) is a thin scorer over pre-normalized match keys. It needs (a) a typed candidate to return (DECOUP-03 recall-first output, D-01 raw score only) and (b) a one-shot load-all data backend (research A5: load-all in-memory beats per-call DB query for the ~391-row table). This plan is purely additive over Phase-49 data — no schema bump (stays v22), no new DAO query (reuses existing `findAllMerchantRows`/`findAllMatchKeyRows`).
Output: Two new domain models + one new repo interface method + DAO/impl wiring + a join-correctness test.
</objective>
