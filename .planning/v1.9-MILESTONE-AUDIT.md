## Tech Debt — Review Before/After Ship

### T-01 (Decision) — Recognition surface dormant in production · RECUX-01 / RECUX-02
The confidence band and alternate chips are fully built, wired through `updateRecognition` plumbing, and unit-tested, but **do not render in production**: `transaction_details_form.dart:1247` gates on `_band != null && (showConfidenceBand || showAlternateChips)`, and both flags default `false` with no production caller enabling them. This is a deliberate, user-directed scope-cut (UAT test 2 hid the chips; a follow-up also hid the band). Reversible by flipping one default.
- **Action:** Decide consciously whether v1.9 ships with the recognition affordance OFF (current state) or re-enables the band before close. Both are valid; the milestone should not ship this divergence silently.

### T-02 (Doc) — Stale UAT note
`52-UAT.md:24` says "the confidence band still renders" — no longer true after the follow-up. Correct the note so the next reader isn't misled.

### T-03 (Confirm) — Two learning loops co-fire at save · RECUX-03
`transaction_details_form.dart:899-910` (legacy Phase-18 merchant→category hook) and `:912-934` (Phase-52 KEYWORD-only correction) both fire at the same save. The RECUX-03 contract is literally honored (merchant table never touched by the *correction* path), but the pre-existing merchant hook teaches the merchant table from the same user action. Confirm intended; not a v1.9 regression.
